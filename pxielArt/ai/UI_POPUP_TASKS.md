# UI 弹窗化改造任务清单

> 目标：将现有"硬切/平铺"式界面升级为弹窗式交互，增加动画过渡和视觉层次感。
> 道具面板（ToolPanel / PaletteInstaller）**不做修改**。

---

## T1: 我的作品 — 预览弹窗改造

| 项目   | 内容 |
|--------|------|
| **状态** | ✅ 已完成 |
| **文件** | `assets/src/ui/myworks/MyWorksPage.ts` → `_showPreviewPopup` / `_dismissPopup` |
| **目标** | 点击作品卡片后，以弹窗形式展示放大预览图 |

### 需求

1. **黑色遮罩**：全屏半透明黑色背景，`opacity = 0.5`（Color alpha = 128）
2. **白色背景卡片**：预览图后面放一个**稍大于预览图**的白色矩形（每边多出约 30px），作为"相框"效果
3. **弹出动画**：整个内容卡片（白色背景 + 预览图 + 名称文字）从 `scale(0.85)` + `opacity(0)` 弹入到 `scale(1.0)` + `opacity(255)`，时长约 0.25s
4. **名称文字**：在白色背景卡片底部显示作品名称，深灰色文字
5. **关闭行为**：点击黑色遮罩区域关闭弹窗；移除原有的"关闭"按钮
6. **关闭动画**：反向缩小 + 淡出后 `destroy`，时长约 0.2s

---

## T2: 我的作品页 → 底部滑入式 Modal Sheet

| 项目   | 内容 |
|--------|------|
| **状态** | ✅ 已完成 |
| **文件** | `assets/src/ui/myworks/MyWorksPage.ts`、`assets/src/ui/home/HomePage.ts`、`assets/src/AppRoot.ts` |
| **目标** | MyWorksPage 从独立全屏页面改为从底部滑入的 Modal 面板，覆盖在 HomePage 上方 |

### 需求

1. **遮罩**：全屏半透明黑色，`alpha = 128`，点击遮罩关闭
2. **面板**：从屏幕底部滑入，高度约占屏幕 80%，白色背景
3. **进入动画**：面板 y 从 `-viewH` 到 `0`，遮罩 opacity 从 0→128，时长 0.3s，easeOut
4. **退出动画**：反向，时长 0.25s
5. **内容**：保留原有 TopBar（标题"我的作品"）+ ScrollView 网格 + 空状态
6. **导航简化**：`AppRoot` 不再用独立 `_myWorksNode`，改为在 HomePage 上叠加弹出

## T3: 关卡详情弹窗（新增）

| 项目   | 内容 |
|--------|------|
| **状态** | ✅ 已完成 |
| **文件** | 新增 `assets/src/ui/popup/LevelDetailPopup.ts`、修改 `HomePage.ts` |
| **目标** | 点击关卡卡片不再直接进入游戏，而是先弹出关卡详情弹窗 |

### 需求

1. **遮罩**：全屏半透明黑色，`alpha = 128`，点击遮罩关闭
2. **卡片容器**：居中白色矩形，约 `viewW × 0.8` 宽、自适应高度
3. **卡片内容**：
   - 关卡预览图（较大，约卡片宽度 × 0.7 的正方形）
   - 关卡名称（Label，32 号字）
   - 网格尺寸（Label，如 "32 × 32"）
   - 颜色数量（Label，如 "8 种颜色"）
   - 当前进度（Label，如 "进度: 45%" 或 "全新关卡"）
   - 「开始挑战」按钮（绿色，点击后关闭弹窗并 `showGame(entry)`）
4. **弹出动画**：`scale(0.85→1.0)` + `opacity(0→255)`，0.25s
5. **关闭动画**：反向，0.2s
6. **预加载**：弹窗打开时即可预加载 JSON，点击「开始」时如已加载则直接使用

## T4: 结算弹窗 → 卡片式

| 项目   | 内容 |
|--------|------|
| **状态** | ✅ 已完成 |
| **文件** | `assets/src/ui/popup/CompletionPopup.ts` |
| **目标** | 将全屏白底结算改为暗色遮罩 + 白色卡片式弹窗 |

### 需求

1. **遮罩**：全屏半透明黑色替代当前全屏纯白色，`alpha = 128`
2. **卡片容器**：居中白色矩形，约 `viewW × 0.85`、`viewH × 0.65`
3. **卡片内容**：保留现有布局（标题 + 回放画布 + 按钮组），但限制在卡片内
4. **进入动画**：遮罩 opacity 渐显 + 卡片 `scale(0.9→1.0)` + `opacity(0→255)`
5. **回放/按钮逻辑**：保持现有 ReplayAnimator 流程不变

## T5: 退出确认弹窗（新增）

| 项目   | 内容 |
|--------|------|
| **状态** | ✅ 已完成 |
| **文件** | 新增 `assets/src/ui/popup/ExitConfirmPopup.ts`、修改 `GamePage.ts` |
| **目标** | 游戏中点击「返回」按钮时弹出确认弹窗，防误触 |

   - 标题「确认退出？」

   - 「继续游戏」按钮（灰色）— 关闭弹窗
   - 「退出」按钮（绿色）— 执行 onBack
4. **弹出/关闭动画**：同 T1

## T6: Toast → Snackbar 样式升级

| 项目   | 内容 |
|--------|------|
| **状态** | ✅ 已完成 |
| **文件** | `assets/src/util/Toast.ts` |
| **目标** | 为 Toast 添加深色背景容器，提升可见性 |

### 需求

1. **背景**：深色半透明矩形（`Color(50, 50, 50, 200)`），宽度自适应文字 + padding
2. **文字**：改为白色
3. **位置**：从屏幕底部偏上位置弹出（约 `y = -viewH/2 + 120`），而非屏幕中央
4. **动画**：保留上浮 + 淡出，起始位置改为底部

---

## 实施顺序

建议按 T1 → T6 → T5 → T4 → T3 → T2 的顺序，因为：
- T1 是用户明确指定的需求，最高优先级
- T6 改动最小，快速见效
- T5 改动小但提升体验
- T4 改结算弹窗，独立性强
- T3 需要新增文件 + 改 HomePage 逻辑
- T2 涉及 AppRoot 导航架构调整，影响面最大，放最后
