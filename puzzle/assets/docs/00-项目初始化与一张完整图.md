# 00 · 项目初始化与一张完整图

> **目标**：打开 Cocos Creator 3.8 → 新建空场景 → 中央显示一张 PNG 图。50 行 TypeScript。

---

## 起点回顾

零。这是第一节。

---

## 1. 新建项目

打开 Cocos Dashboard → "新建" → 选 **空模板**：

- **2D 项目**（拼图是 2D 游戏，不需要 3D 资源）
- **TypeScript** 模板
- 项目名 `cocos-puzzle`，路径自选

进去之后默认场景是 `assets/scenes/Scene.scene`，左侧资源管理器看到的样子：

```
assets/
└── scenes/
    └── Scene.scene
```

---

## 2. 准备一张拼图源图

找一张 PNG 图，**至少 300×300，最好正方形**（如果不是正方形，下一节会按短边裁成方）。比如：

- 一只猫的卡通画
- 一张风景照
- 任何你切了 9 块还能认出来的图

放到 `assets/textures/` 下（没有这个文件夹就右键 `assets` 新建 `textures`）。我后面例子里用 `kitten.png`，你换成自己的就行。

**注意**：图丢进去后，资源管理器里点一下，**右侧 Inspector 把 "Type" 改成 `sprite-frame`**（默认是 `texture`，不能直接给 Sprite 组件用）。

---

## 3. 重命名场景

把 `Scene.scene` 改成 `puzzle.scene`，加进编辑器的"启动场景"列表（点项目设置 → 启动场景 → 选 puzzle）。

---

## 4. 搭场景骨架

打开 `puzzle.scene`，层级里默认有 `Canvas`。在 Canvas 下右键新建：

- **空节点 `PuzzleRoot`**：将来拼图的全部块都挂在这下面
- **空节点 `ImageLayer`**：本节的"参考完整图"挂在这下面

层级树看起来：

```
Scene
└── Canvas
    └── PuzzleRoot
        └── ImageLayer    (本节用)
```

`PuzzleRoot` 的 Position 保持 (0, 0, 0) —— 让它和 Canvas 中心对齐，这样后面所有块的坐标都用 Board-local 算（中心是原点）。

---

## 5. 写第一个脚本

在 `assets/scripts/puzzle/` 下新建 TypeScript 文件 `PuzzleBoard.ts`（没有 `puzzle/` 子目录就先建一个）：

```typescript
import { _decorator, Component, Node, Sprite, SpriteFrame, UITransform } from 'cc';
const { ccclass, property } = _decorator;

/**
 * 拼图主控（00 节版本）
 *
 * 职责：在 ImageLayer 下创建一张 Sprite，把源图渲染成 800×800 的方框。
 * 后续每节都会扩展这个 class，最终承载整个拼图玩法。
 */
@ccclass('PuzzleBoard')
export class PuzzleBoard extends Component {
    /** 拼图源图。在编辑器拖一张 SpriteFrame 进来。 */
    @property(SpriteFrame)
    sourceImage: SpriteFrame | null = null;

    /** 拼图盘的总尺寸（屏幕像素）。所有后续计算都基于这个值。 */
    @property
    boardSize: number = 800;

    onLoad() {
        if (!this.sourceImage) {
            console.error('[PuzzleBoard] 源图未设置 — 请在 Inspector 拖入 SpriteFrame');
            return;
        }
        this.renderFullImage();
    }

    /** 在子节点 ImageLayer 下创建一个 Sprite，显示源图。 */
    private renderFullImage(): void {
        const imageLayer = this.node.getChildByName('ImageLayer');
        if (!imageLayer) {
            console.error('[PuzzleBoard] 找不到 ImageLayer 子节点');
            return;
        }

        const imageNode = new Node('FullImage');
        imageNode.parent = imageLayer;
        imageNode.setPosition(0, 0, 0);

        const transform = imageNode.addComponent(UITransform);
        transform.setContentSize(this.boardSize, this.boardSize);

        const sprite = imageNode.addComponent(Sprite);
        sprite.spriteFrame = this.sourceImage;
        sprite.sizeMode = Sprite.SizeMode.CUSTOM;
    }
}
```

---

## 6. 挂脚本 + 拖资源

回 Cocos Creator：

1. 点 `PuzzleRoot` 节点 → Inspector 右下 "Add Component" → 搜 `PuzzleBoard` → 添加。
2. 把 `assets/textures/kitten.png` **拖到** PuzzleBoard 组件的 `Source Image` 字段上。
3. `Board Size` 字段保持 800（默认值）。

层级树扩展后大概这样：

```
Scene
└── Canvas
    └── PuzzleRoot     [PuzzleBoard 组件已挂]
        └── ImageLayer
```

按 **Ctrl/Cmd + S** 保存场景。

---

## 7. 跑起来

点编辑器右上角 ▶ 预览（浏览器打开）。

**预期**：屏幕中央显示一张 800×800 的图，铺满 Canvas 大部分区域。

**控制台预期**：什么都不打印（脚本无 `console.log`，正常执行无错误）。

**不正常的信号**：

- **白屏 + 红色 error**：`源图未设置` → 你忘了拖 SpriteFrame，回 Inspector 拖。
- **白屏 + `找不到 ImageLayer 子节点`**：你没在 PuzzleRoot 下建 ImageLayer，去层级树新建。
- **图变形（被拉伸）**：源图不是正方形 + `setContentSize(800, 800)` 强行拉成方。本节先这样，01 节切碎片时会按短边裁正方形。
- **图特别小（远没填满 800×800）**：`Sprite.sizeMode` 没设 `CUSTOM`，Sprite 还在按原图尺寸显示。检查代码里那一行 `sprite.sizeMode = Sprite.SizeMode.CUSTOM`。
- **图特别大（出框了）**：你的 boardSize 设大了，或源图分辨率超高 + 手机分辨率低。改 boardSize 到合适值。

## 8. 复盘：这节教了什么

**1. Cocos 的"渲染 = Component"模型**

Cocos 3.x 走的是 **Component-based** 路线（跟 Unity 一样）：

- 节点 (`Node`) 不带渲染能力，只有 transform。
- 渲染由 Component 提供：`Sprite` 渲染图、`Label` 渲染文字、`MeshRenderer` 渲染 3D 模型。
- 一个 Node 可以挂多个 Component，组合起行为。

我们这节做的就是：**在 Node 上挂 `Sprite` + `UITransform`**。`UITransform` 提供"我是 800×800 的"，`Sprite` 提供"我用这张 SpriteFrame 渲染"。

**2. SpriteFrame ≠ Texture2D**

入门最容易踩的概念坑：

- **Texture2D**：纯 GPU 资源，一张原始图。
- **SpriteFrame**：在 Texture2D 上"框出一块矩形"的资源。游戏开发常把好几张图打包到一张大图（图集 atlas），每张小图对应一个 SpriteFrame。

`Sprite.spriteFrame = 某 SpriteFrame` → 渲染那个矩形区域。**01 节我们就要靠 `SpriteFrame.rect = new Rect(x, y, w, h)` 把同一张大图切成 9 个 SpriteFrame**——这个能力是 Cocos 内建的，不需要 Photoshop 切图。

**3. 编辑器拖拽 = `@property` 绑定**

`@property(SpriteFrame) sourceImage`：这一行声明一个**编辑器可见**的字段。Inspector 上自动出现"Source Image"框，你拖资源进去就完成赋值。

这是 Cocos / Unity 这类引擎跟纯代码框架的核心差异：**美术资源 / 数值参数从代码里抽出来，让策划/美术不用懂代码也能调**。

**4. 中心点对齐 = 一切坐标的起点**

`PuzzleRoot.Position = (0, 0, 0)` + `imageNode.setPosition(0, 0, 0)` → 图在 Canvas 中心。

**Cocos 2D 用的是 "anchor 中心" 坐标系**，节点的 (0, 0) = 节点中心。这跟 HTML/CSS 的左上角坐标系不一样，但跟"游戏开发"惯例一致。后续我们所有"slot 位置" 计算都基于"PuzzleRoot 中心 = (0, 0)" 这个前提。

## 9. 常见坑

- **PNG 透明通道问题**：如果你的图有透明背景，看上去可能"边缘锯齿" 或"半透明灰边"。Inspector 里把 SpriteFrame 资源 → "Pixels Per Unit" 维持默认，"Premultiply Alpha" 不要勾（除非你知道自己在干嘛）。
- **Sprite 不显示 = 节点 active 但没渲染**：常见原因①没 SpriteFrame；②`UITransform` 尺寸是 0；③节点不在 Canvas 下（不在 UI 层级）；④Camera 看不到（偏离镜头）。本节用的是 Canvas 下的节点，前 3 条都不会有问题。
- **图不在中央 = anchor 没设对**：默认 UITransform 的 anchor 是 (0.5, 0.5)，节点位置 = 节点中心。如果你看到图偏左下，可能是 anchor 改成 (0, 0) 了。
- **保存场景忘按 Ctrl+S**：编辑器修改场景树（拖节点、加 Component）后**要主动保存**才会写入 .scene 文件。预览前一定 Ctrl/Cmd + S。
- **TS 编译失败**：Creator 自动编译 TS，但如果 import 路径错了或类型不存在，控制台会红字。最常见的是 `import { Sprite } from 'cc'` 漏了 —— Cocos 的所有 API 都从 `'cc'` 导出。
- **预览端口冲突**：预览失败说"端口被占用" → 重启 Creator 即可。

## 10. 下一节预告

整图能显示了，接下来要把它**切成 9 块小图**，每块独立显示成一个 Sprite。

**01 节**：用 `SpriteFrame.rect` 切出 9 个 SpriteFrame、创建 9 个 Node，每块按 (row, col) 摆位。

**01 节预告的关键技术**：

- **`SpriteFrame` 的浅克隆**：从源 SpriteFrame 派生 9 个新 SpriteFrame，每个 rect 不同。
- **数据结构 `slots: number[]`**：长度 9 的数组，记录"第 j 个槽位放着哪一块"。这是后续所有玩法的"真相源"。
- **拼图块 = Node + Sprite + UITransform**，每块按 row/col 算位置。

完成 01 后你的屏幕上是**九宫格的"完整图（拼对了）"** —— 看起来跟整张图没区别，但每块是独立 Node，能各自挪动。
