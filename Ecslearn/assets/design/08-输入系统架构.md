# 08 - 输入系统架构

## 设计目标

| 约束 | 解法 |
|------|------|
| 改键位不改逻辑 | 原始按键 → 语义动作 两层分离 |
| 上下文切换（菜单/对话/战斗） | 在管线入口拦截，System 无需 if/else |
| 斜向移动速度一致 | ActionMapSystem 归一化 moveDir |
| OOP 主角 + ECS 管线共存 | Proxy Entity 模式：PlayerControl 创建代理实体 |

---

## 整体架构：4 层管线

```
┌──────────────┐    ┌──────────────┐    ┌───────────────────┐    ┌──────────────┐
│ RawInputSys  │ →  │ ActionMapSys │ →  │ PlayerControlSys  │ →  │ MoveSyncSys  │
│  ① 键盘采集   │    │ ② 语义映射    │    │  ③ 动作→速度      │    │ ④ ECS→节点   │
└──────────────┘    └──────────────┘    └───────────────────┘    └──────────────┘
    cc.input             KEYMAP           speed × moveDir        node.setPosition
```

**GameLoop** 每帧按顺序驱动：

```typescript
update(dt: number) {
    const entities = this._world.all();
    this._rawInput.update(entities);       // ①
    this._actionMap.update(entities);       // ②
    this._playerControl.update(entities);   // ③
    this._moveSync.update(entities, dt);    // ④
}
```

## 第①层：RawInputSystem — 键盘 / 鼠标采集

**文件**：`game/system/RawInputSystem.ts`

唯一接触 `cc.input` 的地方。构造函数注册全局监听，每帧把缓冲数据写入
RawInputComp，然后清空帧缓冲。

```
cc.input → held / frameDown / frameUp / frameMouseDown
         ↓
RawInputComp { keys, down, up, mouseDown }
```

**关键设计**：
- `held` 持续状态用 `Map<number, boolean>`，支持按住判定
- `down` / `up` 用 `Set<number>`，仅保留一帧，下帧自动清空
- 鼠标左键单独 `mouseDown` 布尔标记

### RawInputComp

```typescript
export class RawInputComp implements IComponent {
    keys      = new Map<number, boolean>();  // 按住
    down      = new Set<number>();           // 本帧按下
    up        = new Set<number>();           // 本帧抬起
    mouseDown = false;                       // 本帧鼠标左键
}
```

## 第②层：ActionMapSystem — 物理键 → 语义动作

**文件**：`game/system/ActionMapSystem.ts`

核心是一张纯数据映射表：

```typescript
const KEYMAP: ReadonlyMap<number, EAction> = new Map([
    [KeyCode.KEY_W, EAction.MoveUp],
    [KeyCode.KEY_S, EAction.MoveDown],
    [KeyCode.KEY_A, EAction.MoveLeft],
    [KeyCode.KEY_D, EAction.MoveRight],
    [KeyCode.KEY_J, EAction.Attack],
    [KeyCode.SPACE, EAction.Dodge],
]);
```

**改键位** → 只改这张表，全项目零影响。

### 归一化移动方向

```typescript
const dx = (right ? 1 : 0) - (left ? 1 : 0);
const dy = (up    ? 1 : 0) - (down ? 1 : 0);
const len = Math.sqrt(dx * dx + dy * dy) || 1;
act.moveDir = { x: dx / len, y: dy / len };
```

斜向自动变 `0.707`，不会出现 `√2` 速度 bug。

### ActionComp

```typescript
export enum EAction { MoveUp, MoveDown, MoveLeft, MoveRight, Attack, Dodge }

export class ActionComp implements IComponent {
    active      = new Set<EAction>();    // 持续按住
    justPressed = new Set<EAction>();    // 本帧刚触发
    moveDir     = { x: 0, y: 0 };       // 归一化方向
}
```

## 第③层：PlayerControlSystem — 语义动作 → 速度

**文件**：`game/system/PlayerControlSystem.ts`

```typescript
export class PlayerControlSystem implements ISystem {
    update(entities: Entity[]) {
        for (const e of entities) {
            const act = e.getComponent(ActionComp);
            const vel = e.getComponent(VelocityComp);
            if (!act || !vel) continue;

            const speed = 200;
            vel.vx = act.moveDir.x * speed;
            vel.vy = act.moveDir.y * speed;
        }
    }
}
```

`speed` 后续由 `PlayerProperty.moveSpeed` 替代，支持 Buff 加速/减速。

### VelocityComp

```typescript
export class VelocityComp implements IComponent {
    vx = 0;
    vy = 0;
}
```

## 第④层：MoveSyncSystem — ECS 速度 → Cocos 节点

**文件**：`game/system/MoveSyncSystem.ts`

```typescript
export class MoveSyncSystem implements ISystem {
    update(entities: Entity[], dt: number) {
        for (const e of entities) {
            const vel = e.getComponent(VelocityComp);
            const ref = e.getComponent(NodeRefComp);
            if (!vel || !ref) continue;

            const pos = ref.node.position;
            ref.node.setPosition(
                pos.x + vel.vx * dt,
                pos.y + vel.vy * dt,
                pos.z,
            );
        }
    }
}
```

### NodeRefComp

```typescript
export class NodeRefComp implements IComponent {
    constructor(public readonly node: Node) {}
}
```

ECS 与 Cocos 的唯一桥接点。持有 Node 引用，系统不知道 Cocos API 以外的任何东西。

## Proxy Entity 模式：OOP ↔ ECS 桥接

### 问题

Player 是 OOP 对象（Cocos Component），但输入管线是 ECS。怎么办？

### 解法

**PlayerControl** 在 `start()` 中创建一个"代理实体"，挂上四个 Component，
注册到 World：

```typescript
private _initProxy() {
    this._entity = new Entity();
    this._entity.addComponent(new RawInputComp());
    this._entity.addComponent(new ActionComp());
    this._entity.addComponent(new VelocityComp());
    this._entity.addComponent(new NodeRefComp(this.node));
    World.inst.add(this._entity);
}
```

- **写入方**：4 个 ECS System 按管线顺序写入各 Component
- **读取方**：PlayerControl.lateUpdate 从 `VelocityComp` / `ActionComp` 读取结果，驱动 FSM

```
 ECS Pipeline (write)          PlayerControl (read)
┌────────────────────┐        ┌─────────────────────┐
│ RawInputSys        │        │                     │
│   → RawInputComp   │        │ lateUpdate() {      │
│ ActionMapSys       │        │   vel = entity.get  │
│   → ActionComp     │───────→│   act = entity.get  │
│ PlayerControlSys   │        │   fsm.tick(dt)      │
│   → VelocityComp   │        │ }                   │
│ MoveSyncSys        │        │                     │
│   → node.position  │        └─────────────────────┘
└────────────────────┘
```

### onDestroy 清理

```typescript
onDestroy() {
    if (this._entity && World.inst) {
        World.inst.remove(this._entity);
    }
}
```

## World 与 GameLoop

### World — 实体注册表

**文件**：`game/core/World.ts`

```typescript
export class World {
    private static _inst: World;
    static get inst(): World { return this._inst; }
    private _entities: Entity[] = [];

    constructor() { World._inst = this; }
    add(entity: Entity): void { ... }
    remove(entity: Entity): void { ... }
    all(): Entity[] { return this._entities; }
}
```

全局单例，`GameLoop.onLoad` 中 `new World()` 自动设置。

### GameLoop — ECS 驱动器

**文件**：`game/core/GameLoop.ts`

Cocos Component，挂在场景根节点。**必须在 PlayerControl 节点之前加载**
（父节点或靠前的兄弟节点），保证 `onLoad` → `new World()` 先于
`PlayerControl.start` → `World.inst.add()`。

## 扩展指南

### 加新按键

1. `EAction` 枚举加一项
2. `ActionMapSystem.KEYMAP` 加一行映射
3. 在需要的 System 或 PlayerControl 中读取 `act.justPressed.has(EAction.Xxx)`

零修改其他管线。

### 加手柄支持

新建 `GamepadInputSystem`，写入 **同一个** `RawInputComp`，ActionMapSystem 不用改。

### 上下文切换（菜单 / 对话）

在 RawInputSystem 前加一个 `InputContextSystem`：

```typescript
export class InputContextSystem implements ISystem {
    private _blocked = false;
    block()   { this._blocked = true; }
    unblock() { this._blocked = false; }

    update(entities: Entity[]) {
        if (!this._blocked) return;
        for (const e of entities) {
            const raw = e.getComponent(RawInputComp);
            if (raw) {
                raw.keys.clear();
                raw.down.clear();
                raw.up.clear();
                raw.mouseDown = false;
            }
        }
    }
}
```

菜单打开时 `block()`，关闭时 `unblock()`。下游所有 System 自动停止响应，零 if/else。
