# 04 - 敌人死亡与金币系统

> 前置：[Step0-遗漏的系统](./Step0-遗漏的系统.md) §1（`goldDrop`）、§2（死亡事件总线化）、§3（事件命名）

>

## 0. 决策摘要（Linus 拍板）

| # | 决策 | 选择 | 否决项 |
|---|-----|------|-------|
| 1 | 金币物件模型 | **一枚物件 = N 金币**（大头=贵币种）| 每死一只生成 N 个小金币（性能垃圾、视觉噪声）|
| 2 | 拾取方式 | **吸附式**（进入 `PickupRange` 自动飞向玩家）| 碰撞拾取（玩家要绕路捡钱，打断战斗节奏）|
| 3 | 拾取半径配置 | **`PickupRange` 属性 + Modifier 链**（和 Attack/CritRate 同一套）| 硬编码 / 全局常量（每次调值要改代码）|
| 4 | 默认 `PickupRange` | **80px**（幸存者标准手感）| 120px（早期太无脑）、50px（后期堆磁吸才好玩）|
| 5 | 其他掉落 | **只做金币**（经验仍走事件直接加）| 物件化经验宝石（幸存者式，但和我们"清完一波"节奏冲突）|
| 6 | 金币物件层级 | **世界层**（和敌人同级，会被摄像机 clamp）| UI 层（转场会消失、错位）|
| 7 | 物件上限 | **场上 ≤ 200 枚**，超限触发自动合并 | 无上限（后期屏幕糊成金色）|

## 1. 总览

### 1.1 数据流

```text
敌人死亡
  │
  ▼
MinionDeadState.enter
  │ emit(EnemyDeath, { enemyId, xpReward, goldDrop, worldPos })
  │
  ├─► PlayerExperience 监听：grantXp(xpReward)
  ├─► RunSession 监听：totalKills++ / comboKills++ / ...
  └─► GoldSystem 监听：
         │
         │ finalAmount = runGoldModifiers(goldDrop, ctx)
         │                 │
         │                 ├─ ComboKillMod (×1.2 if combo ≥ 3)
         │                 ├─ PreciseKillMod (×2 if timeSinceLastKill ≥ 5)
         │                 └─ GreedUpgradeMod (×(1+greedLv*0.1))
         │
         │ emit(GoldDrop, { worldPos, amount: finalAmount })
         ▼
CoinPool.spawn(worldPos, amount)
  │
  │ 生成 1 枚 Coin 物件，denomination = amount
  │
  ▼
每帧：CoinPickupSystem
  │ for each coin:
  │   if dist(coin, player) ≤ PickupRange → 进入 Attracting
  │   if 碰到玩家 Hitbox → gainGold(coin.amount) + recycle
  │
  ▼
RunSession.gold += amount
EventBus.emit(GoldGained, { final, source, worldPos })
  │
  └─► HUD / 跳字 / 音效 / 商店 监听
```

### 1.2 相关系统清单

| 系统 | 职责 | 文件（建议位置）|
|------|------|----------------|
| `EnemyDeath` 事件 | 唯一死亡广播源 | `MinionDeadState.ts`（改造）|
| `GoldSystem` | 监听死亡、跑 Modifier 链、生成物件 | `game/gold/GoldSystem.ts` |

## 2. 敌人死亡事件（`EnemyDeath`）

### 2.1 事件定义

> 这一节是 Step0 §2 的落地版。Step0 定义了"要广播"，本节定义"广播内容"的最终签名。

```typescript
// assets/script/game/events/GameEvents.ts

export const GameEvt = {
    // ─── 敌人 ──────────
    EnemyDeath:      'enemy:death',
    EnemySpawn:      'enemy:spawn',
    // ─── 金币 ──────────
    GoldDrop:        'gold:drop',       // 金币物件已生成（世界坐标层）
    GoldPickupBegin: 'gold:pickup_begin',  // 开始吸附
    GoldPickupEnd:   'gold:pickup_end',    // 到达玩家，入账
    GoldGained:      'gold:gained',     // RunSession 数值入账后
    GoldSpent:       'gold:spent',
    // ─── 其他见 Step0 §3 ───
} as const;

export interface EnemyDeathEvent {
    enemyId: string;
    xpReward: number;
    goldDrop: number;         // baseAmount，Modifier 链前
    worldPos: Readonly<Vec3>;
    killerId?: string;        // 'player' / 'environment' / 'poison'
    /** 可选元信息，用于 Modifier 判定。杀伤 baseDamage 等不放这里（耦合太深）*/
    meta?: {
        isCrit?: boolean;
        isExecute?: boolean;  // 处刑（低血量斩杀）
    };
}
```

### 2.2 广播点

**唯一广播点** = `MinionDeadState.enter`。任何其他"给奖励"的行为都不允许绕过此事件。

```typescript
// assets/script/game/enemy/minion/states/MinionDeadState.ts

import { EventBus } from '../../../../../baseSystem/util';
import { GameEvt, type EnemyDeathEvent } from '../../../events/GameEvents';

enter(ctx: IMinionCtx): void {
    // ... 原有 dissolve/anim 代码不变

    if (!ctx.xpGranted) {
        ctx.xpGranted = true;
        const payload: EnemyDeathEvent = {
            enemyId: ctx.cfg.id,
            xpReward: ctx.cfg.xpReward,
            goldDrop: ctx.cfg.goldDrop,
            worldPos: ctx.node.worldPosition,
        };
        EventBus.emit(GameEvt.EnemyDeath, payload);
    }
}
```

**注意**：原先的 `PlayerControl.instance?.grantXp(...)` 这一行**删除**。经验改由 `PlayerExperience` 监听器调用：

```typescript
// assets/script/game/player/PlayerExperience.ts（新建 or 现有）
EventBus.on(GameEvt.EnemyDeath, (e: EnemyDeathEvent) => {
    PlayerControl.instance?.grantXp(e.xpReward);
});
```

### 2.3 Linus 评价

🟢 **整个重构的根基**。死亡从"硬调一个方法"变"广播一个事实"。新加任何死亡后的反应（击杀音效、成就、额外掉落）都不再动这个文件。

## 3. `GoldSystem` - 金币系统总控

### 3.1 职责

1. 注册 `EnemyDeath` 监听器
2. 跑 Modifier 链计算最终金额
3. 决定本次掉落**几枚物件**、每枚**面值多少**
4. 调用 `CoinPool.spawn` 实例化到世界坐标
5. 维护 Modifier 注册表（升级/Buff 可插拔）

### 3.2 接口

```typescript
// assets/script/game/gold/GoldSystem.ts

export enum GoldSource {
    Kill     = 'kill',       // 击杀敌人
    Pickup   = 'pickup',     // 场上捡的（已有物件）
    Chest    = 'chest',      // 宝箱（后续）
    Quest    = 'quest',      // 任务（后续）
    Cheat    = 'cheat',      // 调试
}

export interface GoldGainContext {
    source: GoldSource;
    baseAmount: number;
    enemyId?: string;
    killerId?: string;
    worldPos?: Readonly<Vec3>;
    meta?: EnemyDeathEvent['meta'];
}

export class GoldSystem {
    private static _inst: GoldSystem;
    static get inst(): GoldSystem { return this._inst ??= new GoldSystem(); }

    private _mods: GoldModifier[] = [];

    /** 由 RunOrchestrator.start 调用 */
    init(): void {
        EventBus.on(GameEvt.EnemyDeath, this._onEnemyDeath);
        // 内置 Modifier
        this.addModifier(new ComboKillModifier());
        this.addModifier(new PreciseKillModifier());
        // 升级系统后续通过 addModifier 动态注入（GreedUpgradeModifier 等）
    }

    destroy(): void {
        EventBus.off(GameEvt.EnemyDeath, this._onEnemyDeath);
        this._mods.length = 0;
    }

    addModifier(mod: GoldModifier): void {
        this._mods.push(mod);
        this._mods.sort((a, b) => a.priority - b.priority);
    }

    removeModifier(id: string): void {
        this._mods = this._mods.filter(m => m.id !== id);
    }

    /** 外部直接加金（测试/任务/商店退款等）*/
    gainGold(ctx: GoldGainContext): number {
        const final = this._runChain(ctx);
        if (final <= 0) return 0;

        if (ctx.source === GoldSource.Kill && ctx.worldPos) {
            // 击杀 → 生成物件，走吸附拾取
            CoinPool.spawn(ctx.worldPos, final);
            EventBus.emit(GameEvt.GoldDrop, { worldPos: ctx.worldPos, amount: final });
        } else {
            // 其他来源 → 直接入账
            this._commit(final, ctx);
        }
        return final;
    }

    /** CoinPickupSystem 拾取完成时调用，已经是"最终到手金额"*/
    commitPickup(amount: number, worldPos: Vec3): void {
        this._commit(amount, { source: GoldSource.Pickup, baseAmount: amount, worldPos });
    }

    private _onEnemyDeath = (e: EnemyDeathEvent) => {
        if (e.goldDrop <= 0) return;  // 不掉钱的敌人直接跳过
        this.gainGold({
            source: GoldSource.Kill,
            baseAmount: e.goldDrop,
            enemyId: e.enemyId,
            killerId: e.killerId,
            worldPos: e.worldPos,
            meta: e.meta,
        });
    };

    private _runChain(ctx: GoldGainContext): number {
        let amount = ctx.baseAmount;
        for (const m of this._mods) amount = m.apply(amount, ctx);
        return Math.max(0, Math.floor(amount));
    }

    private _commit(amount: number, ctx: GoldGainContext): void {
        RunSessionState.current.gold += amount;
        EventBus.emit(GameEvt.GoldGained, { final: amount, source: ctx.source, worldPos: ctx.worldPos });
    }
}
```

### 3.3 为什么拆 `gainGold` 和 `commitPickup`

- `gainGold` = "这笔钱**应该**变成什么"（过 Modifier 链、决定生成物件还是直接入账）
- `commitPickup` = "物件已到手，**直接**入账"（不再跑 Modifier，避免吸附过程重复加成）

**好品味**：同一个"加金"入口，两种语义清晰分离，**零 if 分支**。

## 4. `GoldModifier` - 加成链

### 4.1 接口

```typescript
// assets/script/game/gold/modifiers/GoldModifier.ts

export interface GoldModifier {
    readonly id: string;
    /** 优先级升序执行。加法类放前、乘法类放后 */
    readonly priority: number;
    apply(amount: number, ctx: GoldGainContext): number;
}
```

### 4.2 首版内置 Modifier

#### 4.2.1 `ComboKillModifier` - 连杀加成

```typescript
export class ComboKillModifier implements GoldModifier {
    id = 'combo_kill';
    priority = 200;  // 乘法档

    apply(amount: number, ctx: GoldGainContext): number {
        if (ctx.source !== GoldSource.Kill) return amount;
        const combo = RunSessionState.current.comboKills;
        if (combo >= 10) return amount * 1.5;
        if (combo >= 5)  return amount * 1.3;
        if (combo >= 3)  return amount * 1.2;
        return amount;
    }
}
```

#### 4.2.2 `PreciseKillModifier` - 精巧杀戮

```typescript
export class PreciseKillModifier implements GoldModifier {
    id = 'precise_kill';
    priority = 210;

    apply(amount: number, ctx: GoldGainContext): number {
        if (ctx.source !== GoldSource.Kill) return amount;
        const dt = RunSessionState.current.timeSinceLastKill;
        // 注意：事件到达时 comboKills 已经++，但 timeSinceLastKill 此时 = 0（也是此事件触发的清零）
        // 所以我们要在 RunSession 监听器**之前**消费这个值。
        // 规避：Modifier 不读 session.timeSinceLastKill，改读 ctx.meta.isExecute 或其他独立指标。
        return amount;
    }
}
```

> ⚠️ **设计警告**：`timeSinceLastKill` 会被"本次击杀"清零，读取顺序是坑。**首版暂不启用 PreciseKill**，等升级系统设计时再补（用"独立冷却"实现，而不是读 `timeSinceLastKill`）。

#### 4.2.3 `GreedUpgradeModifier` - 升级: 贪婪（动态注入）

```typescript
export class GreedUpgradeModifier implements GoldModifier {
    id = 'greed_upgrade';
    priority = 220;

    constructor(private level: number) {}

    setLevel(level: number) { this.level = level; }

    apply(amount: number, ctx: GoldGainContext): number {
        return amount * (1 + this.level * 0.1);  // 每级 +10%
    }
}

// 升级选中时：
upgradeSystem.on('upgrade:chosen', (u) => {
    if (u.id === 'greed') {
        const existing = GoldSystem.inst.findModifier<GreedUpgradeModifier>('greed_upgrade');
        if (existing) existing.setLevel(u.newLevel);
        else GoldSystem.inst.addModifier(new GreedUpgradeModifier(u.newLevel));
    }
});
```

### 4.3 优先级约定

| 区间 | 用途 | 例子 |
|------|------|------|
| 0-99 | 加法类（baseAmount 之上直接加常数）| `+3 per kill` |
| 100-199 | 独立乘法（和其他乘法独立相乘）| 首杀 ×2 |
| 200-299 | 叠加乘法（多个效果相加后相乘）| 连杀、贪婪 |
| 300+ | 最终封顶/封底 | Max 999999 |

### 4.4 Linus 评价

🟢 **链式 Modifier 是数据驱动的正确姿势**。新加一种"场外加金来源"就是 `new XxxModifier`，不改 `GoldSystem` 一行。

🔴 **反面例子**（不要这么写）：

```typescript
// 糟糕的写法：
function gainGold(amount, ctx) {
    if (ctx.source === 'kill' && combo >= 3) amount *= 1.2;
    if (ctx.source === 'kill' && combo >= 5) amount *= 1.3;  // 和上一行重叠，还是上一行覆盖？
    if (hasGreedUpgrade) amount *= (1 + greedLevel * 0.1);
    if (isFirstKillOfWave) amount += 10;
    // ... 半年后这里 50 行
}
```

每加一种加成改这个函数 = 🔴 垃圾。**Modifier 链是把 if 地狱翻译成数据表**。

## 5. `CoinEntity` - 金币物件

### 5.1 数据结构

```typescript
// assets/script/game/gold/CoinEntity.ts

export enum CoinTier {
    Bronze = 'bronze',   // 1-9      金色小币
    Silver = 'silver',   // 10-49    银色中币
    Gold   = 'gold',     // 50-249   金色大币
    Ruby   = 'ruby',     // 250+     红宝石（视觉爆炸）
}

@ccclass('CoinEntity')
export class CoinEntity extends Component {
    /** 面值：一枚币值多少金 */
    denomination: number = 1;

    /** 物件状态 */
    state: 'idle' | 'attracting' | 'pickedUp' = 'idle';

    /** 吸附目标（通常是玩家）*/
    private _target: Node | null = null;

    /** 吸附阶段的速度（渐加速）*/
    private _velocity: Vec2 = new Vec2();

    /** 给外部 system 直接读取 */
    get worldPos(): Vec3 { return this.node.worldPosition; }

    /** 根据 denomination 选 tier 贴图 */
    applyTierVisual(): void {
        const tier = CoinEntity.pickTier(this.denomination);
        // 切换 sprite / 颜色 / 缩放
    }

    static pickTier(amount: number): CoinTier {
        if (amount >= 250) return CoinTier.Ruby;
        if (amount >= 50)  return CoinTier.Gold;
        if (amount >= 10)  return CoinTier.Silver;
        return CoinTier.Bronze;
    }

    /** 由 Pool 创建后调用 */
    reset(worldPos: Vec3, amount: number): void {
        this.denomination = amount;
        this.state = 'idle';
        this._target = null;
        this._velocity.set(0, 0);
        this.node.setWorldPosition(worldPos);
        this.applyTierVisual();
        // 入场小跳：随机方向弹射 80-120px
        this._playSpawnTween();
    }

    startAttracting(target: Node): void {
        if (this.state !== 'idle') return;
        this.state = 'attracting';
        this._target = target;
        EventBus.emit(GameEvt.GoldPickupBegin, { amount: this.denomination });
    }

    tickAttracting(dt: number, accel: number, maxSpeed: number): boolean {
        if (!this._target || !this._target.isValid) return false;
        const from = this.node.worldPosition;
        const to = this._target.worldPosition;
        const dx = to.x - from.x, dy = to.y - from.y;
        const dist = Math.hypot(dx, dy);

        if (dist < 8) return true;  // 已到手

        const invD = 1 / Math.max(dist, 0.0001);
        const ax = dx * invD * accel, ay = dy * invD * accel;
        this._velocity.x += ax * dt;
        this._velocity.y += ay * dt;

        const spd = this._velocity.length();
        if (spd > maxSpeed) {
            this._velocity.multiplyScalar(maxSpeed / spd);
        }

        this.node.setWorldPosition(
            from.x + this._velocity.x * dt,
            from.y + this._velocity.y * dt,
            from.z
        );
        return false;
    }

    markPickedUp(): void { this.state = 'pickedUp'; }
}
```

### 5.2 出生跳跃（Spawn Tween）

```typescript
private _playSpawnTween(): void {
    // 随机方向弹射：模拟"从敌人身上掉出来"
    const angle = Math.random() * Math.PI * 2;
    const dist = 60 + Math.random() * 40;  // 60-100px
    const from = this.node.worldPosition.clone();
    const to = new Vec3(from.x + Math.cos(angle) * dist, from.y + Math.sin(angle) * dist, from.z);

    tween(this.node)
        .to(0.25, { worldPosition: to }, { easing: 'quadOut' })
        .call(() => { this.state = 'idle'; })
        .start();

    this.state = 'idle';  // 允许被吸附：就算 tween 没跑完，吸附也能中途接管（直接覆盖位置）
}
```

**好品味**：吸附不等 tween 完成，**数据优先**。玩家靠近时立即开吸，视觉比"等跳完才飞"更爽。

### 5.3 层级与父节点

- 父节点：`MapManager.inst.coinRoot`（地图下的世界层）
- 与敌人 `enemyRoot` 同级，渲染在敌人**之上**、弹道**之下**
- 地图切换时 `coinRoot.removeAllChildren()` 统一清理（未拾取金币**强制入账**，见 §10.2）

## 6. `CoinPool` - 对象池

### 6.1 实现（仿 `ProjectilePool`）

```typescript
// assets/script/game/gold/CoinPool.ts

export class CoinPool {
    private static _pool: Node[] = [];
    private static _parent: Node | null = null;
    private static _prefab: Prefab | null = null;
    private static _active: CoinEntity[] = [];  // 场上活跃金币

    static init(parent: Node, prefab: Prefab): void {
        const layer = new Node('CoinLayer');
        parent.addChild(layer);
        this._parent = layer;
        this._prefab = prefab;
    }

    /** 生成 1 枚物件，面值 = amount */
    static spawn(worldPos: Readonly<Vec3>, amount: number): CoinEntity | null {
        if (this._active.length >= MAX_ACTIVE_COINS) {
            this._autoCompact();
        }

        let node = this._pool.pop();
        if (!node) {
            if (!this._prefab) return null;
            node = instantiate(this._prefab);
        }
        node.active = true;
        this._parent!.addChild(node);

        const coin = node.getComponent(CoinEntity)!;
        coin.reset(worldPos.clone ? worldPos.clone() : new Vec3(worldPos.x, worldPos.y, worldPos.z), amount);
        this._active.push(coin);
        return coin;
    }

    static release(coin: CoinEntity): void {
        const i = this._active.indexOf(coin);
        if (i >= 0) this._active.splice(i, 1);
        const node = coin.node;
        if (!node.isValid) return;
        node.active = false;
        node.removeFromParent();
        this._pool.push(node);
    }

    static get active(): ReadonlyArray<CoinEntity> { return this._active; }

    static clearAll(): void {
        for (const c of this._active) {
            const n = c.node;
            if (n.isValid) n.destroy();
        }
        this._active.length = 0;
        for (const n of this._pool) { if (n.isValid) n.destroy(); }
        this._pool.length = 0;
    }

    /** 场上物件过多时，把最老的 20 枚合并 */
    private static _autoCompact(): void {
        const VICTIMS = 20;
        const merged = this._active.slice(0, VICTIMS);
        let total = 0;
        let lastPos = new Vec3();
        for (const c of merged) {
            total += c.denomination;
            lastPos.set(c.node.worldPosition.x, c.node.worldPosition.y, c.node.worldPosition.z);
            this.release(c);
        }
        // 合并后生成一枚超大面值物件
        this.spawn(lastPos, total);
    }
}

const MAX_ACTIVE_COINS = 200;
```

### 6.2 性能预算

- 一波最高 200 个敌人同时死 → 200 枚物件
- `CoinPickupSystem.update` 每帧 200 次距离判定 = **零压力**（桌面甚至 1000 也无感）
- 超 200 触发合并，把老物件合成 1 枚 → 视觉上玩家感觉"钱越攒越多"，性能可控

## 7. `CoinPickupSystem` - 吸附与拾取

### 7.1 核心循环

```typescript
// assets/script/game/gold/CoinPickupSystem.ts

const ATTRACT_ACCEL = 2400;     // px/s^2，吸附加速度
const ATTRACT_MAX_SPEED = 1200; // px/s，吸附最大速度
const PICKUP_HIT_RADIUS = 20;   // 玩家拾取判定半径

export class CoinPickupSystem {
    update(entities: ReadonlyArray<Entity>, dt: number): void {
        const player = PlayerControl.instance;
        if (!player) return;

        const pickupRange = player.properties.getValue(EPropertyId.PickupRange);
        const pickupRangeSq = pickupRange * pickupRange;
        const playerPos = player.node.worldPosition;

        const coins = CoinPool.active;
        for (let i = coins.length - 1; i >= 0; i--) {
            const coin = coins[i];
            if (!coin.node.isValid) { CoinPool.release(coin); continue; }

            if (coin.state === 'idle') {
                const dx = coin.worldPos.x - playerPos.x;
                const dy = coin.worldPos.y - playerPos.y;
                if (dx * dx + dy * dy <= pickupRangeSq) {
                    coin.startAttracting(player.node);
                }
                continue;
            }

            if (coin.state === 'attracting') {
                const arrived = coin.tickAttracting(dt, ATTRACT_ACCEL, ATTRACT_MAX_SPEED);
                if (arrived) {
                    coin.markPickedUp();
                    GoldSystem.inst.commitPickup(coin.denomination, coin.worldPos);
                    EventBus.emit(GameEvt.GoldPickupEnd, { amount: coin.denomination, worldPos: coin.worldPos });
                    CoinPool.release(coin);
                }
                continue;
            }
        }
    }
}
```

### 7.2 为什么用"平方距离"不用 `hypot`

```typescript
if (dx * dx + dy * dy <= pickupRangeSq) { ... }
```

每帧 200 次 `hypot` = 200 次 `sqrt`。用平方比较**消灭 sqrt**，这是非常经典的好品味优化。**真实地，可以预先算 `pickupRangeSq`，`pickupRange` 变了再重算（通过 EventBus 的 `PlayerAttrChanged` 事件）**。首版直接每帧算一次 `* 自身` 也 OK，代价可忽略。

### 7.3 吸附曲线调参指引

| 参数 | 值 | 效果 |
|------|-----|------|
| `ATTRACT_ACCEL` | 2400 | 加速度越大，吸附"越暴力" |
| `ATTRACT_MAX_SPEED` | 1200 | 封顶速度，避免远处金币飞太快穿模 |
| `PICKUP_HIT_RADIUS` | 20 | 到玩家 20px 内判定入账 |

**手感测试建议**：拾取范围 = 80，金币距离玩家 80px 时，吸附耗时应在 **0.3-0.5s** 之间（够爽、不黏人）。

### 7.4 Linus 评价

🟢 **一个循环，三个 state，没有 if 嵌套**。`idle / attracting / pickedUp` 是数据状态，状态机在**数据里**而不是在**代码里**。

## 10. 特殊场景处理

### 10.1 波次切换（升级/商店）

波次结束（`RunPhase.Upgrade` / `Shop`）时：
- 场上**已经生成**的金币物件 **保留**（玩家看得到）
- 但 `CoinPickupSystem` 不 tick（GameLoop 按 phase 跳过）
- 吸附暂停，玩家可以随便走但不会捡到钱（因为 phase != Spawning/Clearing）
- **为什么这样**：升级窗口期玩家注意力在 UI，不应被地上跳字干扰

**例外决策**：升级面板弹出**之前**，留 **0.8s 宽限期** 让当前正在吸附的金币飞完（`RunPhase.Clearing → Upgrade` 之间加个 `Settle` 过渡态，见 03 文档 §3 待补）。

### 10.2 地图切换

Warp Gate 触发 `MapLoad` 事件前：

```typescript
// MapManager.beforeUnload
const unpicked = CoinPool.active.slice();
for (const c of unpicked) {
    GoldSystem.inst.commitPickup(c.denomination, c.worldPos);  // 自动入账
    CoinPool.release(c);
}
```

**设计理由**：不能让玩家因为"跑得快"或"走位失误"导致金币被销毁浪费。**自动送达**是唯一人道的处理。

### 10.3 玩家死亡

玩家死亡（`PlayerDeath`）时：
- 场上金币**全部销毁**（不入账）
- 这是失败的代价的一部分
- `CoinPool.clearAll()`

### 10.4 商店购买用金币

```typescript
// Shop UI 点击购买
if (RunSessionState.current.gold >= item.price) {
    RunSessionState.current.gold -= item.price;
    EventBus.emit(GameEvt.GoldSpent, { amount: item.price, reason: 'shop' });
    applyItem(item);
}
```

**Linus 原则**：花钱也走事件，HUD / 音效 / 成就监听 `GoldSpent` 就好。

## 11. 数值曲线（策划区）

### 11.1 `goldDrop` 随波次缩放

策划表 `WaveScaling` 里的 `goldMul`（03 文档 §4）控制：

| 波次 | goldMul | 说明 |
|------|---------|------|
| 1-3 | 1.0 | 基础 |
| 4-6 | 1.1 | 轻微成长 |
| 7-9 | 1.2 | 商店档 |
| 10-12 | 1.35 | 后期加速 |
| 13-15 | 1.5 | 终局爽感 |

**实现**：Modifier 链里最后加一个 `WaveScalingModifier`，`priority = 299`：

```typescript
export class WaveScalingModifier implements GoldModifier {
    id = 'wave_scaling';
    priority = 299;
    apply(amount: number): number {
        return amount * getWaveScaling(RunSessionState.current.waveIndex).goldMul;
    }
}
```

### 11.2 每局总金币目标

| 阶段 | 目标金币 | 用途 |
|------|---------|------|
| 第 1 张地图末（Wave 5）| ~200 | 商店 1 次小消费 |
| 第 2 张地图末（Wave 10）| ~700 | 商店 1 次大消费 或 2 次小 |
| 第 3 张地图末（Wave 15）| ~2000 | 通关奖励 + 元进度 |

**校验方法**：每 5 波结束 playtest，记录实际金币数，和目标比对。偏差 > 30% 调 `goldMul`。

### 11.3 物件面值分档

面值算法 = 一次击杀的最终金额直接作为单枚面值。

- 小兵 2 金 → 1 枚 Bronze
- 精英 15 金 → 1 枚 Silver
- Boss 200 金 + 连杀 ×1.5 = 300 金 → 1 枚 Ruby（视觉爆炸）

**不拆分**。"一枚 = N 金"的核心价值就是**简单**。

## 13. 关键设计决策表

| 决策 | 选择 | 否决 | 理由 |
|------|------|------|------|
| 金币表达 | 一枚 = N 金 | 一死一堆小币 | 性能/视觉/代码都更简单 |
| 拾取方式 | 吸附式 | 碰撞式 | 不打断战斗节奏 |
| 拾取半径 | `PickupRange` 属性 | 硬编码常量 | 升级/Buff 零侵入接入 |
| 默认半径 | 80px | 50 / 120 | 幸存者标准 |
| 加成方式 | Modifier 链 | 写死 if-else | 新加一种不改核心代码 |
| 物件层级 | 世界层 | UI 层 | 和地图/摄像机一致 |
| 场上上限 | 200 + 自动合并 | 无上限 / 硬删除 | 性能保底 + 不丢钱 |
| 未拾取金币 | 地图切换自动入账 | 销毁 / 保留 | 不让玩家因走位失误亏钱 |
| 玩家死亡金币 | 销毁 | 保留 | 失败代价的一部分 |
| 经验宝石 | 不做（xp 走事件）| 做（仿幸存者）| 和"清完波"节奏冲突 |
