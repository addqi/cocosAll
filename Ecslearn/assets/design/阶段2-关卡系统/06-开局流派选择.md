# 06 - 开局流派选择（速射流 / 蓄力流）

> 目标：进游戏先弹"二选一流派"面板，玩家选完才开始 Wave 1。每个流派提供**独特的射击模型** + **独特的起手技能 / 被动** + **独享一部分升级池**，让同一套升级系统玩出差异化。
>
> 前置：
> - [05-关卡MVP-波次与升级](./05-关卡MVP-波次与升级.md) 已完成（LevelManager / UpgradeOfferSystem / UI 事件体系）
> - 玩家已有 `HoldToShoot` / `ClickToShoot` / `AutoShoot` 三个现成 ShootPolicy
> - 技能系统已有 `ApplySelfBuffSkill` 原语
>
> 风格参考：[重构步骤-1-职业与基础架构.md](../阶段1-角色战斗/历史文档/重构步骤-1-职业与基础架构.md)

## 一、这一步到底在解决什么问题

当前 MVP 下，玩家永远是"长按射击 + 硬编码三技能（berserk/fireball/ice-ring）"。每局开始完全一样，**没有策略选择感**。

幸存者 / Roguelite 的核心乐趣之一就是 **"开局锁定一条流派路线 → 升级系统围绕这条路线构筑"**。我们现有的 55 条升级 + 7 条进化链已经能支持多流派，**缺的是一个开局岔路口**以及**"升级池按流派过滤"**的基础设施。

这一步做三件事：

1. **把"开局第一次选择"建出来** —— 两条流派、不同射击方式、不同起手技能
2. **给蓄力流真·蓄力手感** —— 按下蓄力、抬起发射、伤害按蓄力时长打曲线
3. **升级池支持流派过滤** —— 配一个 `classIds` 字段，通用升级 + 独有升级并存

## 二、阶段目标

完成本阶段后应达到：

1. 进游戏先弹 **ClassSelectPanel**，屏幕两张卡「速射流 / 蓄力流」
2. 选完前**游戏不开始**：LevelManager 不起 Wave 1
3. **速射流**：射击 = 长按持续发射；装备 1 个起手主动技能 `rapid-burst`（按 1 键，5 秒内射速翻倍）
4. **蓄力流**：射击 = **按下蓄力，抬起发射**；蓄力 0~1 秒线性映射到 1.0x ~ 2.0x 伤害；蓄力中移速 × 0.5
5. 原先硬编码的 berserk/fireball/ice-ring 默认技能**由流派决定的技能组替代**
6. 流派选择走标准事件总线：`ClassChosen` 事件 → PlayerControl 应用
7. **升级池按 `classIds` 过滤**：速射 / 蓄力 各示范 4 条独有升级，其余通用
8. **零 @property 拖拽**，UI 代码生成

export interface ShootModeHold   { readonly type: 'hold'; }
export interface ShootModeClick  { readonly type: 'click'; }
export interface ShootModeCharge {
    readonly type: 'charge';
    /** 达到最大伤害所需蓄力秒数；超过此时长不再增益 */

    readonly slot?: number;

export interface PlayerClassPassiveBuff {
    readonly buffKey: string;
}

export interface PlayerClassDef {
    readonly id: string;
    readonly name: string;
    readonly desc: string;
    readonly shoot: ShootModeDef;
    readonly startSkills: readonly PlayerClassSkillSlot[];
    readonly passiveBuffs: readonly PlayerClassPassiveBuff[];
}

## 七、Step 2.10.4 - ChargeShoot Policy + 蓄力伤害

### 目标

蓄力流的核心手感：**按下开始蓄力 → 抬起发射 → 伤害按蓄力时长打曲线**。

### 重点涉及文件

- `assets/script/game/shoot/ShootPolicies.ts`（新增 ChargeShoot）
- `assets/script/game/player/states/PlayerContext.ts`（新增 `chargeSec` 状态）
- `assets/script/game/player/archer/ArcherAttackState.ts`（读取 chargeSec 写 damageRatio）
- `assets/script/game/player/archer/ArcherBehavior.ts`（累加 chargeSec）

### 设计

**核心决策：`chargeSec` 累加职责归属 PlayerBehavior（帧级循环），不归 Policy（无状态查询）。**

原因：Policy 接口语义是"这一帧要不要射" → 返 `boolean`。要在 Policy 里累加状态就污染了接口。蓄力时长是"玩家状态"，让 PlayerBehavior（每帧被调 tick）累加最干净。

### 改动内容

#### 7.1 新增 `ChargeShoot` Policy

```typescript
export class ChargeShoot implements IShootPolicy {
    readonly priority = 0;
    wantShoot(input: ActionComp): boolean {
        return input.justReleased.has(EAction.Attack);
    }
}
```

**就这一条**：抬起即射。累加在别处。

#### 7.2 `PlayerCtx` 加 `chargeSec`

```typescript
export class PlayerCtx {
    // ... 既有字段
    chargeSec = 0;
    /** 本次 shoot 消耗的蓄力时长（射击瞬间由 behavior 写入，AttackState 读）*/
    pendingChargeSec = 0;
}
```

#### 7.3 `ArcherBehavior.tick` 累加 / 复位

```typescript
// 仅当流派 shoot.type === 'charge' 时累加
tick(dt: number, ctx: PlayerCtx): void {
    if (this._shootMode?.type !== 'charge') return;

    const pressing = ctx.input.active.has(EAction.Attack);
    const released = ctx.input.justReleased.has(EAction.Attack);

    if (released) {
        ctx.pendingChargeSec = ctx.chargeSec;  // 交接给 AttackState
        ctx.chargeSec = 0;
    } else if (pressing) {
        ctx.chargeSec += dt;
    } else {
        ctx.chargeSec = 0;
    }
}
```

#### 7.4 `ArcherAttackState._fire` 应用伤害倍率

```typescript
const projConfig = ShootResolver.snapshotProjectileConfig(ctx.prop);

// 【新】蓄力流：按 pendingChargeSec 计算伤害倍率
const shootMode = ctx.behavior.shootMode;
if (shootMode?.type === 'charge') {
    const t = Math.min(ctx.pendingChargeSec / shootMode.maxChargeSec, 1);
    projConfig.damageRatio = 1 + t * (shootMode.maxDamageRatio - 1);
    ctx.pendingChargeSec = 0;
}
```

#### 7.5 移速衰减

```typescript
// PlayerControlSystem 或移动相关代码里
if (chargeActive) velocity *= shoot.moveSpeedRatio;
```

具体位置取决于现有 PlayerControlSystem 代码形态，实施时就近处理。

### 设计要求

- **Policy 纯函数**：`wantShoot` 不维护任何状态
- **`chargeSec` 归玩家行为层**：只在 `shoot.type === 'charge'` 时累加，省 if
- **抬起即射**：松开瞬间立刻发射，不等下一帧
- **伤害倍率线性**：`1 + t × (maxRatio - 1)`，其中 `t = min(chargeSec / maxChargeSec, 1)`
- **蓄力中移速**：由 `moveSpeedRatio` 控制；0.5 = 半速

### 完成定义

- 选蓄力流 → 按住鼠标 1 秒 → 抬起 → 一发箭，伤害 2x
- 按住 0.25 秒抬起 → 伤害 1.25x
- 秒抬起（< 1 帧）→ 伤害 1.0x
- 蓄力中 WASD 移动速度降低到原来 50%

#### 用例 1：ChargeShoot wantShoot 仅在 justReleased 时返 true
- 操作：构造 ActionComp，justReleased 加 Attack，active 加 Attack
- 预期：`wantShoot` 返 true

#### 用例 2：仅 active 含 Attack（未抬起）时 wantShoot 返 false

#### 8.3 `UpgradeOfferSystem` 构造吃 classId

```typescript
export class UpgradeOfferSystem {
    constructor(
        private readonly _manager: UpgradeManager,
        private readonly _pool: readonly UpgradeConfig[] = ALL_UPGRADES,
        private readonly _currentClassId: string | null = null,
    ) {
        if (_currentClassId === null) {
            console.warn('[UpgradeOfferSystem] currentClassId=null，流派独有升级将被忽略；' +
                '仅通用升级参与抽卡。LevelManager 应在流派选择后再构造本系统。');
        }
    }

    isEligible(cfg: UpgradeConfig): boolean {
        if (this._manager.has(cfg.id)) return false;
        if (cfg.evolvesFrom?.length) {
            if (!cfg.evolvesFrom.every(id => this._manager.has(id))) return false;
        }
        // 【新】流派过滤
        if (cfg.classIds && cfg.classIds.length > 0) {
            if (!this._currentClassId) return false;
            if (!cfg.classIds.includes(this._currentClassId)) return false;
        }
        return true;
    }
}
```

#### 10.2 新增公共方法

```typescript
/**
 * 应用玩家流派：切换射击策略 + 装备起手技能 + 施加被动 Buff
 * 由 GameManager 在 ClassChosen 事件里调一次；LevelManager.start 要等这个调用完成才启动
 */
setPlayerClass(classId: string): boolean {
    const def = getPlayerClassDef(classId);
    if (!def) {
        console.error(`[PlayerControl] 未知流派 id: ${classId}`);
        return false;
    }

    this._currentClassId = classId;

    // 1. 切射击模式（Policy + ShootMode 两块信息都告诉 behavior）
    this.sendBehaviorCommand('set_shoot_mode', def.shoot);

    // 2. 装备起手技能
    for (const s of def.startSkills) {
        const skillDef = getSkillDef(s.skillId);
        if (!skillDef) continue;
        const skill = SkillFactory.create(skillDef) as IActiveSkill;
        this._runtime.skillSystem.equip(skill, s.slot);
    }

    // 3. 施加被动 Buff
    for (const p of def.passiveBuffs) {
        const buffDef = getBuffDef(p.buffKey);
        if (!buffDef) continue;
        this._runtime.buffMgr.addBuff(buffDef, this._runtime.buffOwner);
    }

    console.log(`[PlayerControl] 流派 "${def.name}" 就绪`);
    return true;
}

get currentClassId(): string | null { return this._currentClassId; }
```

## 十八、关键设计决策表

| 决策 | 选择 | 理由 |
|------|------|------|
| 蓄力流的蓄力形态 | **真蓄力曲线 + 被动 +30%** | 被动塑造流派气场，蓄力提供操作深度；双乘叠加感明显 |

| chargeSec 状态归属 | **PlayerBehavior.tick 累加**，不归 Policy | 保持 Policy 无状态，接口纯净 |
| 升级池分离方式 | **同一 JSON + classIds 字段** | 不新增文件、不新增加载器；加字段即扩展 |
