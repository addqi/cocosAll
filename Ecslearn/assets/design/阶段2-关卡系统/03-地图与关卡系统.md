## 本章目标

实现**分波次战斗 + 固定地图 + 商店传送门**的关卡骨架：

- 一局 15 波，每 5 波一张地图，共 3 张地图
- 每波打完弹升级三选一
- 地图末波打完进商店，购物完出现传送门切换到下一张地图
- 第 15 波是最终 Boss，Boss 死即胜利
- **本文档不负责升级效果、敌人数值、具体关卡配置**，只负责**结构与骨架**

## 0. 设计来源与决策记录

### 参考游戏的选择与取舍

| 维度 | 参考来源 | 我们的做法 |
|------|---------|-----------|
| **地图形态** | 吸血鬼幸存者 | 固定地图 + 摄像机跟随 + 边界 |
| **怪物密度** | 吸血鬼幸存者 | 屏幕边缘持续刷、数量递增 |
| **战斗节奏** | 土豆兄弟 | 分波次，波间清场 + 升级打断 |
| **操作手感** | 暖雪 | 手动点击射击（非自动连射）|
| **进度感** | 土豆兄弟 | 每波必升级，有清晰成长台阶 |
| **地图切换** | Hades | 物件传送门，玩家走过去触碰 |

### 已拒绝的设计（明确删除）

| 拒绝项 | 理由 |
|--------|------|
| 房间模板 `RoomTemplate` | 固定地图用不上 |
| `FloorType` 枚举 | 用数据（Wave 的字段）表达，无枚举爆炸 |
| `FloorSequenceGenerator` | 波次表手工配置，不生成 |
| 每层独立的敌人解锁列表 | 改为波次表内直接写 `enemyId` |
| 自动连续射击 | 玩家"点一下 = 一次攻击"，不挂机 |
| 同一升级无限叠加 | 只走 `evolvesFrom` 显式分级，防平衡 bug |
| 经验驱动的升级触发 | 改为"每波必升"，节奏可预测 |
| "清完一层 → 传送门 → 下一层"线性推进 | 改为"波次 + 地图"两层结构 |

```
 │   ┌──────────── Map 1 ─────────────┐                        │

 │   └────────────────────────────────┘  │                     │
 │                                       ▼                     │
 │   ┌──────────── Map 2 ─────────────┐                        │
 │   │  Wave 6  → Upgrade             │                        │
 │   │  ...                           │                        │
 │   │  Wave 10 → Upgrade → Shop → 🚪 │ ─┐                     │
```

### 一波的生命周期

```
 ┌─ Spawning ────────┐   刷怪窗口开启，spawner 按 interval tick
 │                   │
 │  spawners 持续刷  │   玩家战斗
 │                   │
 └────────┬──────────┘
          ▼  (刷怪窗口结束: duration 到)
 ┌─ Clearing ────────┐   停止刷怪，等场上清零
 │                   │
 │  超时强清场保险    │   玩家扫尾
 │                   │
 └────────┬──────────┘
          ▼  (enemiesAlive === 0 || clearingTimeout)
 ┌─ WaveEnd ─────────┐   发波次奖励金币 + 事件广播
 └────────┬──────────┘
          ▼
 ┌─ Upgrade ─────────┐   暂停游戏，弹三选一
 └────────┬──────────┘
          ▼  (玩家选完)
     ┌────┴─────┐
     │ isMapEnd │───── 是 ──▶ Shop → WarpGate → NextMap
     └────┬─────┘
          │ 否
          ▼
     NextWave (回到 Spawning)
```

### 一张地图切换的生命周期

```
  Map K 的 Wave 5 → WaveEnd → Upgrade
                        │
                        ▼
                    ┌─ Shop ──┐   玩家消费，刷新、回血、买升级
                    └────┬────┘
                         │ 关闭商店
                         ▼
                  ┌─ WarpGateReveal ──┐   传送门物件出现（tween）
                  └─────────┬─────────┘
                            │ 玩家走过去触碰
                            ▼
                     ┌─ Transition ──┐   黑屏 / 转场
                     └───────┬───────┘
                             ▼
                       Map K+1 加载、玩家归位 → Wave K×5+1 开始
```

## 2. 数据结构（最重要）

> "Bad programmers worry about the code. Good programmers worry about data structures."

### 2.1 RunSession — 一局运行时状态

```typescript
/** 一局游戏的状态容器。死亡 / 胜利时整体丢弃，新局新建。 */
export interface RunSession {
    // ─── 进度 ──────────────────────────────
    mapIndex: number;        // 0..2（当前第几张地图）
    waveIndex: number;       // 1..15（当前第几波，跨地图连续计数）
    waveIndexInMap: number;  // 1..5（当前地图内的第几波）
    phase: RunPhase;

    // ─── 时间 ──────────────────────────────
    elapsedTotal: number;       // 一局总计经过秒数
    elapsedInWave: number;      // 当前波经过秒数
    timeSinceLastKill: number;  // 距上次击杀秒数（金币策略用）

    // ─── 经济 ──────────────────────────────
    gold: number;            // 当前持有金币

    // ─── 战斗统计 ──────────────────────────
    comboKills: number;      // 当前连杀数（5 秒无击杀则清零）
    totalKills: number;      // 本局累计击杀
    enemiesAlive: number;    // 当前场上敌人数

    // ─── 升级状态（只读视图，真值在 BuffSystem）──
    acquiredUpgrades: Set<string>;  // 已获得的 upgradeId（用于 evolvesFrom 判定）
}

export enum RunPhase {
    Spawning,         // 波次刷怪中
    Clearing,         // 波次已停刷，清场中
    WaveEnd,          // 波结束，准备结算
    Upgrade,          // 升级选择（暂停）
    Shop,             // 商店（暂停，仅地图末波）
    WarpGateWaiting,  // 等玩家触碰传送门
    Transition,       // 地图切换转场
    GameOver,
    Victory,
}
```

### 2.4 SpawnerRule — 刷怪规则

```typescript
export interface SpawnerRule {
    enemyId: string;          // 对接 enemy/config/enemyConfig
    interval: number;         // 每 N 秒触发一次
    batch: number;            // 每次生成 N 只
    maxAlive: number;         // 本规则同时场上上限（达上限则本次 tick 跳过）
    pattern: SpawnPattern;
    /** pattern === Fixed 时使用，世界坐标 */
    fixedPos?: { x: number; y: number };
    /** Cluster 模式用，默认 3 */
    clusterSize?: number;
    /** 启动延迟，秒。默认 0 */
    startDelay?: number;
}

export enum SpawnPattern {
    Edge,     // 摄像机视口边缘随机
    Ring,     // 玩家周围环形（围剿感）
    Cluster,  // 远处聚集一团（精英小队）
    Fixed,    // 地图固定点（Boss）
}
```

### 2.6 GoldGainContext + GoldModifier — 金币修饰器链

> **金币获取的核心设计**：不是"加个变量"，而是一条**修饰器链**。
> 所有"连杀加成 / 精巧杀戮 / 贪婪升级" 都是独立的 Modifier 子类，互不感知。

```typescript
export interface GoldGainContext {
    source: GoldSource;
    baseAmount: number;       // 原始金币（enemyConfig 里定义）
    enemyId?: string;         // source === Kill 时有值
    session: Readonly<RunSession>;  // 只读快照
}

export enum GoldSource {
    Kill,        // 击杀敌人
    WaveClear,   // 波次完成奖励
    Chest,       // 宝箱 / 破坏物
    ShopRefund,  // 商店退款（保留接口）
}

export interface GoldModifier {
    readonly id: string;
    readonly priority: number;   // 小的先执行（百分比和固定值的相对顺序在这里控制）
    apply(ctx: GoldGainContext, currentAmount: number): number;
}
```

**关键设计**：所有策略性金币规则通过 `registerModifier` 挂入，游戏启动注册"内置规则"，升级/Buff 注册"动态规则"，**没有 if 地狱**。

## 4. WaveDirector 状态机

### 4.1 职责

- 按 `MapDef.waves` 顺序推进波次
- 每波驱动所有 `SpawnerRule` 的 interval tick
- 侦测 "刷怪窗口结束" / "场上清零" / "清场超时"
- 发出波次生命周期事件给上层（`RunOrchestrator` / UI / 商店 / 传送门）

### 4.2 状态与转移

```typescript
export class WaveDirector {
    private _phase: WavePhase = WavePhase.Idle;
    private _wave: WaveDef | null = null;
    private _timer = 0;              // 通用计时器
    private _spawnerStates: SpawnerRuntime[] = [];

    startWave(wave: WaveDef): void { /* Idle → Spawning */ }
    tick(dt: number, session: RunSession): void {
        switch (this._phase) {
            case WavePhase.Spawning: this._tickSpawning(dt, session); break;
            case WavePhase.Clearing: this._tickClearing(dt, session); break;
            // Idle / Ended 不 tick
        }
    }
}

enum WavePhase { Idle, Spawning, Clearing, Ended }
```

### 4.3 关键算法

**Spawning tick**：
```
timer += dt
如果 timer >= wave.duration: 转到 Clearing, timer = 0
否则:
    对每个 spawnerState:
        如果 spawnerState.cooldown > 0: cooldown -= dt, continue
        如果 场上此规则敌人数 >= maxAlive: continue
        spawn(rule)
        spawnerState.cooldown = rule.interval
```

**Clearing tick**：
```
timer += dt
如果 session.enemiesAlive === 0: 转到 Ended, 发 'wave:end' 事件
否则如果 timer >= wave.clearingTimeout:
    强杀所有敌人（EnemyBase.forceKill()）
    转到 Ended, 发 'wave:end' 事件
```

**清场超时的设计意图**：防止"有一只怪卡在地图死角永远不死"导致游戏卡死。**无特殊情况**，不需要在场景里加导航网格、敌人寻路回头等复杂方案。

### 4.4 与 `GameSession` 的关系

- `GameSession.ESessionPhase` 是**全局会话阶段**（Playing / Paused / GameOver）
- `RunSession.RunPhase` 是**关卡内部阶段**（Spawning / Shop / ...）
- 关系：`GameSession.phase === Playing` 时，`RunSession.phase` 才可能流动
- 商店 / 升级 期间保持 `GameSession` 为 Playing，但停止 `GameLoop.update` 里的战斗系统（由 `RunOrchestrator` 控制是否 tick `WaveDirector`）

## 5. EnemySpawner — 4 种刷怪模式

> 所有模式都**不依赖地图模板的预置点**，直接从玩家位置 / 摄像机 / 地图参数算坐标。

### 5.1 Edge（最常用）

从摄像机视口边缘随机一点生成，偏玩家视野外一点，玩家"看不到生成瞬间"。

```typescript
function spawnAtEdge(cameraRect: Rect, marginOut = 64): Vec2 {
    const side = Math.floor(Math.random() * 4);  // 0=上 1=右 2=下 3=左
    const r = cameraRect;
    switch (side) {
        case 0: return v2(rand(r.xMin, r.xMax), r.yMax + marginOut);
        case 1: return v2(r.xMax + marginOut, rand(r.yMin, r.yMax));
        case 2: return v2(rand(r.xMin, r.xMax), r.yMin - marginOut);
        case 3: return v2(r.xMin - marginOut, rand(r.yMin, r.yMax));
    }
}
```

### 5.2 Ring（围剿）

以玩家为中心，半径 `R` 的圆周上均匀分布 N 点（`rule.batch` 为 N）。

```typescript
function spawnAtRing(playerPos: Vec2, count: number, radius = 300): Vec2[] {
    const pts: Vec2[] = [];
    const offset = Math.random() * Math.PI * 2;
    for (let i = 0; i < count; i++) {
        const a = offset + (Math.PI * 2 * i) / count;
        pts.push(v2(playerPos.x + Math.cos(a) * radius,
                    playerPos.y + Math.sin(a) * radius));
    }
    return pts;
}
```

### 5.3 Cluster（精英小队）

远离玩家 600px 处随机一点 `C`，围绕 `C` 半径 80 内生成 `clusterSize` 只。

### 5.4 Fixed（Boss）

直接用 `rule.fixedPos`，不随机。通常与 `isBoss: true` 的 Wave 搭配。

### 5.5 生成后接入现有系统

- 调用 `EnemyFactory.create(enemyId, worldPos, parentNode)`（需封装；内部走对象池）
- 注册到 `World` / `EnemyQuery` 供 `findNearestEnemy` 使用
- 敌人死亡时监听 `enemy:death` 事件 → `session.enemiesAlive--`

## 7. 金币系统

### 7.1 架构图

```
 ┌────────────────┐
 │  击杀事件      │
 │  波次结束      │    产生 GoldGainContext
 │  宝箱 / 事件    │ ────────────────────▶┐
 └────────────────┘                       │
                                          ▼
                           ┌─ GoldSystem.gainGold(ctx) ─┐
                           │                            │
                           │  amount = ctx.baseAmount   │
                           │                            │
                           │  for mod in modifiers:     │
                           │    amount = mod.apply(...) │
                           │                            │
                           │  session.gold += floor(a)  │
                           │  emit 'gold:gained'        │
                           └────────────────────────────┘
                                          │
 ┌─ UI HUD 金币跳字 ───────────────────────┤
 ├─ 音效 / 粒子 ──────────────────────────┤
 └─ 成就 / 统计 ──────────────────────────┘
```

### 7.2 内置 Modifier（本文档只负责预留接口，策划可后续扩展）

```typescript
/** 连杀加成：5 连杀内每级 +5%，最高 +50% */
class ComboGoldModifier implements GoldModifier {
    id = 'gold:combo';
    priority = 100;
    apply(ctx, amount) {
        if (ctx.source !== GoldSource.Kill) return amount;
        const bonus = Math.min(ctx.session.comboKills * 0.05, 0.5);
        return amount * (1 + bonus);
    }
}

/** 精巧杀戮：距上次击杀 ≥ 5 秒，下一杀 ×2 */
class PreciseKillModifier implements GoldModifier {
    id = 'gold:precise';
    priority = 110;
    apply(ctx, amount) {
        if (ctx.source !== GoldSource.Kill) return amount;
        return ctx.session.timeSinceLastKill >= 5 ? amount * 2 : amount;
    }
}
```

### 7.3 升级驱动的 Modifier 注册

当玩家获得"贪婪"类升级时：
```typescript
// 在 UpgradeApplier（升级系统文档负责实现）里
case 'gold_bonus_upgrade':
    goldSystem.registerModifier({
        id: `gold:upgrade:${upg.id}`,
        priority: 50,
        apply: (ctx, amount) => amount * (1 + upg.ratio),
    });
    break;
```

### 7.4 数据钩子更新时机

```
// 在 MonsterDeath 事件处理中
session.totalKills++
session.enemiesAlive--
session.comboKills++           // 连杀 +1
session.timeSinceLastKill = 0  // 重置

// 在 RunSession tick 里（每帧）
session.timeSinceLastKill += dt
if (session.timeSinceLastKill >= 5) {
    session.comboKills = 0     // 连杀中断
}
```

**好品味点**：连杀/精巧是 **session 上的两个数字**，不是"事件处理器内部的状态"。任何 Modifier 都能读它，任何 UI 都能显示它。

### 8.1 生命周期

```typescript
export class WarpGateManager {
    private _node: Node | null = null;
    private _targetMapIndex = -1;
    private _onEntered: (() => void) | null = null;

    /** 切图时销毁 */
    cleanup(): void;
}
```

### 8.2 触发半径

建议 40-50px。使用物理 sensor trigger，或每帧距离判定（**距离判定更简单，首版选它**）。

### 8.3 视觉反馈

- 出现时：`scale 0 → 1`，`opacity 0 → 255`，`backOut` 缓动，持续 0.4s
- 常驻：旋转动画 / 粒子特效（Tiny Swords 的"fx/portal"素材）
- 玩家靠近（100px 内）：UI 提示"走入传送门进入下一区域"
- 玩家触碰：黑屏 0.3s → 切图 → 淡入

### 8.4 与 `RunOrchestrator` 的衔接

```typescript
// RunOrchestrator 里，商店关闭后：
warpGate.reveal(
    currentMap.warpGatePos,
    mapRoot,
    () => this._onEnterGate(),
);
session.phase = RunPhase.WarpGateWaiting;

// 玩家触碰后：
private _onEnterGate(): void {
    session.phase = RunPhase.Transition;
    transitionFX.blackFade(0.3, () => {
        this._unloadMap(session.mapIndex);
        session.mapIndex++;
        this._loadMap(session.mapIndex);
        session.phase = RunPhase.Spawning;
        this._startWave(session.waveIndex + 1);
        transitionFX.fadeIn(0.3);
    });
}
```

## 10. 难度缩放

### 10.1 基于 waveIndex 的线性缩放

```typescript
/** 全局敌人属性缩放系数。在 spawner 创建敌人后应用一次。 */
export function getWaveScaling(waveIndex: number): WaveScaling {
    return {
        hpMult:    1 + (waveIndex - 1) * 0.12,   // 每波 +12% HP
        atkMult:   1 + (waveIndex - 1) * 0.08,   // 每波 +8%  ATK
        speedMult: Math.min(1 + (waveIndex - 1) * 0.015, 1.4),  // 上限 ×1.4
    };
}
```

### 10.2 地图间阶梯跳变（可选）

如果觉得"线性 + 15 波"的难度太平，可在地图切换时加一次性跳变：
```typescript
if (session.waveIndexInMap === 1 && session.mapIndex > 0) {
    scaling.hpMult  *= 1.3;
    scaling.atkMult *= 1.2;
}
```

**Linus 建议**：**先只用 10.1 的线性缩放**。10.2 属于"万一不够难再加"，**不要预加优化**。

## 11. 与现有系统的对接

### 11.1 GameLoop / GameSession 需要的改造

当前 `GameLoop.update` 无条件跑所有 system（输入管线），没有"暂停"逻辑。新关卡系统要求：
- 升级 / 商店 / 传送门转场期间，战斗相关系统**不要 tick**
- 输入系统**要 tick**（鼠标移动等 UI 依赖）

建议引入一个 `RunOrchestrator`（本文档定义的新类），持有 `WaveDirector / ShopManager / WarpGateManager`，暴露 `tick(dt)`。
`GameLoop.update` 末尾 `RunOrchestrator.inst?.tick(dt)`，orchestrator 内部根据 `session.phase` 决定 tick 哪些子系统。

### 11.2 与 PlayerExperience 的关系

**不使用经验驱动升级**（本设计走"每波必升"）。
- `PlayerExperience` 现有的 `addXp / onLevelUp` 保留
- 但 "经验满 → 弹升级窗" 的胶水**不再搭**
- 改为 WaveDirector 的 `wave:end` 事件 → 打开升级面板

如果后续想同时保留经验升级（双轨），只要在 orchestrator 里加一条监听即可，**数据结构不需要改**。

### 11.3 与 PlayerControl 的对接点

| 需要 | 现有 | 需要改造 |
|------|------|---------|
| 死亡通知 | `PlayerControl._onDeath` → `GameSession.onPlayerDeath` | 无 |
| 获得金币的入口 | 无 | 新增 `session.gold += amount`（由 GoldSystem 写） |
| 暂停输入 | 无全局暂停机制 | orchestrator 在 Upgrade/Shop 期间冻结 `PlayerControl.update`（通过 session.phase 判断即可） |

### 11.4 与 upgrades.xlsx / BuffSystem 的对接

- 升级系统（外部）提供 `UpgradeRepository` / `UpgradeSelector` / `UpgradeApplier`
- 本文档调用：
  - `UpgradeSelector.select(3, session.acquiredUpgrades)` → 波尾 / 商店
  - `UpgradeApplier.apply(upgradeId, player)` → 玩家选中后
- 应用完毕后：`session.acquiredUpgrades.add(upgradeId)`

## 12. 新增 GamePhase 的建议

《01-核心游戏循环》里原 `GamePhase` 只有 7 个：
`Boot, Home, Loading, Playing, Reward, Transition, GameOver`

本设计需要额外的 Shop 阶段。**建议**在那边补一个：
```typescript
enum GamePhase {
    Boot, Home, Loading,
    Playing,
    Reward,       // 升级三选一
    Shop,         // ← 新增
    Transition,   // 地图切换
    GameOver,
}
```

转换规则：
```
Playing → Reward → (normal) → Playing           （非地图末波）
Playing → Reward → Shop → Transition → Playing  （地图末波）
```

## 13. 与旧文档的决裂点

| 旧文档的设计 | 新设计的替代 |
|------------|-------------|
| `RoomTemplate` 房间模板 | 删除。用 `MapDef` 的单张固定地图 |
| `TileMapRenderer` 的 terrain[][] 驱动 | 保留渲染器，但输入改为 `MapGenerator.generateArena` 的结果 |
| `FloorType` 枚举 6 种 | 删除。改由 `WaveDef.isBoss` / `MapDef.waves[last]` 后的 Shop 流表达 |

| 按层解锁敌人类型 | 删除。波次表直接写 `enemyId`，策划自由 |
| `FloorScaling` 按层缩放 | 改名 `WaveScaling`，基于 waveIndex |
| `WarpGateManager` "清完敌人 → 传送门 → 下一层" | 语义改变：传送门**只在地图末尾**出现，不在每波出现 |
| 玩家触碰传送门立即切层 | 中间插入 Shop 阶段 |

## 15. 关键设计决策表

| 决策 | 选择 | 拒绝项 | 理由 |
|------|------|-------|------|
| 地图形式 | 固定 3 张 | 程序化生成 / 房间模板 | 参与感 > 多样性，固定地图给得了氛围 |
| 波次模型 | 分段波次 | 纯连续流（幸存者）| 每波结算 + 升级 = 清晰的成长节奏 |
| 升级触发 | 每波必升 | 经验驱动 | 节奏可预测，27 条升级正好匹配 15 波 |
| 升级叠加 | 仅 `evolvesFrom` 显式分级 | 同条目无限叠加 | 防平衡 bug，数据可控 |
| 商店位置 | 仅地图末波（5/10 波后）| 每波商店 / 随机商店 | 不打断战斗节奏，给地图切换仪式感 |
| 商店构成 | 3 卡 + 回血 + 刷新 | 消耗品 / 装备 / Meta 换 | 复用升级数据源，无新逻辑 |
| 金币系统 | Modifier 链架构 | 硬编码 if 分支 | 策略规则可无限扩展，零改核心 |
| 地图切换 | 物件传送门 | 菜单按钮 / 自动切 | 仪式感，玩家主动进入新世界 |
| 清场超时 | 15 秒强清场 | 完美寻路 / 复杂导航 | 用数据解决 edge case，不碰 AI |
| 数据驱动程度 | 波次/商店全配表 | 代码写死 | 策划可独立迭代 |
| 固定地图大小 | 比屏幕大 1.5-2 倍 | 无限大 / 单屏 | 走位空间 + 不迷路 |
| `enemiesAlive` 计数的作用 | 仅用于 Clearing 判定 + 场上上限 | 用于通关判定 | 通关靠波次推进，不是清空场地 |
| 摄像机 | 跟随玩家 + 地图边界 | 无边界 / 固定 | 暴露战场范围但不失控 |
