实现让玩家"再来一局"的核心循环：**随机技能选择 + 局内成长 + 死亡重来 + 跨局Meta进度**。这是肉鸽游戏的灵魂。

## 1. 肉鸽游戏的本质

一句话：**用随机性制造"这一局独特的我"的体验**。

```
┌───────────────────────────────────┐
│           一局游戏                 │
│                                   │
│  随机技能组合 → 独特的 Build      │
│  随机敌人配置 → 独特的挑战        │
│  随机奖励掉落 → 独特的决策        │
│                                   │
│  死亡 → 清零局内成长              │
│       → 保留 Meta 进度            │
│       → 带着新策略再来一局         │
└───────────────────────────────────┘
```

### 参考：幽灵射手的做法

幽灵射手其实是**半肉鸽**：
- 技能选择随机三选一 ✅
- 但 `arrSkill` 和金币跨局保留 ❌（更像 Archero 的永久成长）
- 没有程序化地牢 ❌
- 关卡线性推进而非每局重来 ❌

我们要做更纯粹的肉鸽：每局从零开始，死亡清零，但有 Meta 层面的永久解锁。

### 技能选择算法

```typescript
class SkillSelector {
    /**
     * 从技能池中按稀有度权重随机选 N 个不重复技能
     * @param pool 可选技能列表
     * @param count 选几个（通常为 3）
     * @param ownedSkills 已拥有的技能（用于升级判断）
     */
    select(
        pool: SkillDef[],
        count: number,
        ownedSkills: Map<string, number>
    ): SkillOption[] {
        const candidates = this._buildCandidates(pool, ownedSkills);
        const selected: SkillOption[] = [];

        for (let i = 0; i < count && candidates.length > 0; i++) {
            const index = this._weightedRandom(candidates);
            const skill = candidates.splice(index, 1)[0];
            const currentLevel = ownedSkills.get(skill.def.id) ?? 0;

            selected.push({
                skill: skill.def,
                isUpgrade: currentLevel > 0,
                nextLevel: currentLevel + 1,
            });
        }

        return selected;
    }

    private _buildCandidates(
        pool: SkillDef[],
        owned: Map<string, number>
    ): WeightedSkill[] {
        return pool
            .filter(s => {
                const level = owned.get(s.id) ?? 0;
                return level < s.maxLevel;  // 未满级才能出现
            })
            .map(s => ({
                def: s,
                weight: this._rarityWeight(s.rarity),
            }));
    }

    private _rarityWeight(rarity: SkillRarity): number {
        const weights = { 0: 60, 1: 25, 2: 12, 3: 3 };
        return weights[rarity];
    }

    private _weightedRandom(items: WeightedSkill[]): number {
        const total = items.reduce((sum, i) => sum + i.weight, 0);
        let roll = Math.random() * total;
        for (let i = 0; i < items.length; i++) {
            roll -= items[i].weight;
            if (roll <= 0) return i;
        }
        return items.length - 1;
    }
}

interface SkillOption {
    skill: SkillDef;
    isUpgrade: boolean;
    nextLevel: number;
}
```

## 3. 局内成长

### 经验与等级

每局从 1 级开始。升级时触发技能选择。

```typescript
class RunProgress {
    level: number = 1;
    exp: number = 0;
    gold: number = 0;
    skills: Map<string, number> = new Map(); // skillId → level

    private _expToNextLevel(): number {
        return 10 + this.level * 5;  // 1级需要15经验，2级20，3级25...
    }

    addExp(amount: number): boolean {
        this.exp += amount;
        if (this.exp >= this._expToNextLevel()) {
            this.exp -= this._expToNextLevel();
            this.level++;
            return true; // 升级了，触发技能选择
        }
        return false;
    }

    addSkill(skillId: string): void {
        const current = this.skills.get(skillId) ?? 0;
        this.skills.set(skillId, current + 1);
    }
}
```

### 经验来源

| 来源 | 经验值 | 说明 |
|------|--------|------|
| 击杀 Pawn | 2 | 基础小兵 |
| 击杀 Archer | 3 | 远程兵 |
| 击杀 Warrior | 5 | 重装兵 |
| 击杀 Lancer | 4 | 冲刺兵 |
| 击杀 Monk | 6 | 辅助型（优先击杀价值高）|
| 击杀 Boss | 20 | Boss |
| 通过一层 | 5 | 额外奖励 |

## 4. 死亡循环

### 死亡时发生什么

```typescript
function onPlayerDeath(runProgress: RunProgress, metaState: MetaPlayerState): void {
    // 1. 结算本局成就
    const result: RunResult = {
        floorsCleared: runProgress.floor,
        enemiesKilled: runProgress.killCount,
        goldEarned: runProgress.gold,
        skillsUsed: Array.from(runProgress.skills.keys()),
        duration: runProgress.elapsedTime,
    };

    // 2. 更新 Meta 进度
    metaState.highestFloor = Math.max(metaState.highestFloor, result.floorsCleared);
    metaState.totalGold += Math.floor(result.goldEarned * 0.5); // 死亡保留50%金币
    metaState.totalRuns++;

    // 3. 检查解锁条件
    checkUnlocks(result, metaState);

    // 4. 存档
    saveMetaState(metaState);

    // 5. 清零局内状态（下一局从头开始）
    // runProgress 直接丢弃，新局创建新实例
}
```

### 这是个设计决策

**为什么死亡要清零？** 因为这是肉鸽游戏的核心张力来源：
- 每个选择都有重量（选错技能可能导致这局崩盘）
- 每次生存都有成就感（"这一局我打到了第 15 层！"）
- 每局都是新的体验（不同技能组合 = 不同玩法）

幽灵射手没有这个设计，所以它更像一个线性关卡推进游戏。我们通过死亡清零让"每一局"都有意义。

    | { type: 'reach_floor'; floor: number }       // 到达某层

    const base: Record<string, number> = {
        pawn: 1, archer: 2, warrior: 3, lancer: 3, monk: 4

## 7. 随机性的控制

> 好的肉鸽游戏不是完全随机，而是"受控的随机"。

### 保底机制

```typescript
class PitySystem {
    private _noEpicCount: number = 0;
    private _noLegendaryCount: number = 0;

    onSkillSelect(rarity: SkillRarity): void {
        if (rarity >= SkillRarity.Epic) this._noEpicCount = 0;
        else this._noEpicCount++;

        if (rarity >= SkillRarity.Legendary) this._noLegendaryCount = 0;
        else this._noLegendaryCount++;
    }

    getRarityBoost(): Record<SkillRarity, number> {
        return {
            [SkillRarity.Common]: 0,
            [SkillRarity.Rare]: 0,
            [SkillRarity.Epic]: this._noEpicCount >= 5 ? 20 : 0,        // 5次没出Epic，+20权重
            [SkillRarity.Legendary]: this._noLegendaryCount >= 15 ? 10 : 0, // 15次没出传说，+10权重
        };
    }
}
```

## 关键设计决策

| 决策 | 理由 |
|------|------|
| 局内清零 + Meta 永久 | 经典肉鸽双层循环，短期刺激 + 长期目标 |
| 技能按稀有度加权而非等概率 | 常见技能保底体验，稀有技能创造惊喜 |
| 保底系统 | 避免连续 15 次全是 Common 的沮丧体验 |
| 协同加权 | 引导玩家感受"Build"的概念，而非纯随机散装 |
| 死亡保留 50% 金币 | 纯清零太惩罚，部分保留给玩家"这局没白打"的感觉 |
