## 一、核心概念：三层抽象
```
层级 1: 职业 (Class)          ← 决定"怎么打"
         │
         ├── 弓箭手 Archer     → 远程弹道，射箭
         ├── 战士 Warrior      → 近战劈砍，冲锋
         └── 召唤师 Summoner   → 召唤物 AI 攻击
         │
层级 2: 流派 (Archetype)      ← 决定"往哪个方向强化"
         │
         │  弓箭手下的流派：
         ├── 速射流 RapidFire  → 高攻速密度
         ├── 弹幕流 Barrage    → 多弹道覆盖
         ├── 爆发流 Burst      → 蓄力高单发
         └── 持续流 Sustain    → 穿透/弹射/DOT
         │
         │  战士下的流派（示例）：
         ├── 狂战流 Berserker  → 连击速度，牺牲防御
         ├── 铁壁流 Guardian   → 格挡反弹，持久战
         └── 冲锋流 Charger   → 位移 AOE，机动性
         │
层级 3: 升级效果 (Upgrade)    ← 决定"具体强了什么"
         │
         ├── 通用效果: buff（属性）、hit_effect（命中触发）
         └── 职业专属效果: shoot_policy（弓箭手）、melee_combo（战士）
```
### 关键设计规则
| 规则 | 说明 |
|------|------|
| **职业决定行为** | 职业通过 `PlayerBehavior` 决定攻击方式、FSM 状态、专属配置 |
| **流派决定 Buff 池** | 流派通过 `ArchetypeManager` 过滤升级候选池 |
| **职业 × 流派 = 正交** | 职业系统和流派系统解耦。同一个职业下可以有多个流派方向 |
| **新职业 = 新 Behavior** | 不改 PlayerControl，只新增 Behavior + Config |
| **新流派 = 新 JSON 标签** | 不改升级系统代码，只在 JSON 里加 archetype 标签 |

## 三、代码结构详解
### 3.1 PlayerBehavior 抽象类
```typescript
// player/base/PlayerBehavior.ts
import { PlayerBehaviorBase } from '../../baseSystem/player';
import type { IState } from '../../baseSystem/fsm';
import type { ActionComp } from '../component';
import type { PlayerCtx } from '../states/PlayerContext';
import type { PlayerConfigData } from '../config/playerConfig';
import type { PropertyBaseConfig } from '../../entity/EntityPropertyMgr';
import type { UpgradeEffect, EffectContext } from '../../upgrade/types';
export abstract class PlayerBehavior extends PlayerBehaviorBase {
    /** 职业专属运行时配置（动画路径、攻击参数等） */
    abstract readonly config: PlayerConfigData & Record<string, any>;
    /** 职业专属属性初始值（从 JSON 加载） */
    abstract readonly propertyCfg: PropertyBaseConfig;
    /** 动画定义表（key = EPlayerAnim 枚举值） */
    abstract readonly anims: Record<string, AnimEntry>;
    /** 创建职业专属攻击状态 */
    abstract createAttackState(): IState<PlayerCtx>;
    /** 是否想发起攻击（替代原来的 shootPolicy.wantShoot） */
    abstract wantAttack(
        act: ActionComp,
        hasTarget: boolean,
        isMoving: boolean,
    ): boolean;
    /**
     * 职业专属升级效果处理
     * @returns true = 已处理，UpgradeManager 跳过；false = 未处理，走通用逻辑
     */
    applyClassEffect(effect: UpgradeEffect, ctx: EffectContext): boolean {
        return false;
    }
    /** 初始化钩子（PlayerControl.start 调用） */
    onInit(ctx: PlayerCtx): void {}
    /** 销毁钩子 */
    onDispose(): void {}
}
```
### 3.2 ArcherBehavior 示例
```typescript
// player/archer/ArcherBehavior.ts
import { playerBehavior } from '../../baseSystem/player';
import { PlayerBehavior } from '../base/PlayerBehavior';
import { EPlayerClass } from '../base/types';
import { ArcherAttackState } from './ArcherAttackState';
import { archerConfig, archerAnims } from './archerConfig';
import archerProps from './archer.json';
import type { IShootPolicy } from '../../shoot/types';
import { HoldToShoot } from '../../shoot/ShootPolicies';
import type { UpgradeEffect, EffectContext } from '../../upgrade/types';
@playerBehavior
export class ArcherBehavior extends PlayerBehavior {
    readonly typeId = EPlayerClass.Archer;
    readonly config = archerConfig;
    readonly propertyCfg = archerProps;
    readonly anims = archerAnims;
    private _shootPolicy: IShootPolicy = new HoldToShoot();
    createAttackState() {
        return new ArcherAttackState(this);
    }
    wantAttack(act, hasTarget, isMoving) {
        return this._shootPolicy.wantShoot(act, hasTarget, isMoving);
    }
    get shootPolicy() { return this._shootPolicy; }
    setShootPolicy(policy: IShootPolicy) {
        this._shootPolicy = policy;
    }
    resetShootPolicy() {
        this._shootPolicy = new HoldToShoot();
    }
    applyClassEffect(effect: UpgradeEffect, _ctx: EffectContext): boolean {
        if (effect.type === 'shoot_policy') {
            this.setShootPolicy(createShootPolicy(effect.data));
            return true;
        }
        return false;
    }
}
```

### 3.4 PlayerControl 核心改动（关键代码片段）
```typescript
// player/PlayerControl.ts（改造后关键片段）
import { PlayerBehaviorFactory } from '../baseSystem/player';
import { EPlayerClass } from './base/types';
import './behaviors';  // side-effect import 触发所有注册
@ccclass('PlayerControl')
export class PlayerControl extends Component {
    @property({ type: CCString })
    behaviorId: string = EPlayerClass.Archer;
    private _behavior!: PlayerBehavior;
    start() {
        // 从工厂创建职业行为
        this._behavior = PlayerBehaviorFactory.create<PlayerBehavior>(this.behaviorId);
        // 用职业配置初始化属性
        this._playerProp = new PlayerProperty();
        this._playerProp.setInitialValues(this._behavior.propertyCfg);
        // ... 其余初始化不变 ...
        this._behavior.onInit(this._ctx);
    }
    private _initFsm() {
        // ... 共有状态不变 ...
        this._fsm.addState(EPlayerState.Attack, this._behavior.createAttackState());
        // ↑ 原来是 new PlayerShootState()，现在由 behavior 决定
    }
    lateUpdate(dt: number) {
        // ...
        const wantAttack = this._behavior.wantAttack(act, hasTarget, isMoving);
        if (wantAttack) {
            this._fsm.changeState(EPlayerState.Attack);
        } else {
            this._fsm.changeState(isMoving ? EPlayerState.Run : EPlayerState.Idle);
        }
        // ...
    }
}
```

## 四、数据流：一次攻击从头到尾
### 弓箭手
```
输入(鼠标/键盘)
  → ActionComp
  → ArcherBehavior.wantAttack() [检查 ShootPolicy]
  → FSM → EPlayerState.Attack
  → ArcherAttackState.update()
    → ShootResolver.resolve() [读属性: ArrowSpeed, ExtraProjectiles...]
    → ShootResolver.snapshotProjectileConfig() [穿透/弹射/分裂]
    → ProjectilePool.acquire() → ArrowProjectile.init()
      → 飞行 → 碰撞 → onHit callback
        → HitEffectMgr.execute(ctx) [DamageHitEffect → LifestealHitEffect → ...]
```
### 战士（预期）
```
输入(鼠标/键盘)
  → ActionComp
  → WarriorBehavior.wantAttack() [近距离有敌人 + 按键]
  → FSM → EPlayerState.Attack
  → WarriorAttackState.update()
    → 播放挥砍动画
    → 扇形范围检测 [CleaveRange, CleaveAngle]
    → 对范围内敌人遍历
      → HitEffectMgr.execute(ctx) [同一套命中效果管线]
```
### 共享部分
```
两个职业共享：
- 属性系统 (EntityPropertyMgr)
- Buff 系统 (EntityBuffMgr)
- 命中效果管线 (HitEffectMgr)
- 升级系统 (UpgradeManager + UpgradeEffectRegistry)
- 经验系统 (PlayerExperience)
- 移动/闲置/受击/死亡状态
- ECS 输入管线 (RawInputComp → ActionComp → VelocityComp)
```

## 五、升级效果注册体系
```
                    UpgradeEffectRegistry
                  ┌──────────────────────┐
                  │ Map<type, handler>   │
                  │ .apply(effect, ctx)  │
                  │ .remove(effect, ctx) │
                  └──────────┬───────────┘
                             │
          ┌──────────────────┼──────────────────┐
          ▼                  ▼                  ▼
  upgrade/baseEffects.ts   archer/             warrior/
  注册通用效果:             archerEffects.ts   warriorEffects.ts
  - 'buff'                 注册弓箭手效果:     注册战士效果:
  - 'hit_effect'           - 'shoot_policy'   - 'melee_combo'
```
### 效果处理器接口
```typescript
// upgrade/UpgradeEffectRegistry.ts
export interface IEffectHandler {
    apply(data: any, ctx: EffectContext): void;
    remove?(data: any, ctx: EffectContext): void;
}
export interface EffectContext {
    buffMgr: EntityBuffMgr;
    buffOwner: IBuffOwner;
    hitEffectMgr: HitEffectMgr;
}
const handlers = new Map<string, IEffectHandler>();
export const UpgradeEffectRegistry = {
    register(type: string, handler: IEffectHandler): void {
        handlers.set(type, handler);
    },
    apply(effect: UpgradeEffect, ctx: EffectContext): boolean {
        const h = handlers.get(effect.type);
        if (!h) return false;
        h.apply(effect.data, ctx);
        return true;
    },
    remove(effect: UpgradeEffect, ctx: EffectContext): void {
        handlers.get(effect.type)?.remove?.(effect.data, ctx);
    },
};
```
### 通用效果注册
```typescript
// upgrade/baseEffects.ts
import { UpgradeEffectRegistry } from './UpgradeEffectRegistry';
UpgradeEffectRegistry.register('buff', {
    apply(data, ctx) { ctx.buffMgr.addBuff(data, ctx.buffOwner); },
    remove(data, ctx) { ctx.buffMgr.removeBuff(data.id); },
});
UpgradeEffectRegistry.register('hit_effect', {
    apply(data, ctx) { ctx.hitEffectMgr.add(data); },
    remove(data, ctx) { ctx.hitEffectMgr.remove(data.id); },
});
```

## 六、流派系统（ArchetypeManager）
### 与职业的关系
```
职业选择 → 决定 Behavior（代码逻辑层面）
流派选择 → 决定 Buff 池过滤（数据/配置层面）
两者正交：
- 弓箭手可以选 弹幕流/爆发流/持续流
- 战士可以选 狂战流/铁壁流/冲锋流
```
### 数据结构
```typescript
// upgrade/ArchetypeManager.ts
export enum EArchetype {
    // 弓箭手流派
    Barrage  = 'barrage',
    Burst    = 'burst',
    Sustain  = 'sustain',
    // 战士流派
    Berserker = 'berserker',
    Guardian  = 'guardian',
    Charger   = 'charger',
}
export class ArchetypeManager {
    private _classType: EPlayerClass;
    private _archetype: EArchetype | null = null;
    constructor(classType: EPlayerClass) {
        this._classType = classType;
    }
    selectArchetype(archetype: EArchetype): void {
        this._archetype = archetype;
    }
    filterUpgradePool(all: UpgradeConfig[]): UpgradeConfig[] {
        return all.filter(u => {
            // 按职业过滤
            if (u.classTag !== 'all' && u.classTag !== this._classType) return false;
            // 按流派过滤核心 Buff
            if (u.category === 'archetype_core') {
                return u.archetype === this._archetype;
            }
            return true;
        });
    }
}
```

### Step 5：编写 WarriorAttackState.ts（20 分钟）
```typescript
import type { IState } from '../../baseSystem/fsm';
import type { PlayerCtx } from '../states/PlayerContext';
import { EPropertyId } from '../../config/enum/propertyEnum';
import { EnemyControl } from '../../enemy/EnemyControl';
import { createHitContext } from '../../hitEffects/types';
import { Vec3 } from 'cc';
export class WarriorAttackState implements IState<PlayerCtx> {
    enter(ctx: PlayerCtx) {}
    update(ctx: PlayerCtx, dt: number) {
        if (ctx.shootCooldown > 0) return;
        const atkSpeed = Math.max(ctx.prop.getValue(EPropertyId.AttackSpeed), 0.1);
        const baseDur = ctx.anim.animator.getClipDuration('attack');
        ctx.shootCooldown = baseDur / atkSpeed;
        ctx.anim.playOnce('attack');
        this._cleave(ctx);
    }
    exit(_ctx: PlayerCtx) {}
    private _cleave(ctx: PlayerCtx) {
        const range = ctx.prop.getValue(EPropertyId.CleaveRange);
        const angle = ctx.prop.getValue(EPropertyId.CleaveAngle);
        const facingAngle = ctx.body.scale.x >= 0 ? 0 : Math.PI;
        for (const enemy of EnemyControl.allEnemies) {
            if (!enemy.node.isValid || enemy.combat.isDead) continue;
            const dist = Vec3.distance(ctx.node.worldPosition, enemy.node.worldPosition);
            if (dist > range) continue;
            // 扇形角度检测
            const dir = enemy.node.worldPosition.clone().subtract(ctx.node.worldPosition);
            const enemyAngle = Math.atan2(dir.y, dir.x);
            const diff = Math.abs(enemyAngle - facingAngle);
            if (diff > (angle / 2) * (Math.PI / 180)) continue;
            // 复用同一套命中效果管线
            const hitCtx = createHitContext(
                ctx.prop, ctx.combat, enemy.combat, enemy.buffMgr, enemy.buffOwner,
            );
            hitCtx.targetNode = enemy.node;
            hitCtx.hitOriginPos = ctx.node.worldPosition.clone();
            ctx.hitEffectMgr.execute(hitCtx);
        }
    }
}
```

如果战士需要 `CleaveRange`、`CleaveAngle` 等新属性：
1. `config/enum/propertyEnum.ts` 加枚举
2. `config/propertyConfig/` 加对应 JSON
3. `config/propertyConfig/attributeConfigs.ts` 引入
4. `config/propertyConfig/propConfigMap.ts` 加映射
### Step 9：切换到战士（1 分钟）
在场景编辑器中，选中 Player 节点，将 `PlayerControl.behaviorId` 改为 `"warrior"`。
**总耗时：约 1-2 小时。核心代码（PlayerControl/UpgradeManager）改动：0 行。**

## 十、FAQ
### Q: 弓箭手的 ShootResolver 是弓箭手专属的吗？
**是的。** `ShootResolver` 计算弹道曲线，只有弓箭手用。战士不需要弹道，直接在 `WarriorAttackState` 里做范围判定。`ShootResolver` 由 `ArcherAttackState` 调用，不在 `PlayerControl` 里。
### Q: HitEffectMgr 是所有职业共享的吗？
**是的。** 无论弓箭手的箭命中还是战士的劈砍命中，最终都走 `HitEffectMgr.execute(ctx)`。灼烧、吸血、击退等效果对所有职业一视同仁。
### Q: 战士需要 ProjectilePool 吗？
**不需要。** 战士的攻击是即时范围判定，不生成弹道实体。`ProjectilePool` 和 `ArrowProjectile` 完全由弓箭手模块使用。
### Q: 新职业的升级 JSON 怎么和现有升级区分？
通过 `classTag` 字段。`"classTag": "warrior"` 的升级只有战士能选到。`"classTag": "all"` 的是通用升级（如攻击+30%、吸血率+8%），所有职业共享。
### Q: 流派系统现在需要实现吗？
**不需要。** 流派是数据层的过滤逻辑（`ArchetypeManager.filterUpgradePool`），不影响架构。当前先把职业行为框架搭好，流派系统按原设计文档在 P6 阶段实现。
### Q: 如果一个升级需要同时包含通用效果和职业专属效果怎么办？
一个 `UpgradeConfig` 的 `effects` 数组里可以混合多种 `type`。`UpgradeEffectRegistry` 逐个查表派发：
```json
{
    "id": "warrior-flame-strike",
    "classTag": "warrior",
    "effects": [
        { "type": "buff", "data": { "id": 3001, ... } },
        { "type": "melee_combo", "data": { "comboBonus": 0.5 } }
    ]
}
```
第一个走通用 `buff` handler，第二个走战士 `melee_combo` handler。混合使用，零冲突。
