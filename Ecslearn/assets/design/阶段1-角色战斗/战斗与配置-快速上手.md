## 一、配置目录结构

所有配置统一从 `game/config/index.ts` 导出，只需一行 import：

```typescript
import { getBuffDef, getSkillDef, getPayloadDef, ... } from '../config';
```

```
game/config/
├── index.ts              ← 统一入口（所有子模块从此导出）
├── enum/
│   └── propertyEnum.ts   ← EPropertyId / EPropertyConfigId
│
├── propertyConfig/       ← 属性曲线（每属性一个 JSON + 结构定义）
│   ├── attributeConfigs.ts
│   ├── propConfigMap.ts
│   ├── hp.json / attack.json / defense.json / ...
│   └── （共 13 个属性 JSON）
│
├── buffConfig/           ← Buff + 命中效果
│   ├── BuffConfigLoader.ts
│   ├── buffs.json        ← Buff 定义（id → BuffData）
│   ├── hitEffects.json   ← 命中效果定义（id → HitEffectData）
│   └── attackBoost.json / critBoost.json / ...（独立 Buff 文件）
│
├── skillConfig/          ← 技能
```

## 二、Buff 系统

### 架构层级

```
baseSystem/buff/           ← 框架层（与游戏无关）
├── BuffFactory.ts         ← 按 effectClass 创建 BuffRuntimeInfo
├── BuffEffectBase.ts      ← 效果脚本基类（onApply/onTick/onRemove）
├── BuffRuntimeInfo.ts     ← 运行时 Buff 实例
└── types.ts               ← BuffData / IBuffOwner / AttributeChange

game/entity/
├── EntityBuffMgr.ts       ← 挂载在实体上的 Buff 管理器
└── AttributeChangeResolver.ts  ← BuffEffect → PropertyMgr 修饰器桥

game/skill/effects/        ← 具体效果脚本（注册到 BuffFactory）
├── AttrModifierEffect.ts  ← 新版属性修改（替代 SimpleAttrBuffEffect）
├── PeriodicHealEffect.ts  ← 周期回血
├── SimpleAttrBuffEffect.ts ← 旧版属性修改（兼容）
├── AttackBoostEffect.ts   ← 攻击提升
├── CritBoostEffect.ts     ← 暴击提升
└── ...
```

### 添加新 Buff 的步骤

**方式 A：纯 JSON（属性修改类 Buff）**

在 `buffConfig/buffs.json` 中添加一条：

```json
{
    "buff.your_new_buff": {
        "id": 9001,
        "name": "新Buff",
        "duration": 10,
        "maxStack": 3,
        "effectClass": "SimpleAttrBuffEffect",
        "targetAttr": "Attack-Mul-Buff",
        "valuePerStack": 0.15
    }
}
```

**完成。** `effectClass` 对应已注册的效果脚本类名。

**方式 B：自定义效果脚本**

1. 在 `game/skill/effects/` 新建 `YourEffect.ts`：

```typescript
import { BuffEffectBase, buffEffect } from '../../../baseSystem/buff';

@buffEffect
export class YourEffect extends BuffEffectBase {
    static readonly effectClass = 'YourEffect';

    onApply(owner: IBuffOwner): void { /* 施加时 */ }
    onTick(owner: IBuffOwner, dt: number): void { /* 每帧 */ }
    onRemove(owner: IBuffOwner): void { /* 移除时 */ }
}
```

2. 在 `game/skill/effects/index.ts` 加一行 import
3. JSON 中 `"effectClass": "YourEffect"` 即可引用

### targetAttr 命名规则

```
{属性名}-{修饰类型}-Buff

属性名：Attack / Defense / Hp / CritRate / AttackSpeed / MoveSpeed / ...
修饰类型：
  Value → 加固定值
  Mul   → 乘百分比

例：
  "Attack-Value-Buff"    → 攻击 +30
  "AttackSpeed-Mul-Buff" → 攻速 ×1.2
  "CritRate-Value-Buff"  → 暴击率 +0.1
```

## 三、命中效果系统（HitEffect）

命中效果 = **弹道命中目标后**触发的逻辑链。

### 架构

```
baseSystem/hitEffect/
├── HitEffectFactory.ts    ← 按 effectClass 创建实例
└── HitEffectBase.ts       ← 基类（apply 方法）

game/entity/
└── HitEffectMgr.ts        ← 管理当前实体的命中效果列表，按 priority 排序执行

已实现的效果：
├── BaseDamageEffect       ← 基础伤害（ATK × ratio - DEF）
├── KnockbackEffect        ← 击退
├── BurnOnHitEffect        ← 灼烧 DOT
├── FrostOnHitEffect       ← 减速冰冻
└── ChainLightningEffect   ← 闪电链
```

### 添加新命中效果

1. 在 `game/skill/effects/` 或 `game/entity/hitEffects/` 新建效果类
2. 在 `buffConfig/hitEffects.json` 添加定义：

```json
{
    "your-effect": {
        "id": "your-effect",
        "effectClass": "YourHitEffect",
        "priority": 70,
        "yourParam": 0.5
    }
}
```

3. 在升级项或攻击载荷中引用 `"hitEffects": ["your-effect"]`

### 命中效果执行顺序

按 `priority` 降序执行：
- 100: 基础伤害
- 90: 击退
- 80: 闪电链
- 50: 灼烧 / 冰冻
- ...

## 四、技能系统

### 数据驱动流程

```
skills.json 定义 SkillDef
    → SkillFactory.create(def) → IActiveSkill 实例
    → SkillSystem.equip(skill)
    → 按键 tryUseBySlot(i, ctx) → CD 检查 → skill.activate(ctx)
```

### skills.json 格式

```json
{
    "fireball": {
        "id": "fireball",
        "name": "火球术",
        "skillClass": "SpawnProjectileSkill",
        "cooldown": 5,
        "payloadRef": "payload.burn",
        "tags": ["fire", "magic", "projectile"],
        "projectileCount": 1,
        "projectileSpeed": 400,
        "damageRatio": 1.2
    }
}
```

### 已有技能原语（skillClass）

| skillClass | 作用 | 关键参数 |
|-----------|------|---------|
| `SpawnProjectileSkill` | 发射弹道 | projectileCount / projectileSpeed / payloadRef |
| `AreaDamageSkill` | 范围 AOE | radius / payloadRef |
| `ApplySelfBuffSkill` | 给自己加 Buff | buffRef |
| `DashAttackSkill` | 冲刺攻击 | dashDistance / payloadRef |
| `SummonSkill` | 召唤物 | summonId / duration |
| `ArrowStormSkill` | 箭雨（硬编码） | 独立实现 |
| `DashShotSkill` | 冲刺射击（硬编码） | 独立实现 |

### 添加新技能

**方式 A：用现有原语（只改 JSON）**

在 `skills.json` 加一条，`skillClass` 用已有原语名：

```json
{
    "ice_nova": {
        "id": "ice_nova",
        "name": "冰霜新星",
        "skillClass": "AreaDamageSkill",
        "cooldown": 8,
        "payloadRef": "payload.frost",
        "radius": 200,
        "tags": ["ice", "magic", "aoe"]
    }
}
```

**方式 B：自定义技能类**

1. 实现 `IActiveSkill`
2. `SkillFactory.register('YourSkill', YourCtor)`
3. JSON 中 `"skillClass": "YourSkill"`

## 五、攻击载荷（Payload）

载荷 = "命中后发生什么"的完整描述，被技能和普攻引用。

```json
{
    "payload.burn": {
        "id": "payload.burn",
        "damageRatio": 1.2,
        "hitEffects": ["base-damage"],
        "targetBuffs": ["buff.burn_dot"],
        "tags": ["fire", "magic"]
    }
}
```

| 字段 | 作用 |
|------|------|
| `damageRatio` | 伤害系数（× 攻击力） |
| `hitEffects` | 触发的命中效果 ID 列表 |
| `targetBuffs` | 给目标施加的 Buff |
| `attackerBuffs` | 给攻击者施加的 Buff（如吸血） |
| `tags` | 元素/类型标签 |

## 六、升级系统（肉鸽三选一）

### upgrades.json 格式

```json
{
    "id": "fire-arrow",
    "name": "烈焰箭",
    "desc": "命中施加灼烧DOT",
    "tier": 1,
    "rarity": "rare",
    "category": "onhit",
    "effects": [
        {
            "type": "hit_effect",
            "data": {
                "id": "upg-burn",
                "effectClass": "BurnOnHitEffect",
                "priority": 50,
                "burnRatio": 0.2,
                "burnDuration": 3
            }
        }
    ]
}
```

### effects 类型

| type | 作用 | data 要求 |
|------|------|----------|
| `"buff"` | 给玩家加永久 Buff | 完整 BuffData |
| `"hit_effect"` | 注册命中效果 | 完整 HitEffectData |

### 添加新升级项

只需在 `upgrades.json` 追加一个条目。category 建议：
- `attr` — 属性类（攻击/攻速/暴击/移速）
- `proj` — 弹道类（多重/穿透/弹射/追踪）
- `onhit` — 命中类（灼烧/冰冻/闪电链/击退）

## 七、属性系统

### 核心概念

```
最终值 = base + Σ(Value修饰器) × (1 + Σ(Mul修饰器))

修饰器来源：
  1. Buff（通过 AttributeChangeResolver 自动挂/卸）
  2. 装备（未实现）
  3. Meta 永久升级（未实现）
```

### 属性列表（EPropertyId）

| 属性 | 说明 | 默认基底 |
|------|------|---------|
| Hp | 生命值 | 1000 |
| Attack | 攻击力 | 100 |
| Defense | 防御力 | 50 |
| CritRate | 暴击率 | 0.05 |
| CritDmg | 暴击伤害倍率 | 1.5 |
| AttackSpeed | 攻击速度 | 1.0 |
| MoveSpeed | 移动速度 | 100 |
| LifestealRate | 吸血率 | 0.2 |
| ExtraProjectiles | 额外弹道数 | 0 |
| PierceCount | 穿透数 | 0 |
| BounceCount | 弹射数 | 0 |
| SpreadAngle | 扇形角度 | 15 |
| HomingStrength | 追踪强度 | 0 |
| ArrowSpeed | 箭矢速度 | 300 |

## 八、伤害公式

### 玩家 → 敌人

```
PlayerCombat.attack(target):
    rawDmg = Attack属性值
    暴击? → rawDmg × CritDmg
    finalDmg = max(1, rawDmg - target.Defense × 0.5)
    target.takeDamage(finalDmg)
    吸血 → heal(finalDmg × LifestealRate)
```

### 敌人 → 玩家

```
EnemyCombat → PlayerCombat.takeDamage(rawDmg):
    finalDmg = max(1, rawDmg - Defense × 0.5)
    hp -= finalDmg
    无敌帧 0.5s
```

## 十、Loader 模式（所有配置统一范式）

每个配置子目录遵循相同模式：

```
someConfig/
├── data.json          ← 纯数据，策划可直接编辑
└── SomeConfigLoader.ts ← 加载 + 缓存 + 提供查询 API
```

Loader 规范：
1. `import raw from './data.json'` 静态导入
2. 转为 `Map` 或 `Record` 做索引
3. 导出查询函数：`getDef(id)` / `allIds()` / `getByTag()` 等
4. 统一在 `game/config/index.ts` re-export

**添加新配置域：** 建目录 → 写 JSON → 写 Loader → 在 `index.ts` 加一行 export。
