## P0：没有就不是完整循环

### 1. 三选一升级 UI ❌

**需求**：杀怪升级后弹出 3 个随机升级选项，玩家选 1 个。

| 项 | 说明 |
|----|------|
| 触发 | `PlayerExperience.onLevelUp` 回调 |
| 数据源 | `UpgradeManager` + `ALL_UPGRADES` JSON |
| 池过滤 | 按 tier/archetype 过滤，排除已满 stack |
| UI | 全屏弹窗，3 卡片横排，显示名称/描述/稀有度颜色 |
| 暂停 | 弹窗期间 `GameSession` 切 Paused |
| 依赖 | `GameSession`✅ + `UpgradeManager`✅ + `PlayerExperience`✅ |

**文件规划**：`game/ui/UpgradePickPanel.ts`

---

### 2. GameStateMachine ❌

**需求**：完整游戏状态流转 Boot → Home → Playing → Reward → GameOver

| 项 | 说明 |
|----|------|
| 设计文档 | `阶段3-游戏整体/01-核心游戏循环.md` 已有完整设计 |
| 当前状态 | `GameLoop` 启动即 Playing，无法停止 |
| `GameSession` | 已实现 Playing/Paused/AwaitRevive/GameOver 子阶段 |
| 还差什么 | Home 界面、Loading 阶段、Transition 阶段、全局状态机壳 |

**文件规划**：`game/core/GameStateMachine.ts`

## P1：没有体验极差

### 4. 技能热键绑定 ❌

**需求**：1/2/3 键触发已装备的主动技能

| 项 | 说明 |
|----|------|
| 当前状态 | `SkillSystem.tryUse()` 只在 `BattleTestPanel` 手动调用 |
| 实现方式 | `ActionComp` 新增 `EAction.Skill1/2/3`，`PlayerControl.update` 监听并调用 `skillSystem.tryUse(slotIndex, ctx)` |
| `RawInputSystem` | 监听 `KeyCode.DIGIT_1/2/3` |

**改动范围**：`component/ActionComp.ts` + `system/ActionMapSystem.ts` + `PlayerControl.ts`

---

### 5. 闪避/Dash ❌

**需求**：空格键闪避，短距位移 + 0.3s 无敌帧

| 项 | 说明 |
|----|------|
| 当前状态 | `EAction.Dodge` 已映射空格，**无系统消费** |
| 实现方式 | `PlayerDodgeState`：读 `ActionComp` 移动方向，按方向位移 N 像素，期间 `invincibleTimer` 置高 |
| 冷却 | 1~2 秒，可被 Buff 修改 |

**文件规划**：`states/PlayerDodgeState.ts` + `EPlayerState.Dodge`

---

### 6. DashShotSkill 实际位移 ❌

**需求**：闪身射击应有真实位移 + 无敌帧

| 项 | 说明 |
|----|------|
| 当前状态 | `DashShotSkill` 仅 buff 攻速 + AutoShoot，**无位移** |
| 实现方式 | `execute()` 中获取面向方向，`playerNode.setPosition` 瞬移 + 无敌帧 0.3s + 连射 |

**改动范围**：`skill/DashShotSkill.ts`

### 8. 追踪弹 (HomingStrength) ❌

**改动范围**：`projectile/ArrowProjectile.ts`
