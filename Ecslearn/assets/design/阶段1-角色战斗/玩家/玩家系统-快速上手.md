## 二、核心架构：PlayerControl 怎么组装一个玩家

```
PlayerControl.onLoad()
    │
    ├─ 1. PlayerProperty        ← 从 player.json 加载属性基底
    ├─ 2. PlayerCombat          ← 注入 property，提供 takeDamage/heal
    ├─ 3. PlayerExperience      ← 注入 playerConfig，提供 addXp/levelUp
    ├─ 4. PlayerBehaviorFactory.create(behaviorId)
    │      └─ 根据 typeId 返回对应职业 Behavior 实例
    ├─ 5. PlayerRuntime         ← 组装运行时服务包
    │      ├─ EntityBuffMgr
    │      ├─ HitEffectMgr
    │      ├─ UpgradeManager
    │      └─ SkillSystem
    ├─ 6. StateMachine          ← 注册 Idle/Run/Attack/Hurt/Dead 状态
    │      └─ Attack 状态来自 behavior.createAttackState()
    └─ 7. Entity (ECS)          ← 挂载 Input/Action/Velocity/NodeRef 组件
```

**关键点：`_behaviorId` 当前写死为 `'archer'`。** 换职业需要改这一行（或改成编辑器属性/存档读取）。

## 三、职业抽象：PlayerBehavior

每个职业是一个 `PlayerBehavior` 子类，通过 `@playerBehavior` 装饰器自动注册到 `PlayerBehaviorFactory`。

```typescript
// 基类要求实现的核心方法：
abstract class PlayerBehavior {
    abstract readonly typeId: string;           // 工厂 key
    abstract wantAttack(d: IAttackDecision): boolean;  // 是否发起攻击
    abstract createAttackState(): IState<PlayerCtx>;   // 攻击 FSM 状态
    abstract buildSkillContext(src: ISkillContextSource): SkillContext;

    // 可选覆写
    setServices?(services: PlayerServices): void;
    onBehaviorCommand?(cmd: string, data: unknown): void;
}
```

**弓箭手示例（已实现）：**

```
ArcherBehavior
├── wantAttack: 有目标 + 不在移动 → true
├── createAttackState: 返回 ArcherAttackState（攻速缩放动画 + 射箭）
├── buildSkillContext: 注入 behavior + services
├── setServices: 保存 PlayerServices 引用
└── onBehaviorCommand:
    ├── 'set_shoot_policy' → 切换射击策略
    └── 'execute_attack'   → （待实现）
```

## 四、如何新增一个职业（完整步骤）

### Step 1：创建行为类

在 `game/player/<job_name>/` 下新建 `<JobName>Behavior.ts`：

```typescript
import { playerBehavior } from '../../../baseSystem/player';
import { PlayerBehavior } from '../base/PlayerBehavior';
import type { IAttackDecision, ISkillContextSource } from '../base/types';
import type { SkillContext } from '../../skill/SkillTypes';
import type { IState } from '../../../baseSystem/fsm';
import type { PlayerCtx } from '../states/PlayerContext';
import type { PlayerServices } from '../runtime/PlayerServices';

@playerBehavior
export class MageBehavior extends PlayerBehavior {
    readonly typeId = 'mage';

    private _services: PlayerServices | null = null;

    setServices(s: PlayerServices): void {
        this._services = s;
    }

    wantAttack(d: IAttackDecision): boolean {
        // 你的攻击判断逻辑
        return d.hasTarget && !d.isMoving;
    }

    createAttackState(): IState<PlayerCtx> {
        // 返回你的攻击状态实例
        return new MageAttackState();
    }

    buildSkillContext(src: ISkillContextSource): SkillContext {
        return {
            ...src,
            behavior: this,
            services: this._services!,
        };
    }

    onBehaviorCommand(cmd: string, data: unknown): void {
        // 处理升级/技能发来的指令
    }
}
```

### Step 2：创建攻击状态

在同目录下新建 `MageAttackState.ts`，实现 `IState<PlayerCtx>`：

```typescript
import type { IState } from '../../../baseSystem/fsm';
import type { PlayerCtx } from '../states/PlayerContext';
import { EPlayerState } from '../states/PlayerContext';

export class MageAttackState implements IState<PlayerCtx> {
    enter(ctx: PlayerCtx): void {
        // 播攻击动画、开始施法
    }
    update(ctx: PlayerCtx, dt: number): void {
        // 施法计时、发射弹道/AOE
        // 完成后 ctx.fsm.changeState(EPlayerState.Idle)
    }
    exit(ctx: PlayerCtx): void {
        // 清理
    }
}
```

### Step 3：注册职业

在 `game/player/behaviors.ts` 增加一行：

```typescript
import './mage/MageBehavior';     // ← 加这一行
```

### Step 4：（可选）职业专属配置

在 `game/player/mage/mageConfig.ts` 写职业常量：

```typescript
export const mageConfig = {
    castRange: 500,
    spellSpeed: 200,
    // ...
};
```

### Step 5：切换职业

在 `PlayerControl.ts` 中把 `_behaviorId` 改为 `'mage'`：

```typescript
private _behaviorId = 'mage';  // 原来是 'archer'
```

**完成。** 核心代码零改动，只需：1 个 Behavior + 1 个 AttackState + 1 行 import + 改 behaviorId。

## 五、运行时数据流

```
每帧 PlayerControl.update(dt)
    │
    ├─ PlayerRuntime.tick(dt)
    │   ├─ EntityBuffMgr.update(dt)     ← Buff 计时/移除
    │   ├─ SkillSystem.tick(dt)         ← 技能 CD
    │   └─ FlashWhite.update(dt)        ← 受击闪白
    │
    ├─ StateMachine.update(dt)          ← 当前状态的 update
    │   └─ 如 ArcherAttackState.update → ShootResolver → ArrowProjectile
    │
    └─ GameSession.tick(dt)             ← （如有）全局会话逻辑
```

### 伤害流

```
ArrowProjectile 命中敌人
    → HitEffectMgr.applyAll(attackerProps, target, baseDmg)
        → 各 HitEffect 依 priority 排序执行
        → KnockbackEffect / BurnOnHitEffect / FrostOnHitEffect / ...
    → EnemyCombat.takeDamage(finalDmg)
    → 暴击判定 → DamagePopup
    → 吸血 → PlayerCombat.heal
```

## 六、属性系统要点

```
player.json 定义属性基底：
{
    "Hp":          { "base": 1000, "min": 0 },
    "Attack":      { "base": 100 },
    "CritRate":    { "base": 0.05, "min": 0, "max": 1.0 },
    ...
}

运行时属性 = base + Σ(modifier)

Buff 通过 AttributeChangeResolver 往 PropertyMgr 上挂/卸修饰器：
  - Value-Buff: 加固定值（如 Attack +30）
  - Mul-Buff:   乘百分比（如 AttackSpeed × 1.2）

属性 ID 枚举：game/config/enum/propertyEnum.ts
属性曲线 JSON：game/config/propertyConfig/*.json
```

## 七、技能系统要点

```
skills.json 定义技能：
{
    "fireball": {
        "id": "fireball",
        "skillClass": "SpawnProjectileSkill",
        "cooldown": 5,
        "payloadRef": "payload.burn",
        ...
    }
}

SkillFactory.create(def) → IActiveSkill 实例
SkillSystem.equip(skill) → 装入槽位
SkillSystem.tryUseBySlot(i, ctx) → CD 检查 → skill.activate(ctx)

技能原语（game/skill/primitives/）：
  SpawnProjectileSkill / AreaDamageSkill / ApplySelfBuffSkill /
  DashAttackSkill / SummonSkill

自定义技能：实现 IActiveSkill + SkillFactory.register('YourClass', YourCtor)
```

## 八、升级系统要点

```
upgrades.json 定义升级项（肉鸽三选一）：
[
    {
        "id": "rapid-fire",
        "name": "急速连射",
        "tier": 1, "rarity": "common",
        "effects": [{ "type": "buff", "data": { ... } }]
    },
    ...
]

UpgradeManager.apply(upgrade)
  → 遍历 effects
  → type="buff" → EntityBuffMgr.addBuff
  → type="hit_effect" → HitEffectMgr.register
```

## 九、当前已知限制

| 问题 | 说明 |
|------|------|
| `_behaviorId` 写死 | PlayerControl 硬编码 `'archer'`，换职业需改代码 |
| 战士/召唤师未实现 | `wantAttack` / `createAttackState` 为 warn 占位 |
| `PlayerShootState` 遗留 | 与 `ArcherAttackState` 重复，历史兼容 |
| `execute_attack` 未落地 | 技能原语发出的攻击指令在弓箭手行为中未处理 |
| `playerConfig.anims` 偏弓箭手 | 动画路径写死 Archer 资源，多职业需改为按 typeId 分表 |
