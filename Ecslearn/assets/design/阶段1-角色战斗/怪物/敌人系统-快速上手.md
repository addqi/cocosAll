## 二、核心架构：MinionControl 怎么组装一个小怪

```
MinionControl.onLoad()
    │
    ├─ 1. EnemyBehaviorFactory.create(behaviorId)
    │      └─ 根据 EMinionType 返回对应 MinionBehavior 实例
    │      └─ Behavior 持有合并后的 config + propertyCfg
    │
    ├─ 2. super.onLoad()  [EnemyBase]
    │      ├─ 创建 Body 节点（Sprite + EnemyAnimation）
    │      ├─ 创建 GroundFX / UIAnchor 节点
    │      ├─ EnemyProperty(propertyCfg)
    │      ├─ EnemyCombat(property)
    │      ├─ EnemyBuffOwner(property, combat)
    │      ├─ EntityBuffMgr
    │      ├─ EnemyVisual（血条/探测圈/指示器/溶解）
    │      ├─ EnemyMovement
    │      ├─ 物理 RigidBody2D + CircleCollider2D
    │      └─ 注册到 EnemyBase.allEnemies 静态列表
    │
    └─ 3. _initFsm()
           └─ StateMachine<EMobState, IMinionCtx>
               注册：Idle/Wander/Chase/WindUp/Attack/Recovery/Dead
```

## 三、Behavior 模式：三种小怪对比

所有小怪共享 `MinionBehavior` 基类的 FSM + 钩子，差异通过**覆写钩子**实现：

| 钩子方法 | 战士 Warrior | 自爆兵 Bomber | 射手 Ranger |
|----------|-------------|--------------|-------------|
| `drawIndicator` | 扇形（距离+角度） | 圆形（距离） | 自定义光束节点 |
| `checkHit` | 距离 + facingAngle 扇形 | 距离（360°圆） | 不用（射箭） |
| `onAttackFrame` | 基类默认：`applyDamage` | 基类默认 + `takePureDamage(self)` | 覆写：`_fireArrow` → EnemyArrow |
| `afterAttackState` | Recovery → Idle | Dead（自爆后直接死） | Recovery → Idle |
| `indicatorNeedsRotation` | `true`（跟随朝向） | `false` | N/A（自定义） |

### 数据来源

每个 Behavior 在构造时做一件事：

```typescript
const _data = getEnemyData('warrior');  // 从 enemies.json 读取

class WarriorBehavior extends MinionBehavior {
    readonly typeId = 'warrior';
    config = { ...enemyConfig, ..._data.overrides } as EnemyConfigData;
    propertyCfg = _data.properties;
}
```

`enemyConfig` 是全局默认值，`_data.overrides` 是该种类的差异覆盖。

## 四、FSM 状态流转

```
         ┌──────────────────────────────────────────┐
         │                                          │
    ┌────▼────┐  发现玩家   ┌─────────┐  进入攻击距离  ┌──────────┐
    │  Idle   │──────────→│  Chase  │──────────────→│ WindUp   │
    └────┬────┘            └────┬────┘               │ (前摇)   │
         │                      │                    └────┬─────┘
    超时 │                 丢失目标                        │ 前摇结束
         ▼                      │                         ▼
    ┌─────────┐                 │                    ┌──────────┐
    │ Wander  │◄────────────────┘                    │ Attack   │
    └─────────┘                                      │ (动画)   │
                                                     └────┬─────┘
                                                          │ 动画结束
                                                          ▼
                                                     ┌──────────┐
                                                     │ Recovery │
                                                     │ (冷却)   │
                                                     └────┬─────┘
                                                          │
                                                     回到 Idle
                                                          
    任意状态 → hp <= 0 → Dead（溶解/经验/销毁）
```

**WindUp 细节：**
- 创建攻击指示器（`behavior.createIndicator`）
- 每帧更新指示器朝向（`behavior.tickIndicator`）
- 面向玩家
- 计时 `attackWindUp` 秒后进入 Attack

**Attack 细节：**
- 播放 attack 动画
- 在第 `attackHitFrame` 帧调用 `behavior.onAttackFrame`
- 近战命中用 `behavior.checkHit` 判定
- 远程（Ranger）直接发射箭矢
- 动画结束调 `behavior.afterAttackState` 决定下一状态

## 五、如何新增一种小怪（完整步骤）

### Step 1：在 enemies.json 添加数值

文件位置：`game/config/enemyConfig/enemies.json`

```json
{
    "id": "lancer",
    "name": "枪兵",
    "category": "minion",
    "properties": {
        "Hp":        { "base": 1500, "min": 0 },
        "Attack":    { "base": 70 },
        "Defense":   { "base": 20 },
        "MoveSpeed": { "base": 130 }
    },
    "overrides": {
        "detectionRange": 250,
        "attackRange": 70,
        "attackAngle": 60,
        "attackWindUp": 0.3,
        "attackCooldown": 1.2,
        "attackHitFrame": 1,
        "xpReward": 8
    }
}
```

### Step 2：创建 Behavior 类

新建 `game/enemy/minion/lancer/LancerBehavior.ts`：

```typescript
import { enemyBehavior } from '../../../../baseSystem/enemy';
import { MinionBehavior } from '../MinionBehavior';
import { getEnemyData } from '../../../../config/enemyConfig';
import { enemyConfig } from '../../config/enemyConfig';
import type { EnemyConfigData } from '../../config/enemyConfig';

const _data = getEnemyData('lancer');

@enemyBehavior
export class LancerBehavior extends MinionBehavior {
    readonly typeId = 'lancer';
    config = { ...enemyConfig, ..._data.overrides } as EnemyConfigData;
    propertyCfg = _data.properties;

    // 默认近战扇形已够用，无需覆写
    // 若需要冲刺攻击，覆写 onAttackFrame / checkHit
}
```

### Step 3：注册

在 `game/enemy/minion/behaviors.ts` 中：

```typescript
// 1. 枚举加一项
export enum EMinionType {
    Warrior = 'warrior',
    Bomber  = 'bomber',
    Ranger  = 'ranger',
    Lancer  = 'lancer',     // ← 加这一行
}

// 2. import 加一行
import './lancer/LancerBehavior';   // ← 加这一行
```

**完成。** 新怪物会自动获得完整的 FSM、血条、探测圈、攻击指示器、死亡溶解、经验掉落。

### Step 4：（可选）特殊行为

如果想做冲刺攻击（非标准近战），覆写钩子：

```typescript
@enemyBehavior
export class LancerBehavior extends MinionBehavior {
    // ...

    // 覆写指示器：画一条直线
    drawIndicator(g: Graphics, range: number): void {
        g.moveTo(0, 0);
        g.lineTo(range, 0);
        g.stroke();
    }

    indicatorNeedsRotation = true;

    // 覆写命中判定：矩形区域
    checkHit(ctx: IMinionCtx): boolean {
        // 自定义矩形判定
    }

    // 覆写攻击帧：冲刺 + 命中
    onAttackFrame(ctx: IMinionCtx): void {
        // 瞬间位移到目标方向 + 近战伤害
    }
}
```

## 六、配置系统要点

### enemyConfig.ts（默认值）

所有小怪的默认参数。Behavior 用 `{ ...enemyConfig, ..._data.overrides }` 合并。

```typescript
export const enemyConfig: EnemyConfigData = {
    frameSize: 192,
    displayWidth: 150,
    displayHeight: 150,
    anims: {
        idle:   { frameDir: 'Warrior/idle',    fps: 8,  loop: true },
        run:    { frameDir: 'Warrior/run',     fps: 10, loop: true },
        attack: { frameDir: 'Warrior/attack',  fps: 12, loop: false },
    },
    detectionRange: 200,
    attackRange: 50,
    attackAngle: 100,
    attackWindUp: 0.6,
    attackCooldown: 1.5,
    attackHitFrame: 2,
    idleTimeMin: 1,
    idleTimeMax: 3,
    wanderTimeMin: 1,
    wanderTimeMax: 2,
    wanderSpeedRatio: 0.4,
    xpReward: 10,
};
```

### enemies.json（差异覆盖）

只需写**与默认值不同的部分**：

```json
{
    "id": "bomber",
    "overrides": {
        "attackRange": 80,
        "attackAngle": 360,
        "attackWindUp": 0,
        "attackCooldown": 0,
        "xpReward": 15
    }
}
```

### 动画资源

当前三种小怪**共用 Warrior 动画路径**。独立动画需要：
1. 准备帧序列到 `res/` 对应目录
2. 在 `overrides` 或 `enemyConfig` 中设置不同的 `anims.frameDir`

## 七、EnemyBase 生命周期

```
场景实例化 MinionControl 节点
    → onLoad: 创建 Behavior → 初始化基类 → 建 FSM
    → update(dt):
        ├─ EntityBuffMgr.update(dt)    ← Buff 计时
        ├─ fsm.update(dt)             ← 状态逻辑
        ├─ EnemyVisual.updateHP()     ← 血条
        └─ 死亡检测 → changeState(Dead)
    → onDestroy: 从 allEnemies 移除
```

---

## 八、伤害接收流

```
玩家弹道命中
    → EnemyCombat.takeDamage(rawDmg)
        → 防御减免: finalDmg = max(1, rawDmg - defense * 0.5)
        → hp -= finalDmg
        → 触发受击闪白
    → hp <= 0
        → MinionControl 检测到 → changeState(Dead)
        → MinionDeadState:
            ├─ 隐藏血条/地面 FX
            ├─ 播 die 动画（若有）或立即溶解
            ├─ PlayerControl.grantXp(xpReward)
            └─ node.destroy()
```

## 九、全局查询：findNearestEnemy

```typescript
// EnemyQuery.ts
export function findNearestEnemy(
    from: Vec2,
    range?: number,
    exclude?: EnemyBase
): EnemyBase | null
```

遍历 `EnemyBase.allEnemies`，返回存活且最近的。玩家射击系统用这个函数锁定目标。

## 十、关键文件速查表

| 你想做什么 | 改哪个文件 |
|-----------|-----------|
| 加新怪物种类（小怪） | `config/enemyConfig/enemies.json` + `minion/<name>/Behavior.ts` + `behaviors.ts` |
| 调某种怪的数值 | `config/enemyConfig/enemies.json` 的 `overrides` |
| 调全局默认参数 | `enemy/config/enemyConfig.ts` |
| 改攻击判定方式 | 覆写 `MinionBehavior.checkHit` / `onAttackFrame` |
| 改死亡表现（溶解/动画） | `base/EnemyVisual.ts` + `states/MinionDeadState.ts` |
| 改血条样式 | `base/EnemyVisual.ts` |
| 加精英/Boss | `elite/` 或 `boss/` 下新建管线（参考 `minion/` 模式） |
| 改掉落经验 | `enemies.json` 的 `overrides.xpReward` |
| 改探测圈/资源常量 | `config/enemyResConfig.ts` |
