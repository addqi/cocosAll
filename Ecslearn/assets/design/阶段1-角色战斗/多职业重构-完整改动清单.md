# 多职业重构 —— 完整改动清单

> 目标：将当前单职业弓箭手架构改造为可扩展多职业（弓箭手、战士、召唤师）架构。
> 改造完成后新增一个职业 = **写 1 个 Behavior + 1 个 Config + 1 行 import**，核心代码 0 改动。
>
> 当前约束：
> - 运行时**只读取 JSON 配置**
> - Excel 仅作为策划源文件，后续通过自动化脚本同步到 JSON
> - 现阶段所有重构都以“程序可直接改 JSON 快速测试”为准
>
> 前置阅读：
> - [玩家-职业与流派指引](./玩家/职业与流派-架构指引.md)
> - [项目分步重构路线图](./重构项目-分步路线图.md)
> - [Buff 与升级系统重构方案](./重构专项-Buff与升级系统.md)
>
> 推荐执行顺序（按步骤实施）：

## 三、现有文件改动清单

### 3.1 player/PlayerControl.ts（核心改造）

**改动要点**：
- 新增 `@property behaviorId: EPlayerClass` 编辑器属性
- `start()` 中用 `PlayerBehaviorFactory.create(this.behaviorId)` 获取 behavior
- `_shootPolicy` 移入 `ArcherBehavior`，PlayerControl 不再持有
- `lateUpdate` 中 `wantShoot` → `this._behavior.wantAttack(act, hasTarget, isMoving)`
- `EPlayerState.Shoot` → `EPlayerState.Attack`
- `_initFsm` 中 `this._fsm.addState(EPlayerState.Attack, this._behavior.createAttackState())`
- 配置读取从 `playerConfig` 改为 `this._behavior.config`
- `PlayerControl.instance` 单例接口、`applyDamage`、`grantXp`、`revive` 等外部接口 **完全不变**

**影响的外部依赖**（无需修改）：
- `MinionBehavior.onAttackFrame` → `PlayerControl.instance.applyDamage` ✅
- `GameSession` → `PlayerControl.instance` ✅
- `EnemyQuery` → `this.node.worldPosition` ✅

### 9.2 hitEffects/ — "施加Buff类"效果合并为数据驱动

#### 9.2.1 问题识别

当前 `BurnOnHitEffect` 和 `FrostOnHitEffect` 结构 100% 相同：

```
读 this.data 的配置字段 → 构造 BuffData → ctx.targetBuffMgr.addBuff()
```

唯一区别是字段名不同（burnRatio vs frostSlowPerStack）。
每新增一个 DOT/debuff 类命中效果（中毒、流血、眩晕……）都要写一个几乎一样的 TS 文件。

#### 9.2.2 方案：ApplyBuffOnHitEffect（通用"施加Buff"命中效果）

合并为一个数据驱动类，所有"命中时给目标上 buff"的效果纯 JSON 配置：

```typescript
// hitEffects/ApplyBuffOnHitEffect.ts
@hitEffect
export class ApplyBuffOnHitEffect extends HitEffectBase {
    onHit(ctx: GameHitContext): void {
        const cfg = this.data;
        const buff: BuffData = { ...cfg.buff };

        if (cfg.scaleWithBaseDamage != null) {
            const ticksPerSec = 1 / (buff.tickInterval || 1);
            buff.damagePerStack = Math.max(1,
                Math.round(ctx.baseDamage * cfg.scaleWithBaseDamage / ticksPerSec));
        }

        ctx.targetBuffMgr.addBuff(buff, ctx.targetBuffOwner);
    }
}
```

JSON 配置示例：

```json

    "effectClass": "ApplyBuffOnHitEffect",
    "scaleWithBaseDamage": 0.2,

        "duration": 5,
        "maxStack": 99,
        "tickInterval": 0.1,

    }
}

// 冻伤 — 以前需要 FrostOnHitEffect.ts
{
    "id": "frost-on-hit",
    "effectClass": "ApplyBuffOnHitEffect",
    "buff": {

        "name": "冻伤(-15%速)",
        "duration": 3,
        "maxStack": 5,
        "effectClass": "FrostSlowEffect",
        "slowPerStack": 0.15
    }
}

// 中毒（新效果，零代码）
{
    "id": "poison-on-hit",
    "effectClass": "ApplyBuffOnHitEffect",
    "scaleWithBaseDamage": 0.1,
    "buff": {

        "name": "中毒(10%)",
        "duration": 8,

    }

```
