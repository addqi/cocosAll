# 02 - 角色与战斗系统

> 前置阅读：[01-核心游戏循环](../阶段3-游戏整体/01-核心游戏循环.md) | [项目总览](../项目概述/00-项目总览.md)

## 本章目标

让玩家角色**能移动、能射箭、能打中敌人、敌人能掉血能死**。这是游戏「好不好玩」的核心 —— 手感从这里来。

### 实现状态：⏳ 核心战斗完成，AI 与死亡循环未做

| 子模块 | 状态 | 说明 |
|--------|------|------|
| 玩家移动 | ✅ | OOP/ECS 混合：Proxy Entity + FSM(Idle/Run/Shoot) |
| 玩家射箭 | ✅ | ArrowProjectile + ProjectilePool，鼠标/J 键触发 |
| 敌人显示 | ✅ | EnemyControl + EnemyAnimation + Tiny Swords 素材 |
| 弹道碰撞 | ✅ | ArrowProjectile.update 距离检测 → EnemyCombat.takeDamage |
| 伤害公式 | ✅ | V2 级：攻-防 + 暴击 + 百分比减伤 + 吸血回血 |
| 敌人 AI | ❌ | 无 AISystem / 敌人行为状态机 |
| 死亡与重生 | ❌ | 无 GameOver 逻辑 / 传送门 |

## 1. 实体设计：ECS 视角

所有游戏内可交互对象都是 Entity，通过挂载不同 Component 获得能力：

```
┌─────────────────────────────────────────────┐
│                   Entity                     │
├──────────┬──────────┬──────────┬────────────┤
│ Transform│  Health  │ Combat   │   AI       │
│ Component│ Component│ Component│ Component  │
├──────────┼──────────┼──────────┼────────────┤
│ 位置/朝向 │ HP/MaxHP │ 攻/防/速  │ 行为树/状态 │
└──────────┴──────────┴──────────┴────────────┘
```

### 玩家 Entity 组成

| Component | 数据 | 说明 |
|-----------|------|------|
| TransformComp | `position`, `rotation` | 位置与朝向 |
| InputComp | `moveDir`, `isMoving` | 来自摇杆的输入 |
| VelocityComp | `speed`, `velocity` | 移动速度 |
| HealthComp | `hp`, `maxHp` | 生命值 |
| CombatComp | `atk`, `def`, `atkSpeed`, `atkRange`, `critRate`, `critDmg` | 战斗属性 |
| AnimComp | `currentAnim`, `spriteNode` | 动画状态引用 |
| PlayerTag | (无数据，纯标记) | 标识为玩家 |

### 敌人 Entity 组成

| Component | 数据 | 说明 |
|-----------|------|------|
| TransformComp | 同上 | |
| VelocityComp | 同上 | |
| HealthComp | 同上 | |
| CombatComp | 同上 | |
| AnimComp | 同上 | |
| AIComp | `aiType`, `targetEntity`, `state` | AI 行为控制 |
| EnemyTag | `enemyType`, `rewardGold` | 敌人类型与掉落 |

### 弹道 Entity 组成

| Component | 数据 | 说明 |
|-----------|------|------|
| TransformComp | 同上 | |
| VelocityComp | `speed`, `direction` | 飞行方向和速度 |
| ProjectileComp | `damage`, `owner`, `lifeTime`, `piercing` | 弹道属性 |

## 2. 移动系统

### 玩家移动

> 参考：幽灵射手用 Joystick → CharacterRigid → RigidBody
> 我们：Joystick → InputComp → MovementSystem → 直接设置 position

**为什么不用物理引擎？** 这类游戏不需要真实物理模拟。碰撞检测用简单的距离判断就够了，省掉物理引擎的复杂性和性能开销。

```typescript
class MovementSystem implements GameSystem {
    update(dt: number): void {
        // 遍历所有有 VelocityComp + TransformComp 的实体
        for (const entity of EntityManager.query(VelocityComp, TransformComp)) {
            const vel = entity.get(VelocityComp);
            const tf = entity.get(TransformComp);

            if (vel.velocity.lengthSqr() < 0.001) continue;

            tf.position.add(vel.velocity.clone().multiplyScalar(dt));

            // 朝向跟随移动方向（敌人和玩家都适用）
            if (vel.velocity.x !== 0) {
                tf.facingRight = vel.velocity.x > 0;
            }
        }
    }
}
```

### InputSystem：摇杆 → 组件数据

```typescript
class InputSystem implements GameSystem {
    update(dt: number): void {
        const player = EntityManager.queryOne(PlayerTag);
        if (!player) return;

        const input = player.get(InputComp);
        const vel = player.get(VelocityComp);
        const combat = player.get(CombatComp);

        if (input.isMoving) {
            vel.velocity = input.moveDir.clone().multiplyScalar(vel.speed);
        } else {
            vel.velocity = Vec2.ZERO.clone();

            // 停下时自动锁定最近敌人（Archero 核心机制）
            const nearestEnemy = this._findNearestEnemy(player);
            if (nearestEnemy) {
                combat.autoTarget = nearestEnemy;
            }
        }
    }

    private _findNearestEnemy(player: Entity): Entity | null {
        const playerPos = player.get(TransformComp).position;
        let nearest: Entity | null = null;
        let minDist = Infinity;

        for (const enemy of EntityManager.query(EnemyTag)) {
            const dist = Vec2.distance(playerPos, enemy.get(TransformComp).position);
            if (dist < minDist) {
                minDist = dist;
                nearest = enemy;
            }
        }
        return nearest;
    }
}
```

## 3. 战斗系统

### 核心循环：停下 → 锁定 → 攻击

这是 Archero 类游戏的核心手感：

```
移动中 ──→ 停止移动 ──→ 寻找最近敌人 ──→ 转向 ──→ 攻击动画 ──→ 发射弹道
                                                                    │
                                         攻速冷却 ←──────────────────┘
```

```typescript
class CombatSystem implements GameSystem {
    update(dt: number): void {
        for (const entity of EntityManager.query(CombatComp)) {
            const combat = entity.get(CombatComp);
            combat.atkCooldown -= dt;

            if (!combat.autoTarget) continue;
            if (combat.atkCooldown > 0) continue;

            // 目标已死？清除
            const targetHealth = combat.autoTarget.get(HealthComp);
            if (!targetHealth || targetHealth.hp <= 0) {
                combat.autoTarget = null;
                continue;
            }

            this._performAttack(entity, combat);
        }
    }

    private _performAttack(attacker: Entity, combat: CombatComp): void {
        combat.atkCooldown = 1.0 / combat.atkSpeed;

        // 播放攻击动画
        EventBus.emit('entity:attack', { entity: attacker });

        // 生成弹道
        this._spawnProjectile(attacker, combat.autoTarget, combat.atk);
    }

    private _spawnProjectile(
        attacker: Entity,
        target: Entity,
        damage: number
    ): void {
        const from = attacker.get(TransformComp).position;
        const to = target.get(TransformComp).position;
        const dir = to.clone().subtract(from).normalize();

        const projectile = EntityManager.create();
        projectile.add(new TransformComp(from.clone()));
        projectile.add(new VelocityComp(dir, 600)); // 箭速 600
        projectile.add(new ProjectileComp(damage, attacker, 3.0));
    }
}
```

### 弹道碰撞

```typescript
class ProjectileSystem implements GameSystem {
    update(dt: number): void {
        for (const entity of EntityManager.query(ProjectileComp, TransformComp)) {
            const proj = entity.get(ProjectileComp);
            const tf = entity.get(TransformComp);

            // 生命周期
            proj.lifeTime -= dt;
            if (proj.lifeTime <= 0) {
                EntityManager.destroy(entity);
                continue;
            }

            // 碰撞检测：简单圆形
            this._checkHit(entity, tf, proj);
        }
    }

    private _checkHit(
        projectile: Entity,
        tf: TransformComp,
        proj: ProjectileComp
    ): void {
        const hitRadius = 20; // 碰撞半径

        for (const target of EntityManager.query(HealthComp, TransformComp)) {
            // 不打自己人
            if (target === proj.owner) continue;
            // 玩家弹道只打敌人，敌人弹道只打玩家
            if (!this._isValidTarget(proj.owner, target)) continue;

            const dist = Vec2.distance(tf.position, target.get(TransformComp).position);
            if (dist < hitRadius) {
                this._applyDamage(target, proj.damage, proj.owner);
                if (!proj.piercing) {
                    EntityManager.destroy(projectile);
                }
                return;
            }
        }
    }

    private _isValidTarget(owner: Entity, target: Entity): boolean {
        const ownerIsPlayer = owner.has(PlayerTag);
        const targetIsEnemy = target.has(EnemyTag);
        return ownerIsPlayer === targetIsEnemy;
    }

    private _applyDamage(
        target: Entity,
        rawDamage: number,
        attacker: Entity
    ): void {
        const health = target.get(HealthComp);
        const combat = target.get(CombatComp);

        // 伤害公式（简单版，后续通过属性系统扩展）
        const def = combat?.def ?? 0;
        const finalDamage = Math.max(1, rawDamage - def);

        health.hp -= finalDamage;

        EventBus.emit('entity:damaged', {
            target,
            attacker,
            damage: finalDamage,
        });

        if (health.hp <= 0) {
            EventBus.emit('entity:death', { entity: target, killer: attacker });
        }
    }
}
```

## 4. 伤害公式

从简单开始，后续通过属性系统和 Buff 扩展：

### V1：最简版（先用这个）

```
最终伤害 = max(1, 攻击力 - 防御力)
```

### V2：带暴击

```
基础伤害 = max(1, 攻击力 - 防御力)
是否暴击 = random() < 暴击率
最终伤害 = 暴击 ? 基础伤害 × 暴击倍率 : 基础伤害
```

### V3：带属性修饰器（整合属性系统后）

```
实际攻击力 = PropertyManager.getFinalValue(attacker, 'atk')
             ↑ 这里面包含了 base + 所有 Modifier 的叠加
实际防御力 = PropertyManager.getFinalValue(target, 'def')
最终伤害 = max(1, 实际攻击力 - 实际防御力) × 暴击倍率
```

## 5. 敌人类型设计

用 Tiny Swords 素材包里的兵种做敌人：

| 类型 | 素材 | 行为 | 属性特点 |
|------|------|------|---------|
| **Pawn（小兵）** | Units/Pawn | 直线走向玩家，近战攻击 | 低攻低血，数量多 |
| **Archer（弓手）** | Units/Archer | 保持距离，远程射箭 | 中攻低血，会走位 |
| **Warrior（战士）** | Units/Warrior | 冲向玩家，近战高伤 | 高攻高血，速度中等 |
| **Lancer（枪兵）** | Units/Lancer | 冲刺攻击，有前摇 | 高攻中血，冲刺速度快 |
| **Monk（僧侣）** | Units/Monk | 给其他敌人加 Buff | 低攻中血，治疗/增益 |

### Boss 设计（每 5 层一个）

| Boss | 特点 | 技能 |
|------|------|------|
| **黑骑士** | 高血高防 | 冲刺斩击、召唤小兵 |
| **大法师** | 远程高伤 | 三向弹幕、范围爆炸、瞬移 |
| **领主** | 均衡型 | 多阶段（50%血切换攻击模式）|

## 6. 与已有系统的整合点

### 属性系统（`baseSystem/properties/`）

```typescript
// 创建玩家时，用属性系统管理 CombatComp 的值
const playerProps = new GeneralPropertyMgr();
playerProps.registerProperty('atk', new BaseValueProperty(10));
playerProps.registerProperty('def', new BaseValueProperty(3));
playerProps.registerProperty('atkSpeed', new BaseValueProperty(1.0));
playerProps.registerProperty('critRate', new BaseValueProperty(0.05));
playerProps.registerProperty('critDmg', new BaseValueProperty(1.5));

// 技能/Buff 通过 Modifier 修改属性
playerProps.addModifier('atk', new Modifier(ModifierType.Flat, 5)); // +5 攻击
playerProps.addModifier('atkSpeed', new Modifier(ModifierType.Percent, 0.2)); // +20% 攻速
```

### Buff 系统（`baseSystem/buff/`）

战斗中的状态效果直接走 Buff 系统：
- 中毒：每秒扣 HP
- 减速：修改移动速度
- 护盾：吸收伤害
- 燃烧：持续范围伤害

详见 [05-Buff与技能系统](./05-Buff与技能系统.md)。

## 关键设计决策

| 决策 | 理由 |
|------|------|
| 不用物理引擎做碰撞 | 圆形碰撞检测够用，避免 RigidBody 的复杂性 |
| 弹道是 Entity 而非纯特效 | 需要碰撞检测、穿透、追踪等逻辑，Entity 更灵活 |
| 伤害公式从最简开始 | V1 够玩了，复杂度随需求递增，不提前过度设计 |
| 敌人类型用配置区分 | 同一套 ECS 组件，不同数据 = 不同敌人，零额外代码 |
