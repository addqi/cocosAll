# 07 - 战斗系统实战：属性 × Buff × 战斗 统合

## 目标

用现有的属性系统 + Buff 系统，实现一个完整的"玩家攻击敌人"战斗演示。

### 场景描述

- 一个玩家（已有属性/战斗模块）
- 一个敌人示例（仅有血量 + 防御）
- 若干按钮：
  1. 玩家攻击力 +10%
  2. 玩家暴击率 +10%
  3. 敌人防御 -10%
  4. 敌人添加回血 Buff（最大生命 10%/秒，持续 5 次 tick）
  5. 玩家添加吸血 Buff（造成伤害时回复一定比例 HP）
  6. 玩家攻击敌人（执行一次伤害流程）

## 属性公式回顾

每个属性的计算结构（以 Attack 为例）：

```
Attack-Value = Attack-Value-Config + Attack-Value-Buff + Attack-Value-Other
Attack       = Attack-Value × (1 + Attack-Mul-Buff + Attack-Mul-Other)
```

- **加成**：往 `*-Value-Buff` 加 Add 修饰器 → 固定值加成
- **百分比**：往 `*-Mul-Buff` 加 Add 修饰器（注意不是 Mul！）→ 百分比加成
- Buff 通过 `AttributeChangeResolver.apply(changes, propMgr)` 桥接到属性节点

## 伤害流程

```
攻击方                                  防御方
PlayerCombat                            EnemyCombat
    │                                       │
    │  baseDmg = getValue(Attack)           │
    │  { damage, isCrit } =                 │
    │     calcOutgoingDamage(baseDmg)       │
    │                                       │
    │  ────── damage ──────────────────►    │
    │                                  takeDamage(damage)
    │                                  actual = max(damage - defense, 1)
    │                                  currentHp -= actual
    │                                       │
    │  ◄────── actual ─────────────────     │
    │                                       │
    │  if (lifestealRate > 0)               │
    │     heal(actual * lifestealRate)       │
```

## 实施步骤

### 第一步：新增 LifestealRate 属性

**为什么需要**：吸血 Buff 通过修改 `LifestealRate` 属性来生效，PlayerCombat 在造成伤害后读取该属性回血。不需要在战斗逻辑里硬编码任何 Buff 判断。

1. 新建 `game/config/propertyConfig/lifesteal_rate.json`

```json
{
  "attribute": "LifestealRate",
  "valueNodes": [
    { "id": "LifestealRate-Value-Config", "value": 0, "tag": "base" },
    { "id": "LifestealRate-Value-Buff",   "value": 0, "tag": "add-buff" },
    { "id": "LifestealRate-Value-Other",  "value": 0, "tag": "add-other" },
    { "id": "LifestealRate-Mul-Buff",     "value": 0, "tag": "mul-buff" },
    { "id": "LifestealRate-Mul-Other",    "value": 0, "tag": "mul-other" }
  ],
  "computeNodes": [
    { "id": "LifestealRate-Value", "expression": "{{LifestealRate-Value-Config}} + {{LifestealRate-Value-Buff}} + {{LifestealRate-Value-Other}}" },
    { "id": "LifestealRate",       "expression": "{{LifestealRate-Value}} * (1 + {{LifestealRate-Mul-Buff}} + {{LifestealRate-Mul-Other}})" }
  ]
}
```

2. `EPropertyId` 枚举新增 `LifestealRate = 'LifestealRate'`
3. `attributeConfigs.ts` 导入新 JSON
4. `propConfigMap.ts` 新增 LifestealRate 映射
5. `player.json` 新增 `"LifestealRate": { "base": 0, "min": 0, "max": 1.0 }`

### 第三步：5 个 Buff 效果 ✅

每个 Buff 需要：**一个 JSON 配置** + **一个 BuffEffect 子类**。
Buff 1/2/3/5 共用 `SimpleAttrBuffEffect` 基类（ADD targetAttr × valuePerStack × stack）。

#### 架构：SimpleAttrBuffEffect 继承树

```
BuffEffectBase (abstract)
├── SimpleAttrBuffEffect ← getChanges + 通用日志
│   ├── AttackBoostEffect   (id:2001, Attack-Mul-Buff, +0.1/层)
│   ├── CritBoostEffect     (id:2002, CritRate-Value-Buff, +0.1/层)
│   ├── DefenseReduceEffect (id:3001, Defense-Mul-Buff, -0.1/层)
│   └── LifestealEffect     (id:2003, LifestealRate-Value-Buff, +0.1/层)
├── RegenEffect              (id:3002, onTick 回血, duration=5s)
└── AddHpBuffEffect          (id:1001, 已有)
```

#### Buff 1~3, 5：简单属性修改

四个子类各一行继承 `SimpleAttrBuffEffect`，由 JSON `targetAttr` + `valuePerStack` 驱动：

| Buff | id | targetAttr | valuePerStack | maxStack | 效果 |
|------|----|-----------|---------------|----------|------|
| 攻击强化 | 2001 | Attack-Mul-Buff | 0.1 | 5 | 攻击 +10%/层 |
| 暴击强化 | 2002 | CritRate-Value-Buff | 0.1 | 5 | 暴击率 +10%/层 |
| 破甲 | 3001 | Defense-Mul-Buff | -0.1 | 5 | 敌人防御 -10%/层 |
| 吸血 | 2003 | LifestealRate-Value-Buff | 0.1 | 3 | 吸血率 +10%/层 |

#### Buff 4：回血 Buff（敌人） — RegenEffect

不修改属性，靠 `onTick` 每秒治疗 `maxHp × healPercent`。`duration=5` + `tickInterval=1` → 共 5 次。

**IBuffOwner 扩展**：新增可选 `heal?(amount: number): void`，`EnemyBuffOwner` 构造接收 `EnemyCombat` 引用桥接 `heal()`，`PlayerControl` 内联 owner 同理。

```typescript
// RegenEffect.onTick — 读 maxHp 属性，调 owner.heal()
onTick() {
    const owner = this.runtime.owner;
    const maxHp = owner.getPropertyManager().getValue(EPropertyId.Hp);
    const healAmt = maxHp * (this.data.healPercent ?? 0.1);
    owner.heal?.(healAmt);
}
```

#### Buff 5：吸血 — LifestealEffect + PlayerCombat

`LifestealEffect` 修改 `LifestealRate` 属性值，`PlayerCombat.attack()` 在造成伤害后读取该属性自动回血，Buff 与战斗代码完全解耦。

## 关键设计决策

### 为什么吸血做成属性而不是特殊逻辑？

```
❌ 在 PlayerCombat 里 hardcode "检查是否有吸血 Buff"
   → 每加一种战斗效果就改一次战斗代码，耦合越来越重

✅ 吸血率是一个属性（LifestealRate），Buff 只改属性值
   → PlayerCombat 只看属性，不知道 Buff 的存在
   → 新增"法术吸血"只需要新属性 + 新 Buff，零改动战斗代码
```

### 为什么回血 Buff 用 onTick 而不是属性？

回血是**瞬时动作**（每秒扣一次 currentHp），不是持续的属性修改。用 `onTick` + `owner.heal()` 是正确的模型。如果做成属性（如 HpRegen），还需要一个系统每帧读属性执行回血，反而更复杂。

### Buff 1/2/3/5 → SimpleAttrBuffEffect 基类

它们都是"往某个属性节点 ADD (valuePerStack × stack)"。四份一样的代码是不必要的重复。

**已实现**：抽取 `SimpleAttrBuffEffect` 基类（含通用 `getChanges` + 日志），四个子类各一行继承：

```typescript
// SimpleAttrBuffEffect.ts — 公共逻辑
export class SimpleAttrBuffEffect extends BuffEffectBase {
    getChanges(): AttributeChange[] {
        return [{ attrId: this.data.targetAttr, type: EChangeType.ADD, value: this.data.valuePerStack * this.runtime.stack }];
    }
    onAdd() { ... }  onRemove() { ... }  onStack() { ... }
}

// CritBoostEffect.ts — 零重复
@buffEffect export class CritBoostEffect extends SimpleAttrBuffEffect {}
```

未来某个 Buff 需要特殊的 onAdd/onTick，直接 override 即可，不影响其他子类。

```
鼠标左键 / KEY_J
    ↓
PlayerControl.lateUpdate 检测到 Attack
    ↓
_findNearestEnemy() 遍历 EnemyControl.allEnemies
    ↓
  有敌人在 attackRange 内 → 设置 ctx.targetEnemy → Shoot 状态翻转朝向 → Arrow 飞向敌人 → 命中触发 combat.attack()
  无敌人 → ctx.targetEnemy = null → 保持朝向 → Arrow 朝面朝方向飞 attackRange 距离后销毁
```
