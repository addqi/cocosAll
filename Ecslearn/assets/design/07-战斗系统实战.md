# 07 - 战斗系统实战：属性 × Buff × 战斗 统合

## 目标

用现有的属性系统 + Buff 系统，实现一个完整的"玩家攻击敌人"战斗演示。

### 场景描述

- 一个玩家（已有属性/战斗模块）
- 一个敌人示例（仅有血量 + 防御）
- 若干按钮：
  1. 玩家攻击力 +10%
  2. 玩家暴击率 +10%
  3. 敌人防御 -10%
  4. 敌人添加回血 Buff（最大生命 10%/秒，持续 5 次 tick）
  5. 玩家添加吸血 Buff（造成伤害时回复一定比例 HP）
  6. 玩家攻击敌人（执行一次伤害流程）

## 属性公式回顾

每个属性的计算结构（以 Attack 为例）：

```
Attack-Value = Attack-Value-Config + Attack-Value-Buff + Attack-Value-Other
Attack       = Attack-Value × (1 + Attack-Mul-Buff + Attack-Mul-Other)
```

- **加成**：往 `*-Value-Buff` 加 Add 修饰器 → 固定值加成
- **百分比**：往 `*-Mul-Buff` 加 Add 修饰器（注意不是 Mul！）→ 百分比加成
- Buff 通过 `AttributeChangeResolver.apply(changes, propMgr)` 桥接到属性节点

## 伤害流程

```
攻击方                                  防御方
PlayerCombat                            EnemyCombat
    │                                       │
    │  baseDmg = getValue(Attack)           │
    │  { damage, isCrit } =                 │
    │     calcOutgoingDamage(baseDmg)       │
    │                                       │
    │  ────── damage ──────────────────►    │
    │                                  takeDamage(damage)
    │                                  actual = max(damage - defense, 1)
    │                                  currentHp -= actual
    │                                       │
    │  ◄────── actual ─────────────────     │
    │                                       │
    │  if (lifestealRate > 0)               │
    │     heal(actual * lifestealRate)       │
```

## 实施步骤

### 第一步：新增 LifestealRate 属性

**为什么需要**：吸血 Buff 通过修改 `LifestealRate` 属性来生效，PlayerCombat 在造成伤害后读取该属性回血。不需要在战斗逻辑里硬编码任何 Buff 判断。

1. 新建 `game/config/propertyConfig/lifesteal_rate.json`

```json
{
  "attribute": "LifestealRate",
  "valueNodes": [
    { "id": "LifestealRate-Value-Config", "value": 0, "tag": "base" },
    { "id": "LifestealRate-Value-Buff",   "value": 0, "tag": "add-buff" },
    { "id": "LifestealRate-Value-Other",  "value": 0, "tag": "add-other" },
    { "id": "LifestealRate-Mul-Buff",     "value": 0, "tag": "mul-buff" },
    { "id": "LifestealRate-Mul-Other",    "value": 0, "tag": "mul-other" }
  ],
  "computeNodes": [
    { "id": "LifestealRate-Value", "expression": "{{LifestealRate-Value-Config}} + {{LifestealRate-Value-Buff}} + {{LifestealRate-Value-Other}}" },
    { "id": "LifestealRate",       "expression": "{{LifestealRate-Value}} * (1 + {{LifestealRate-Mul-Buff}} + {{LifestealRate-Mul-Other}})" }
  ]
}
```

2. `EPropertyId` 枚举新增 `LifestealRate = 'LifestealRate'`
3. `attributeConfigs.ts` 导入新 JSON
4. `propConfigMap.ts` 新增 LifestealRate 映射
5. `player.json` 新增 `"LifestealRate": { "base": 0, "min": 0, "max": 1.0 }`

### 第三步：5 个 Buff 效果

每个 Buff 需要：**一个 JSON 配置** + **一个 BuffEffect 子类**

#### Buff 1：攻击力 +10%（玩家）

**原理**：往 `Attack-Mul-Buff` 加 0.1（因为公式是 `Attack-Value × (1 + Mul-Buff + ...)`，加 0.1 就是 +10%）

`buffConfig/attackBoost.json`：
```json
{
  "id": 2001,
  "name": "攻击强化",
  "duration": 0,
  "maxStack": 5,
  "effectClass": "AttackBoostEffect",
  "targetAttr": "Attack-Mul-Buff",
  "valuePerStack": 0.1
}
```

`effects/AttackBoostEffect.ts`：
```typescript
@buffEffect
export class AttackBoostEffect extends BuffEffectBase {
    getChanges(): AttributeChange[] {
        return [{
            attrId: this.data.targetAttr,
            type: EChangeType.ADD,
            value: this.data.valuePerStack * this.runtime.stack
        }];
    }
}
```

#### Buff 2：暴击率 +10%（玩家）

**原理**：往 `CritRate-Value-Buff` 加 0.1（CritRate 范围 0~1，+0.1 即 +10 个百分点）

`buffConfig/critBoost.json`：
```json
{
  "id": 2002,
  "name": "暴击强化",
  "duration": 0,
  "maxStack": 5,
  "effectClass": "CritBoostEffect",
  "targetAttr": "CritRate-Value-Buff",
  "valuePerStack": 0.1
}
```

`effects/CritBoostEffect.ts`：同结构，`getChanges` 返回 ADD 到 targetAttr。

> 注意：Buff 1 和 Buff 2 的 Effect 代码结构完全一致。可以做成一个通用的 `SimpleAttrBuffEffect`，但先保持独立类以便后续扩展。

#### Buff 3：防御 -10%（敌人）

**原理**：往 `Defense-Mul-Buff` 加 -0.1（减益效果用负数）

`buffConfig/defenseReduce.json`：
```json
{
  "id": 3001,
  "name": "破甲",
  "duration": 0,
  "maxStack": 5,
  "effectClass": "DefenseReduceEffect",
  "targetAttr": "Defense-Mul-Buff",
  "valuePerStack": -0.1
}
```

`effects/DefenseReduceEffect.ts`：同结构。

#### Buff 4：回血 Buff（敌人）

**原理**：`tickInterval` 触发 `onTick`，每次 tick 回复最大生命 10%。持续 5 次后自然结束。

`buffConfig/regenBuff.json`：
```json
{
  "id": 3002,
  "name": "生命回复",
  "duration": 5,
  "maxStack": 1,
  "tickInterval": 1,
  "effectClass": "RegenEffect",
  "healPercent": 0.1
}
```

`effects/RegenEffect.ts`：
```typescript
@buffEffect
export class RegenEffect extends BuffEffectBase {
    getChanges(): AttributeChange[] {
        return []; // 不修改属性，靠 onTick 治疗
    }

    onTick() {
        const owner = this.runtime.owner;
        const maxHp = owner.getPropertyManager().get('Hp');
        const healAmt = maxHp * this.data.healPercent;
        // owner 需要暴露 combat.heal() 接口
        owner.heal?.(healAmt);
    }
}
```

**关键问题**：`IBuffOwner` 当前只有 `getPropertyManager()`，没有 `heal()`。需要扩展 `IBuffOwner` 接口或让 owner 实现 heal 方法。

**方案**：在 `IBuffOwner` 中新增可选方法 `heal?(amount: number): void`，敌人的 owner 实现时桥接到 `EnemyCombat.heal()`。

#### Buff 5：吸血 Buff（玩家）

**原理**：修改 `LifestealRate` 属性，PlayerCombat 在造成伤害后自动读取该值回血。

`buffConfig/lifestealBuff.json`：
```json
{
  "id": 2003,
  "name": "吸血",
  "duration": 0,
  "maxStack": 3,
  "effectClass": "LifestealEffect",
  "targetAttr": "LifestealRate-Value-Buff",
  "valuePerStack": 0.1
}
```

`effects/LifestealEffect.ts`：同 Buff 1/2/3 的结构，ADD 到 targetAttr。

**PlayerCombat 修改**：攻击后检查 LifestealRate，如果 > 0 则 heal：

```typescript
attack(target: EnemyCombat): { damage: number; isCrit: boolean; healed: number } {
    const baseDmg = this.prop.getValue(EPropertyId.Attack);
    const { damage, isCrit } = this.calcOutgoingDamage(baseDmg);
    const actual = target.takeDamage(damage);

    let healed = 0;
    const lifesteal = this.prop.getValue(EPropertyId.LifestealRate);
    if (lifesteal > 0) {
        healed = this.heal(actual * lifesteal);
    }
    return { damage: actual, isCrit, healed };
}
```

### 第四步：战斗演示脚本

```
game/test/
└── BattleDemo.ts   ← 挂在场景节点上
```

```typescript
@ccclass('BattleDemo')
export class BattleDemo extends Component {
    // 玩家
    private playerProp: PlayerProperty;
    private playerBuffOwner: IBuffOwner;
    private playerBuffMgr: EntityBuffMgr;
    private playerCombat: PlayerCombat;

    // 敌人
    private enemyProp: EnemyProperty;
    private enemyBuffOwner: IBuffOwner;
    private enemyBuffMgr: EntityBuffMgr;
    private enemyCombat: EnemyCombat;

    onLoad() {
        // 初始化玩家
        this.playerProp = new PlayerProperty();
        this.playerBuffOwner = new PlayerBuffOwner(this.playerProp);
        this.playerBuffMgr = new EntityBuffMgr(this.playerProp);
        this.playerCombat = new PlayerCombat(this.playerProp);

        // 初始化敌人
        this.enemyProp = new EnemyProperty();
        this.enemyBuffOwner = new EnemyBuffOwner(this.enemyProp, this.enemyCombat);
        this.enemyBuffMgr = new EntityBuffMgr(this.enemyProp);
        this.enemyCombat = new EnemyCombat(this.enemyProp);
    }

    update(dt: number) {
        this.playerBuffMgr.update(dt);
        this.enemyBuffMgr.update(dt);
    }

    // 按钮 1：玩家攻击力 +10%
    onClickAttackBoost() {
        this.playerBuffMgr.addBuff(attackBoostConfig, this.playerBuffOwner);
    }

    // 按钮 2：玩家暴击率 +10%
    onClickCritBoost() {
        this.playerBuffMgr.addBuff(critBoostConfig, this.playerBuffOwner);
    }

    // 按钮 3：敌人防御 -10%
    onClickDefReduce() {
        this.enemyBuffMgr.addBuff(defReduceConfig, this.enemyBuffOwner);
    }

    // 按钮 4：敌人回血 Buff
    onClickRegen() {
        this.enemyBuffMgr.addBuff(regenConfig, this.enemyBuffOwner);
    }

    // 按钮 5：玩家吸血 Buff
    onClickLifesteal() {
        this.playerBuffMgr.addBuff(lifestealConfig, this.playerBuffOwner);
    }

    // 按钮 6：玩家攻击敌人
    onClickAttack() {
        const result = this.playerCombat.attack(this.enemyCombat);
        // 显示结果
    }
}
```

## 关键设计决策

### 为什么吸血做成属性而不是特殊逻辑？

```
❌ 在 PlayerCombat 里 hardcode "检查是否有吸血 Buff"
   → 每加一种战斗效果就改一次战斗代码，耦合越来越重

✅ 吸血率是一个属性（LifestealRate），Buff 只改属性值
   → PlayerCombat 只看属性，不知道 Buff 的存在
   → 新增"法术吸血"只需要新属性 + 新 Buff，零改动战斗代码
```

### 为什么回血 Buff 用 onTick 而不是属性？

回血是**瞬时动作**（每秒扣一次 currentHp），不是持续的属性修改。用 `onTick` + `owner.heal()` 是正确的模型。如果做成属性（如 HpRegen），还需要一个系统每帧读属性执行回血，反而更复杂。

### Buff 1/2/3/5 的 Effect 代码几乎一样

它们都是"往某个节点加一个值 × 层数"。如果觉得冗余，可以抽一个 `SimpleAttrEffect` 通用类，但先保持独立——未来每个 Buff 的 onAdd/onRemove/onTick 可能不同。
