# 00 - 肉鸽游戏策划案：项目总览

## 我们要做什么

一款 **Cocos Creator 3.x 2D 俯视角肉鸽射击游戏**，核心玩法参考「幽灵射手」（Archero 类）：

- 玩家操控角色在有限地图中移动
- 停下时自动攻击最近的敌人
- 清完一层的敌人后通过传送门进入下一层
- 每层结束后随机三选一获得技能/增益
- 死亡后重头开始，部分成长保留（Meta 进度）

**与参考项目的关键差异**：我们基于已有的 **ECS 架构 + Buff 系统 + 属性系统**来构建，而不是幽灵射手的「全局 GameManager + 散落 Component」模式。数据结构先行，代码自然清晰。

---

## 参考项目「幽灵射手」核心拆解

| 系统 | 幽灵射手做法 | 我们的改进方向 |
|------|-------------|---------------|
| **架构** | GameManager 静态单例持有所有引用 | ECS：Entity + Component + System 分离 |
| **移动** | 虚拟摇杆 → RigidBody.move | 摇杆 → InputComponent → MovementSystem |
| **攻击** | Player 脚本内 update 检测 + 动画回调射箭 | CombatSystem 统一处理攻击节奏与弹道 |
| **敌人** | Monster/Boss 各自脚本，GameManager.arrMonster 数组管理 | ECS Entity + AIComponent，System 统一调度 |
| **技能** | PlayerData.arrSkill + parsePlayerSkill 重算属性 | Buff 系统 + 属性修饰器链，干净解耦 |
| **关卡** | CSV checkpoint 固定配置，map 表随机选一张 | 房间模板 + 程序化组合，更「肉鸽」|
| **数据** | CSV 文件 + LocalConfig 全局加载 | JSON 配置 + TypeScript 类型安全 |

## 技术选型决策

### 用 ECS，但不要过度 ECS

ECS 用在它该用的地方：

- **实体管理**：玩家、敌人、弹道、拾取物 → Entity + Component
- **行为系统**：移动、碰撞、战斗、AI → System
- **属性计算**：基础值 + 修饰器 → 已有属性系统

**不用 ECS 的地方**：

- UI 管理（Cocos 节点树天然适合）
- 场景切换 / 游戏状态（简单状态机够了）
- 音效 / 粒子特效（Cocos 原生 API 直接用）

### 数据格式：JSON > CSV

幽灵射手用 CSV，我们用 JSON：
- TypeScript 天然解析，不需要额外 CSV 解析器
- 支持嵌套结构，描述技能效果更自然
- 已有配置文件（`buffConfig/`、`propertyConfig/`）都是 JSON

## 游戏核心数据结构（最重要的事）

> "Bad programmers worry about the code. Good programmers worry about data structures."

在写任何代码之前，先把核心数据结构定义清楚：

```typescript
/** 一局游戏的运行时状态 */
interface RunState {
    floor: number;              // 当前层数
    roomIndex: number;          // 当前房间索引
    phase: 'combat' | 'reward' | 'shop' | 'transition';
    enemiesAlive: number;       // 剩余敌人数
}

/** 玩家局内状态（死亡清零） */
interface RunPlayerState {
    hp: number;
    maxHp: number;
    gold: number;
    skills: SkillId[];          // 本局获得的技能
    buffs: BuffId[];            // 当前激活的 Buff
}

/** 玩家持久化状态（跨局保留）*/
interface MetaPlayerState {
    totalGold: number;
    highestFloor: number;
    unlockedSkills: SkillId[];  // 解锁池
    permanentUpgrades: Record<UpgradeId, number>;
}
```

这三个结构是整个游戏的**数据根节点**。所有系统都围绕它们运转。
