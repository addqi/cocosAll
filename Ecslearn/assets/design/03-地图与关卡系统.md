# 03 - 地图与关卡系统

> 前置阅读：[01-核心游戏循环](./01-核心游戏循环.md)

## 本章目标

实现**房间式关卡系统**：每一层是一个独立房间，清完敌人后通过传送门进入下一层。地图使用 Tiny Swords 的地形素材，支持模板化生成。

---

## 1. 幽灵射手 vs 我们的方案

| 维度 | 幽灵射手 | 我们的方案 |
|------|---------|-----------|
| **地图来源** | CSV 表定义固定格子配置 | JSON 房间模板 + 随机组合 |
| **关卡结构** | 线性：第 1 层 → 第 2 层 → ... | 线性 + 分支事件（战斗/商店/宝箱）|
| **地图多样性** | 多张 map 表用 `#` 分隔随机选一张 | 多个房间模板随机选取 + 敌人配置随机 |
| **传送门** | 清完 NPC/爱心后生成 | 清完所有敌人后生成 |

---

## 2. 房间模板系统

### 核心思路

不做完全程序化生成（太复杂，新手不友好），而是：
1. **手工设计 10-20 个房间模板**
2. **每层随机选一个模板**
3. **在模板的预设出生点随机放敌人**

这是幽灵射手的做法（CSV map 表），但我们用更结构化的 JSON。

### 房间模板数据结构

```typescript
interface RoomTemplate {
    id: string;
    width: number;                // 房间宽（瓦片数）
    height: number;               // 房间高（瓦片数）
    tileSize: number;             // 每个瓦片像素
    terrain: TerrainType[][];     // 地形二维数组
    spawnPoints: SpawnPoint[];    // 敌人出生点
    playerSpawn: Vec2;            // 玩家出生点
    warpGatePos: Vec2;            // 传送门位置
    obstacles: ObstacleInfo[];    // 障碍物
    decorations: DecoInfo[];      // 纯装饰物
}

enum TerrainType {
    Grass = 0,       // 可行走
    Water = 1,       // 不可行走
    Bridge = 2,      // 可行走（视觉区分）
    Wall = 3,        // 不可行走（视觉遮挡）
}

interface SpawnPoint {
    position: Vec2;
    allowedTypes: string[];  // 可以在此点生成的敌人类型
    weight: number;          // 权重（越高越可能使用此点）
}
```

## 3. 关卡流转机制

### 一局游戏的关卡序列

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ 第1层    │──→│ 技能选择 │──→│ 第2层    │──→│ 技能选择 │──→│ 第3层    │
│ 战斗     │   │ 三选一   │   │ 战斗     │   │ 三选一   │   │ 战斗     │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
                                                              │
                                                              ▼
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ 第4层    │──→│ 商店     │──→│ 第5层    │──→│ 技能选择 │──→│ BOSS    │
│ 战斗     │   │ 购买道具 │   │ 战斗     │   │ 三选一   │   │ 第6层    │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
```

### 层类型

```typescript
enum FloorType {
    Combat,     // 普通战斗层（最常见）
    Elite,      // 精英层（更强的敌人，更好的奖励）
    Boss,       // Boss 层（每 5 层一次）
    Shop,       // 商店层（可购买/升级）
    Rest,       // 休息层（回血/移除负面Buff）
    Treasure,   // 宝箱层（随机奖励）
}
```

### 层序列生成器

```typescript
class FloorSequenceGenerator {
    generate(totalFloors: number): FloorType[] {
        const sequence: FloorType[] = [];

        for (let i = 1; i <= totalFloors; i++) {
            if (i % 10 === 0) {
                sequence.push(FloorType.Boss);     // 每10层一个Boss
            } else if (i % 5 === 0) {
                sequence.push(FloorType.Shop);     // 每5层一个商店
            } else if (i % 3 === 0 && Math.random() < 0.3) {
                sequence.push(FloorType.Elite);    // 每3层30%概率精英
            } else {
                sequence.push(FloorType.Combat);   // 其余普通战斗
            }
        }

        return sequence;
    }
}
```

## 4. 敌人配置与难度曲线

### 每层敌人数量公式

```typescript
function getEnemyCount(floor: number): number {
    const base = 3;
    const growth = Math.floor(floor / 3);
    const max = 10;
    return Math.min(base + growth, max);
}
// 第1层: 3只  第3层: 4只  第6层: 5只  第12层: 7只  第21层+: 10只
```

### 敌人属性缩放

> 参考：幽灵射手的 `checkpoint.csv` 定义每层的攻防血速加成倍率

```typescript
interface FloorScaling {
    hpMultiplier: number;
    atkMultiplier: number;
    defMultiplier: number;
    speedMultiplier: number;
}

function getFloorScaling(floor: number): FloorScaling {
    return {
        hpMultiplier:    1 + floor * 0.15,    // 每层+15%
        atkMultiplier:   1 + floor * 0.10,    // 每层+10%
        defMultiplier:   1 + floor * 0.05,    // 每层+5%
        speedMultiplier: 1 + floor * 0.02,    // 每层+2%（不能太快）
    };
}
```

### 敌人选择策略

```typescript
function selectEnemiesForFloor(
    floor: number,
    count: number,
    floorType: FloorType
): EnemyConfig[] {
    // 根据层数解锁敌人类型
    const pool: string[] = ['pawn'];
    if (floor >= 3) pool.push('archer');
    if (floor >= 6) pool.push('warrior');
    if (floor >= 9) pool.push('lancer');
    if (floor >= 12) pool.push('monk');

    const enemies: EnemyConfig[] = [];
    const scaling = getFloorScaling(floor);

    for (let i = 0; i < count; i++) {
        const type = pool[Math.floor(Math.random() * pool.length)];
        const base = ENEMY_BASE_STATS[type];
        enemies.push({
            type,
            hp: Math.round(base.hp * scaling.hpMultiplier),
            atk: Math.round(base.atk * scaling.atkMultiplier),
            def: Math.round(base.def * scaling.defMultiplier),
            speed: base.speed * scaling.speedMultiplier,
        });
    }

    return enemies;
}
```

## 5. 传送门机制

```typescript
class WarpGateManager {
    private _gateNode: Node | null = null;

    spawnGate(position: Vec2, parentNode: Node): void {
        // 从对象池获取或实例化
        this._gateNode = this._createGateNode();
        this._gateNode.setPosition(position.x, position.y);
        parentNode.addChild(this._gateNode);

        // 出现特效
        this._gateNode.setScale(0, 0);
        tween(this._gateNode)
            .to(0.5, { scale: new Vec3(1, 1, 1) }, { easing: 'backOut' })
            .start();
    }

    checkPlayerEnter(playerPos: Vec2): boolean {
        if (!this._gateNode) return false;
        const gatePos = new Vec2(
            this._gateNode.position.x,
            this._gateNode.position.y
        );
        return Vec2.distance(playerPos, gatePos) < 30;
    }

    cleanup(): void {
        this._gateNode?.removeFromParent();
        this._gateNode = null;
    }
}
```

## 7. 新手实现步骤

- [ ] 创建一个 JSON 房间模板文件

- [ ] 在场景中渲染出一个矩形房间

### Step 2：替换美术（预计 1 小时）

- [ ] 添加装饰物
- **验收**：房间看起来像个中世纪场景

- [ ] 实现 `selectEnemiesForFloor` 函数
- [ ] 在房间模板的 spawnPoints 处生成敌人
- [ ] 敌人全灭后生成传送门

- **验收**：打完一层进下一层，敌人逐渐变强

- [ ] 设计 3-5 个不同布局的房间模板
- [ ] 每层随机选一个
- **验收**：连续打几层，每层布局不同

## 关键设计决策

| 决策 | 理由 |
|------|------|
| 模板 + 随机组合 > 纯程序化生成 | 手工设计保证质量，随机选择保证新鲜感，对新手友好 |
| 敌人类型随层数解锁 | 玩家逐步认识新敌人，不会一上来被信息轰炸 |
| 线性缩放而非指数缩放 | 指数缩放会让后期难度爆炸，线性更可控 |
| 传送门位置写在模板里 | 避免生成在障碍物上的 edge case |
