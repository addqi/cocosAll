# 射击与弹道系统教程

> 基于 Ecslearn 项目 `game/shoot` + `game/projectile` 模块，适用于 Cocos Creator 3.x + TypeScript。

---

## 一、概述

### 1.1 射击系统的五个阶段

```
①触发判定     ②弹道决议     ③弹丸配置     ④飞行         ⑤命中
IShootPolicy → ShootResolver → ProjectileConfig → ArrowProjectile → HitEffectMgr
             → ShotDescriptor[]  （属性快照）    （Bezier 飞行）   （伤害管线）
```

- **触发**：何时射击（按住/点击/自动）
- **决议**：多少箭、朝哪飞（扇形角度、有无目标）
- **配置**：穿透/弹射/分裂/追踪等参数快照
- **飞行**：沿二次贝塞尔曲线移动，到终点或出屏回收
- **命中**：进入 `HitEffectMgr` 管线（见命中效果教程）

### 1.2 设计理念

| 原则 | 说明 |
|------|------|
| **策略互斥** | `IShootPolicy` 通过 `priority` 排他，同时只生效一种射击模式 |
| **属性驱动** | 箭矢数量、角度、穿透等全部从 `PlayerProperty` 读取 |
| **配置快照** | 射击瞬间冻结弹道参数，飞行途中不受后续属性变化影响 |
| **对象池** | `ProjectilePool` 复用节点，避免频繁 `new Node()` |

## 二、触发判定 —— IShootPolicy

### 2.1 接口定义

```typescript
export interface IShootPolicy {
    readonly priority: number;
    wantShoot(input: ActionComp, hasTarget: boolean, isMoving: boolean): boolean;
}
```

每帧由 `PlayerControl.lateUpdate` 调用 `wantShoot()`，返回 `true` 则进入射击状态。

### 2.2 三种策略

| 类 | priority | 触发条件 |
|----|----------|----------|
| `ClickToShoot` | 0 | `justPressed.has(Attack)` — 点一下射一次 |
| `HoldToShoot` | 1 | `active.has(Attack)` — 按住持续射 |
| `AutoShoot` | 2 | Lv1: 静止+有目标；Lv2: 有目标（移动中也可）；Lv3: 始终射击 |

```typescript
export class HoldToShoot implements IShootPolicy {
    readonly priority = 1;
    wantShoot(input: ActionComp): boolean {
        return input.active.has(EAction.Attack);
    }
}

export class AutoShoot implements IShootPolicy {
    readonly priority = 2;
    constructor(private _level: number = 1) {}

    wantShoot(_input: ActionComp, hasTarget: boolean, isMoving: boolean): boolean {
        if (this._level >= 3) return true;
        if (!hasTarget) return false;
        if (this._level >= 2) return true;
        return !isMoving;
    }
}
```

### 2.3 切换策略

`PlayerControl` 持有当前 `IShootPolicy`，通过 `setShootPolicy()` 替换：

```typescript
// 升级系统切换
ctx.setShootPolicy(new AutoShoot(2));

// 技能结束恢复
ctx.setShootPolicy(new HoldToShoot());
```

策略之间**互斥**——同时只有一个生效，无需优先级仲裁。

## 三、弹道决议 —— ShootResolver

### 3.1 职责

读取属性，计算出一组 `ShotDescriptor[]`（每个箭矢的飞行曲线）：

```typescript
export interface ShotDescriptor {
    curve: QuadBezier;      // 二次贝塞尔曲线
    duration: number;       // 飞行时长（秒）
    hasTarget: boolean;     // 是否有锁定目标
    target: EnemyControl | null;
}
```

### 3.2 核心方法

```typescript
static resolve(
    shooterPos: Vec3,      // 射手位置
    facingX: number,       // 面向方向（+1 右，-1 左）
    target: EnemyControl | null,
    prop: PlayerProperty,
): ShotDescriptor[]
```

**流程**：
1. 读 `ExtraProjectiles` → 总箭数 = `1 + extraProj`
2. 读 `SpreadAngle` → 多箭扇形角度
3. 读 `ArrowSpeed` → 计算飞行时长
4. 有目标 → `_resolveTargeted()`；无目标 → `_resolveUntargeted()`

### 3.3 有目标弹道

多箭时通过**弧度倍率**区分轨迹（不是角度偏移）：

```typescript
// 单箭
const curve = QuadBezier.createArc(start, end, dist * arrowArcRatio * facing);

// 多箭：每支箭不同弧度
for (let i = 0; i < count; i++) {
    const arcMul = 0.5 + i / (count - 1);
    const curve = QuadBezier.createArc(start, end, arcBase * arcMul * facing);
    results.push({ curve, duration, hasTarget: true, target });
}
```

所有箭锁定同一目标，但飞行轨迹不同 → 视觉上呈扇形散开再汇聚。

### 3.4 无目标弹道

按 `SpreadAngle` 在面向方向上均匀分布：

```typescript
const halfSpread = (spreadDeg * DEG2RAD) / 2;
const step = count > 1 ? (spreadDeg * DEG2RAD) / (count - 1) : 0;

for (let i = 0; i < count; i++) {
    const angle = baseAngle - halfSpread + step * i;
    const end = new Vec3(
        start.x + Math.cos(angle) * range,
        start.y + Math.sin(angle) * range, 0
    );
    // ...
}
```

### 3.5 配置快照

```typescript
static snapshotProjectileConfig(prop: PlayerProperty): ProjectileConfig {
    return {
        pierceCount:    Math.floor(prop.getValue(EPropertyId.PierceCount)),
        bounceCount:    Math.floor(prop.getValue(EPropertyId.BounceCount)),
        splitCount:     0,
        splitDamageRatio: 0.4,
        homingStrength: prop.getValue(EPropertyId.HomingStrength),
        damageRatio:    1,
    };
}
```

射击瞬间调用一次，生成的 `ProjectileConfig` 随箭矢携带，飞行途中不再读属性。

## 四、弹道配置 —— ProjectileConfig

```typescript
export interface ProjectileConfig {
    pierceCount: number;       // 穿透次数（0 = 不穿透）
    bounceCount: number;       // 弹射次数（0 = 不弹射）
    splitCount: number;        // 分裂数量（0 = 不分裂）
    splitDamageRatio: number;  // 分裂箭伤害比例（如 0.4 = 40%）
    homingStrength: number;    // 追踪强度（保留，当前未在飞行中使用）
    damageRatio: number;       // 基础伤害比例（1 = 全额）
}
```

**行为优先级**：`穿透 → 弹射 → 分裂`

## 五、飞行 —— ArrowProjectile

### 5.1 生命周期

```
ProjectilePool.acquire() → arrow.init() → update() × N → 命中/出屏 → ProjectilePool.release()
```

### 5.2 init 方法

```typescript
arrow.init(
    curve,              // QuadBezier 曲线
    duration,           // 飞行时长
    hasTarget,          // 是否锁定目标
    target,             // 目标敌人（可 null）
    arrowWidth,         // 贴图宽度
    arrowHeight,        // 贴图高度
    arrowTexture,       // 贴图路径
    projConfig,         // ProjectileConfig 快照
    onHit,              // 命中回调
);
```

### 5.3 飞行逻辑（update）

**阶段一：曲线飞行**

```
每帧：
  elapsed += dt
  t = elapsed / duration
  curve.getPoint(t, pos)     → 更新位置
  curve.getTangent(t, tan)   → 更新旋转
  t >= 1 → _onArrival()
```

**阶段二：穿透直线**（`pierceCount > 0` 时进入）

```
沿最终切线方向直线飞行
逐帧碰撞检测（距离 < 40px）
每命中一个敌人：
  pierceCount--
  damageRatio *= 0.7          // 穿透衰减
  触发 onHit
```

### 5.4 到达终点后的分支

```
_onArrival()
  ├── 有目标 → 触发 onHit
  ├── pierceCount > 0 → _enterPierce()（直线穿透）
  └── _postPierceCheck()
       ├── bounceCount > 0 → _retarget()（弹射到最近敌人）
       ├── splitCount > 0  → _split()（生成子箭矢）
       └── 回收
```

### 5.5 弹射

找最近未命中敌人，重建曲线继续飞行：

```typescript
private _retarget(from: Vec3, next: EnemyControl) {
    const end = next.node.worldPosition.clone();
    const dist = Vec3.distance(from, end);
    const facing = end.x > from.x ? 1 : -1;
    this._curve = QuadBezier.createArc(from, end, dist * arrowArcRatio * facing);
    this._duration = dist / arrowSpeed;
    this._elapsed = 0;
    this._hasTarget = true;
    this._target = next;
}
```

### 5.6 分裂

生成多支子箭矢，各自锁定不同敌人，伤害按 `splitDamageRatio` 衰减：

```typescript
const splitCfg: ProjectileConfig = {
    pierceCount: 0, bounceCount: 0, splitCount: 0,
    splitDamageRatio: 0, homingStrength: 0,
    damageRatio: this._damageRatio * (this._config?.splitDamageRatio ?? 0.4),
};
```

子箭矢**不会**再分裂/弹射，避免指数爆炸。

## 六、对象池 —— ProjectilePool

```typescript
export class ProjectilePool {
    private static _pool: Node[] = [];
    private static _parent: Node | null = null;

    static init(parent: Node) {
        const layer = new Node('ProjectileLayer');
        parent.addChild(layer);
        this._parent = layer;
    }

    static acquire(): Node {
        const node = this._pool.pop() ?? new Node('Arrow');
        node.active = true;
        this._parent!.addChild(node);
        return node;
    }

    static release(node: Node) {
        if (!node.isValid) return;
        node.active = false;
        node.removeFromParent();
        this._pool.push(node);
    }
}
```

**使用要点**：
- `GameLoop.onLoad` 中 `ProjectilePool.init(this.node)` → 在 GameLoop 下挂 ProjectileLayer
- `acquire` 优先从池中取，池空时 `new Node`
- `release` 时 `active = false` + 移出场景树，但**不 destroy**

## 七、属性与弹道的关联

| 属性 ID | 影响环节 | 说明 |
|---------|---------|------|
| `ExtraProjectiles` | ShootResolver | 额外箭矢数量（+1 = 双箭） |
| `SpreadAngle` | ShootResolver | 无目标时多箭扇形角度（度） |
| `ArrowSpeed` | ShootResolver | 飞行速度 → 飞行时长 |
| `PierceCount` | ProjectileConfig → ArrowProjectile | 穿透次数 |
| `BounceCount` | ProjectileConfig → ArrowProjectile | 弹射次数 |
| `HomingStrength` | ProjectileConfig（保留） | 追踪强度 |
| `AttackRange` | PlayerControl 索敌 | 最大锁敌距离 |
| `AttackSpeed` | PlayerShootState 冷却 | 射击间隔 |
