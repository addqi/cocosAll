# Cocos Creator 3.8 属性系统教程（修饰器版）

> 适合小白的逐步教程 · TypeScript · 完整可运行代码 · 可应对复杂技能效果

---

## 一、为什么需要修饰器版属性系统？

### 1.1 简单游戏的数值系统

在技能效果不复杂的游戏里，传统数值系统通常够用：
- 基础数值 + 各种加成 = 最终数值
- 所有加成加在一起即可

### 1.2 遇到的两个痛点

**效果一：按生命值损失的暴击加成**

> 英雄每损失 10% 生命值，增加 5% 暴击概率。

传统做法会遇到：
- 删掉旧暴击 buff，再加新的暴击 buff
- 或减去旧加成，再加新加成

都不够直接，理想做法是：**用新暴击值覆盖旧的，然后重新计算**。

**效果二：短暂屏蔽敌方增益**

> 技能效果：屏蔽敌方所有数值增益数秒（不清除，只是暂时不生效）。

传统系统把所有加成「混在一起」，很难单独控制某一项增益的开关，实现会很别扭。

### 1.3 核心需求

- **每个数值变动都要单独可控**：可以单独添加、删除、替换、开关
- **支持「覆盖式刷新」**：直接替换某个 buff 的数值即可，不用先减再加
- **支持「屏蔽」**：某些增益暂时不参与计算，但不删除
- **支持多种运算方式**：加法、乘法、覆盖、限制范围

## 二、整体结构介绍

本系统由 **4 个核心模块** 构成：

```
IProperty<T>           —— 属性接口（统一行为）
BaseValueProperty       —— 基础属性（可被 Modifier 修改）
ComputeValueProperty    —— 计算属性（依赖其它属性动态计算）
PropertyManager         —— 属性管理器（统一管理所有属性）
IPropertyModifier<T>    —— 修饰器（加法、乘法、覆盖、限制等）
```

### 2.1 最终效果

类似 Unity Buff 系统的计算方式：

```
最终移动速度 = (基础速度 + buff加速 + 装备加速) × buff倍速 × 装备倍速

加上 Override/Clamp 后更强大
```

### 2.2 模块关系图

```
                    PropertyManager
                          │
        ┌─────────────────┼─────────────────┐
        ▼                 ▼                 ▼
BaseValueProperty   BaseValueProperty   ComputeValueProperty
(基础值+Modifier)    (基础值+Modifier)    (依赖其它属性计算)
        │                 │                 │
        └────────┬────────┴────────┬────────┘
                 ▼                 ▼
            MoveSpeed-Value   MoveSpeed (最终)
```

## 三、修饰器类型（Modifier Merge Type）

### 3.1 四种修饰器

| 类型 | 含义 | 示例 |
|------|------|------|
| **Additive** | 加法 | +10、+50 |
| **Multiplicative** | 乘法 | ×1.2（+20%）、×0.8（-20%） |
| **Override** | 覆盖 | 直接覆盖为指定值，多选一取最高优先级 |
| **Clamp** | 限制范围 | 限制在 [min, max] 之间 |

### 3.2 为什么需要 Override

- **覆盖式刷新**：英雄按生命损失加暴击时，只需改一个修饰器的 `value`，或直接用 Override 覆盖
- 不用先删再加，逻辑更简洁

### 3.3 为什么需要 Clamp

- 移动速度封顶、最低保底
- 暴击率限制在 0%~100%
- 生命值不能为负

### 3.4 计算顺序（重要）

```
1. 先处理 Additive（加法）
2. 再处理 Multiplicative（乘法）
3. 再处理 Clamp（限制范围）
4. 最后 Override（若有则覆盖为指定值）
```

## 四、第一步：定义修饰器类型枚举

在 `properties/` 目录下新建 `ModifierType.ts`：

```ts
/**
 * 属性修饰器类型
 * 定义修饰器对数值的影响方式
 */
export enum EModifierMergeType {
    /** 加法，例如：+10 */
    Additive,
    /** 乘法，例如：×1.2 */
    Multiplicative,
    /** 覆盖，多个覆盖时取优先级最高的 */
    Override,
    /** 限制范围（Clamp） */
    Clamp,
}
```

---

## 五、第二步：实现各类型修饰器

### 5.1 修饰器接口

所有修饰器必须实现此接口，提供 `mergeType` 和 `priority`（优先级，数值越大越后算）：

```ts
export interface IPropertyModifier<T> {
    priority: number;
    mergeType: EModifierMergeType;
}
```

### 5.2 加法型 Modifier

```ts
export class PropertyAddModifier implements IPropertyModifier<number> {
    mergeType = EModifierMergeType.Additive;
    constructor(
        public value: number,   // 加多少
        public priority: number = 0
    ) {}
}
```

### 5.3 乘法型 Modifier

```ts
export class PropertyMulModifier implements IPropertyModifier<number> {
    mergeType = EModifierMergeType.Multiplicative;
    constructor(
        public value: number,   // 乘以多少，1.2 = +20%
        public priority: number = 0
    ) {}
}
```

### 5.4 覆盖型 Modifier

```ts
export class PropertyOverrideModifier implements IPropertyModifier<number> {
    mergeType = EModifierMergeType.Override;
    constructor(
        public value: number,   // 直接覆盖为此值
        public priority: number = 0
    ) {}
}
```

### 5.5 限制范围型 Modifier

```ts
export class PropertyClampModifier implements IPropertyModifier<number> {
    mergeType = EModifierMergeType.Clamp;
    constructor(
        public min: number,
        public max: number,
        public priority: number = 0
    ) {}
}
```

## 六、第三步：属性接口 IProperty<T>

### 6.1 统一行为

所有属性（基础型、计算型）都实现此接口，便于 PropertyManager 统一管理：

```ts
export interface IProperty<T> {
    propertyId: string;
    getValue(): T;
    makeDirty(): void;
    addModifier(mod: IPropertyModifier<T>): void;
    removeModifier(mod: IPropertyModifier<T>): void;
}
```

### 6.2 说明

- `propertyId`：唯一标识，用于在 PropertyManager 中查找
- `getValue()`：获取当前最终值（会自动触发重新计算）
- `makeDirty()`：标记为脏，下次 getValue 时重新计算
- `addModifier` / `removeModifier`：添加或移除修饰器（计算型属性可空实现）

---

## 七、第四步：BaseValueProperty —— 基础属性

### 7.1 职责

- 保存基础值 `baseValue`
- 管理多个修饰器（Add / Mul / Override / Clamp）
- 按顺序计算，得到最终值
- 使用 `isDirty` + `cachedValue` 做缓存，避免重复计算

### 7.2 核心逻辑（getValue 流程）

```ts
getValue(): number {
    if (!this.isDirty) return this.cachedValue;  // 缓存命中

    let v = this.baseValue;

    // 1. 加法
    Additive 修饰器 → v += value

    // 2. 乘法
    Multiplicative 修饰器 → v *= value

    // 3. Clamp
    Clamp 修饰器 → v = Math.min(Math.max(v, min), max)

    // 4. Override（若有，取优先级最高者覆盖）
    Override 修饰器 → v = 最高优先级的 value

    this.cachedValue = v;
    this.isDirty = false;
    return v;
}
```

### 7.3 添加/移除修饰器

- `addModifier(mod)`：push 进数组，`makeDirty()`
- `removeModifier(mod)`：从数组移除，`makeDirty()`

每次增删后，下次 `getValue()` 会重新计算。

---

## 八、第五步：ComputeValueProperty —— 计算属性

### 8.1 职责

- 不存修饰器，只通过**计算函数**动态得到结果
- 依赖 PropertyManager 中的其它属性
- 用于「多个子属性组合成最终值」的场景

### 8.2 示例

```ts
// 最终移动速度 = Value × Buff倍速 × 其它倍速
new ComputeValueProperty(() =>
    pm.get("MoveSpeed-Value") *
    pm.get("MoveSpeed-Mul-Buff") *
    pm.get("MoveSpeed-Mul-Other"),
    "MoveSpeed"
)
```

### 8.3 说明

- `addModifier` / `removeModifier` 可空实现（计算属性不直接挂修饰器）
- `makeDirty()` 用于在依赖变化时触发重新计算
- 依赖的 Base 属性变化时，需要调用此属性的 `makeDirty()`

---

## 九、第六步：PropertyManager —— 属性管理器

### 9.1 职责

- 用 Map 存储所有属性，key 为 `propertyId`
- `register(prop)`：注册属性
- `get(id)`：获取属性最终值（内部调 `getValue()`）
- `getProperty(id)`：获取属性对象（用于添加 Modifier）

### 9.2 使用方式

```ts
const pm = new PropertyManager();
pm.register(new BaseValueProperty(100, "MoveSpeed-Value-Config"));

// 需要加 buff 时
pm.getProperty("MoveSpeed-Value-Buff")?.addModifier(new PropertyAddModifier(50));

// 读取最终值
console.log(pm.get("MoveSpeed"));
```

## 十、完整示例：MoveSpeed 属性配置

### 10.1 设计思路

```
MoveSpeed-Value = Config + Buff + Other（三个加法和）
MoveSpeed = Value × MulBuff × MulOther（三个乘法积）
```

### 10.2 注册步骤

```ts
const pm = new PropertyManager();

// 1. 基础 Value 来源（三个加数）
pm.register(new BaseValueProperty(100, "MoveSpeed-Value-Config"));
pm.register(new BaseValueProperty(0, "MoveSpeed-Value-Buff"));
pm.register(new BaseValueProperty(0, "MoveSpeed-Value-Other"));

// 2. Value = 三者之和
pm.register(new ComputeValueProperty(() =>
    pm.get("MoveSpeed-Value-Config") +
    pm.get("MoveSpeed-Value-Buff") +
    pm.get("MoveSpeed-Value-Other"),
    "MoveSpeed-Value"
));

// 3. 乘法系数（默认 1）
pm.register(new BaseValueProperty(1, "MoveSpeed-Mul-Buff"));
pm.register(new BaseValueProperty(1, "MoveSpeed-Mul-Other"));

// 4. 最终 MoveSpeed
pm.register(new ComputeValueProperty(() =>
    pm.get("MoveSpeed-Value") *
    pm.get("MoveSpeed-Mul-Buff") *
    pm.get("MoveSpeed-Mul-Other"),
    "MoveSpeed"
));
```

### 10.3 使用示例

**加法 Buff：**
```ts
const buff = new PropertyAddModifier(50);
pm.getProperty("MoveSpeed-Value-Buff")?.addModifier(buff);
// MoveSpeed = (100 + 50) × 1 × 1 = 150
```

**乘法 Buff：**
```ts
const speedUp = new PropertyMulModifier(1.5);
pm.getProperty("MoveSpeed-Mul-Buff")?.addModifier(speedUp);
// MoveSpeed = 150 × 1.5 × 1 = 225
```

**Clamp 封顶：**
```ts
pm.getProperty("MoveSpeed-Value")?.addModifier(
    new PropertyClampModifier(0, 300)
);
```

**Override 强制覆盖：**
```ts
pm.getProperty("MoveSpeed-Value-Buff")?.addModifier(
    new PropertyOverrideModifier(80)
);
```

## 十一、效果一：用 Override 实现按生命值的暴击

### 11.1 需求回顾

> 英雄每损失 10% 生命值，增加 5% 暴击概率。

### 11.2 实现思路

1. 暴击率使用 `BaseValueProperty`，挂一个 `PropertyOverrideModifier`
2. 监听生命变化，计算「损失百分比」
3. 根据损失算出新暴击（如损失 30% → +15%）
4. **直接改这个 Override 修饰器的 value**，然后 `makeDirty()` 或下次 `getValue()` 自动重算

### 11.3 关键点

- 保存修饰器引用：`critModifier = new PropertyOverrideModifier(0)`
- 生命变化时：`critModifier.value = 新值`
- 调用 `getProperty("CritRate")?.makeDirty()` 触发重算

---

## 十二、效果二：用 Enable 实现「屏蔽」增益

### 12.1 扩展修饰器

在修饰器基类或各实现中增加 `enabled: boolean = true`。

### 12.2 修改 BaseValueProperty 的 getValue

在累加/累乘时，只处理 `modifier.enabled === true` 的修饰器。

### 12.3 实现屏蔽

1. 技能生效：把需要屏蔽的修饰器 `enabled = false`
2. 调用 `makeDirty()`
3. 数秒后：`enabled = true`，再 `makeDirty()`

## 十三、目录结构与 Cocos 3.8 使用建议

### 13.1 推荐目录

```
assets/script/baseSystem/properties/
├── ModifierType.ts        # EModifierMergeType
├── Modifier.ts            # 各类型 Modifier 实现
├── IProperty.ts           # IProperty 接口

├── ComputeValueProperty.ts# 计算属性
├── PropertyManager.ts     # 属性管理器
└── index.ts               # 统一导出
```

### 13.2 与 Component 配合

- 在角色 Component 的 `start()` 中创建 `PropertyManager`，注册所有属性
- 技能/Buff 系统通过 `getProperty()` 拿到对应属性，添加/移除修饰器
- 需要刷新 UI 时，在 `getValue()` 后发事件或调 UI 更新

## 十四、小结

| 概念 | 作用 |
|------|------|
| **BaseValueProperty** | 基础值 + Modifier，支持 Add/Mul/Override/Clamp |
| **ComputeValueProperty** | 依赖其它属性，用函数动态计算 |
| **PropertyManager** | 统一注册、查找、获取属性 |
| **Additive** | 加法，如 +50 |
| **Multiplicative** | 乘法，如 ×1.2 |
| **Override** | 覆盖，实现「刷新」效果 |
| **Clamp** | 限制范围，封顶/保底 |

核心思路：**用修饰器把每个数值变动单独管理**，配合 Base + Compute 的分层设计，即可应对复杂技能效果。
