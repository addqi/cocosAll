# 技能系统教程

> 基于 Ecslearn 项目 `game/skill` 模块，适用于 Cocos Creator 3.x + TypeScript。

---

## 一、概述

### 1.1 技能系统定位

技能系统**只管主动技能**——玩家手动释放、有冷却时间的技能。被动效果（如穿透+1、攻速+30%）走**升级系统**。

```
升级系统（被动）  →  UpgradeManager  → Buff / HitEffect / ShootPolicy
技能系统（主动）  →  SkillSystem     → IActiveSkill.execute()
```

### 1.2 设计理念

| 原则 | 说明 |
|------|------|
| **接口驱动** | 所有技能实现 `IActiveSkill`，SkillSystem 不关心具体实现 |
| **CD 自治** | 每个技能自己管理冷却倒计时 |
| **上下文注入** | 释放时传入 `SkillContext`，技能从中获取属性/战斗/Buff 等引用 |
| **零全局依赖** | 技能不持有 PlayerControl 引用，只通过 SkillContext 交互 |

### 1.3 整体架构

```
PlayerControl
    ├── SkillSystem
    │   ├── ArrowStormSkill  (id: "arrow-storm")
    │   └── DashShotSkill    (id: "dash-shot")
    │
    └── 每帧：skillSystem.tick(dt)
        按键：skillSystem.tryUse("arrow-storm", ctx)
```

## 二、SkillContext —— 技能上下文

```typescript
export interface SkillContext {
    playerProp: PlayerProperty;       // 玩家属性（读攻击/额外弹道等）
    playerCombat: PlayerCombat;       // 玩家战斗（回血等）
    playerNode: Node;                 // 玩家节点（位置/方向）
    hitEffectMgr: HitEffectMgr;       // 命中效果管理器
    buffMgr: EntityBuffMgr;           // Buff 管理器
    buffOwner: IBuffOwner;            // Buff 持有者
    mouseWorldPos: Vec3;              // 鼠标世界坐标（技能释放位置）
    setShootPolicy(policy: IShootPolicy): void;  // 切换射击策略
}
```

`PlayerControl` 在释放技能时构造 `SkillContext`，把自身持有的各模块引用传入。技能实现通过此上下文与外部交互，**无需知道 PlayerControl 的存在**。

## 三、IActiveSkill —— 主动技能接口

```typescript
export interface IActiveSkill {
    readonly id: string;           // 唯一标识
    readonly name: string;         // 显示名
    readonly maxCooldown: number;  // 最大冷却时间（秒）
    currentCd: number;             // 当前冷却倒计时
    level: number;                 // 技能等级

    canUse(): boolean;             // 是否可释放
    execute(ctx: SkillContext): void;  // 释放逻辑
    tick(dt: number): void;        // 每帧更新（CD 递减等）
    dispose?(ctx: SkillContext): void; // 卸载时清理
}
```

**生命周期**：

```
equip → tick × N → canUse? → execute → tick × N → ... → unequip → dispose
```

## 四、SkillSystem —— 技能管理器

```typescript
export class SkillSystem {
    private _skills = new Map<string, IActiveSkill>();

    equip(skill: IActiveSkill): void {
        this._skills.set(skill.id, skill);
    }

    unequip(id: string, ctx: SkillContext): void {
        const skill = this._skills.get(id);
        if (!skill) return;
        skill.dispose?.(ctx);
        this._skills.delete(id);
    }

    tick(dt: number): void {
        for (const skill of this._skills.values()) {
            skill.tick(dt);
        }
    }

    tryUse(id: string, ctx: SkillContext): boolean {
        const skill = this._skills.get(id);
        if (!skill?.canUse()) return false;
        skill.execute(ctx);
        return true;
    }

    // 辅助方法
    has(id: string): boolean;
    getSkill(id: string): IActiveSkill | null;
    get allSkills(): IActiveSkill[];
    get count(): number;
}
```

**使用要点**：
- `tick(dt)` 在 `PlayerControl.update` 中每帧调用，驱动所有技能 CD
- `tryUse` 自动检查 `canUse()` → 安全调用 `execute()`
- `unequip` 会调 `dispose()` 清理技能残留效果（如 DashShot 的攻速 Buff）

## 五、实战：箭雨倾泻（ArrowStormSkill）

### 5.1 效果描述

在鼠标位置降下大量箭矢，散布在一定范围内，落点附近的敌人受到伤害。

### 5.2 关键参数

```typescript
constructor(cfg?: Partial<{
    cooldown: number;        // CD（默认 8 秒）
    arrowMultiplier: number; // 箭矢倍率（默认 3×）
    skyHeight: number;       // 天空起始高度（默认 500）
    scatter: number;         // 散布范围（默认 200）
}>)
```

### 5.3 箭矢数量计算

```typescript
const extraProj   = Math.floor(ctx.playerProp.getValue(EPropertyId.ExtraProjectiles));
const totalArrows = this.level * (1 + extraProj) * this._arrowMultiplier;
```

- **与属性联动**：额外弹道越多，箭雨越密集
- **等级缩放**：`level × 基数` → 升级后效果增强

### 5.4 弹道生成

每支箭矢独立：
1. 起点 = 鼠标位置上方 `skyHeight` 处 + 随机水平偏移
2. 终点 = 鼠标位置附近随机散布
3. 落点 120px 内有敌人 → 自动吸附命中
4. 贝塞尔曲线 + 随机飞行时长 → 视觉效果自然

```typescript
for (let i = 0; i < totalArrows; i++) {
    const start = new Vec3(center.x + offX, center.y + this._skyHeight, 0);
    const landing = new Vec3(endX, endY, 0);
    const near = this._findNearLanding(landing, 120);
    const end = near ? near.node.worldPosition.clone() : landing;

    const curve = QuadBezier.createArc(start, end, arc);
    const dur = dist / arrowSpeed * (0.8 + Math.random() * 0.4);

    const arrowNode = ProjectilePool.acquire();
    let arrow = arrowNode.getComponent(ArrowProjectile);
    if (!arrow) arrow = arrowNode.addComponent(ArrowProjectile);
    arrow.init(curve, dur, !!near, near, /* ... */, projConfig, onHit);
}
```

### 5.5 配置特殊性

箭雨的 `ProjectileConfig` 强制清零穿透/弹射/分裂：

```typescript
projConfig.pierceCount = 0;
projConfig.bounceCount = 0;
projConfig.splitCount  = 0;
```

→ 每支箭只命中一个敌人，避免箭雨 + 穿透的指数爆炸。

## 六、实战：闪身射击（DashShotSkill）

### 6.1 效果描述

激活后持续一段时间，期间攻速大幅提升，自动射击等级提升到 Lv3（始终射击）。

### 6.2 关键参数

```typescript
constructor(cfg?: Partial<{
    cooldown: number;      // CD（默认 10 秒）
    duration: number;      // 持续时间（默认 5 秒）
    atkSpeedBoost: number; // 攻速加成倍率（默认 1.0 → +100%）
}>)
```

### 6.3 释放逻辑

```typescript
execute(ctx: SkillContext): void {
    this.currentCd = this.maxCooldown;
    this._effectTimer = this._duration;
    this._ctx = ctx;

    // 添加攻速 Buff
    const buff: BuffData = {
        id: 9001,
        name: '闪身射击',
        duration: this._duration,
        effectClass: 'SimpleAttrBuffEffect',
        targetAttr: 'AttackSpeed-Mul-Buff',
        valuePerStack: this._atkSpeedBoost,
    };
    ctx.buffMgr.addBuff(buff, ctx.buffOwner);

    // 切换到最高级自动射击
    ctx.setShootPolicy(new AutoShoot(3));
}
```

### 6.4 效果结束

```typescript
private _endEffect(): void {
    this._effectTimer = 0;
    if (!this._ctx) return;
    this._ctx.buffMgr.removeBuff(9001);           // 移除攻速 Buff
    this._ctx.setShootPolicy(new HoldToShoot());   // 恢复默认射击策略
    this._ctx = null;
}
```

**技能系统 × Buff 系统 × 射击策略** 三者联动：释放时加 Buff + 切策略，结束时反向回退。

### 6.5 tick 中的双计时

```typescript
tick(dt: number): void {
    if (this.currentCd > 0) this.currentCd = Math.max(0, this.currentCd - dt);
    if (this._effectTimer > 0) {
        this._effectTimer -= dt;
        if (this._effectTimer <= 0) this._endEffect();
    }
}
```

- `currentCd`：技能冷却（决定何时可再次释放）
- `_effectTimer`：效果持续时间（决定何时结束 Buff + 策略）

| 类名 | 继承 | 说明 |

| `SimpleAttrBuffEffect` | `BuffEffectBase` | 通用属性 Buff（通过 `targetAttr` + `valuePerStack` 配置） |
| `AttackBoostEffect` | `SimpleAttrBuffEffect` | 攻击加成（薄封装） |
| `CritBoostEffect` | `SimpleAttrBuffEffect` | 暴击加成 |
| `DefenseReduceEffect` | `SimpleAttrBuffEffect` | 防御削减 |
| `LifestealEffect` | `SimpleAttrBuffEffect` | 吸血率加成 |
| `AddHpBuffEffect` | `BuffEffectBase` | 直接加血 |
| `RegenEffect` | `BuffEffectBase` | 持续回复（onTick） |

`game/index.ts` 中 `import './skill'` → 触发 `skill/index.ts` 中 `import './effects'` → 注册所有效果类到 `BuffFactory`。

## 八、新建技能完整指南

### 8.1 步骤清单

1. 在 `game/skill/` 下新建 `XxxSkill.ts`
2. 实现 `IActiveSkill` 接口（id、name、maxCooldown、canUse、execute、tick）
3. 在 `skill/index.ts` 中导出
4. 在 `PlayerControl` 中 `skillSystem.equip(new XxxSkill())`
5. 绑定按键触发 `skillSystem.tryUse('xxx', ctx)`

### 8.2 模板

```typescript
import type { IActiveSkill, SkillContext } from './SkillTypes';

export class XxxSkill implements IActiveSkill {
    readonly id = 'xxx';
    readonly name = '技能名';
    readonly maxCooldown = 12;
    currentCd = 0;
    level = 1;

    canUse(): boolean {
        return this.currentCd <= 0;
    }

    tick(dt: number): void {
        if (this.currentCd > 0) this.currentCd = Math.max(0, this.currentCd - dt);
    }

    execute(ctx: SkillContext): void {
        this.currentCd = this.maxCooldown;
        // 技能逻辑
    }

    dispose(ctx: SkillContext): void {
        // 清理残留 Buff / 策略
    }
}
```

## 十、常见问题

### Q1：技能和升级的区别？

| 维度 | 技能（SkillSystem） | 升级（UpgradeManager） |
|------|---------------------|----------------------|
| 触发 | 手动按键 | 选择后永久生效 |
| CD | 有 | 无 |
| 可移除 | equip / unequip | apply / remove |
| 效果类型 | 任意（弹道/Buff/策略） | buff / hit_effect / shoot_policy |

### Q2：技能内如何安全访问玩家状态？

始终通过 `SkillContext` 的引用，**不要**直接 import `PlayerControl`。这样技能可以独立测试，也方便未来给敌人加主动技能。

### Q3：DashShotSkill 的 Buff 到期和 _effectTimer 到期哪个先触发？

两者时长相同，但 `_effectTimer` 在 `tick` 中手动递减，`Buff` 由 `EntityBuffMgr.update` 递减。正常情况下几乎同时到期；`_endEffect` 中会主动 `removeBuff`，即使 Buff 先自然过期也不影响。
