# 命中效果系统教程

> 基于 Ecslearn 项目 `game/entity/HitEffectMgr` + `game/hitEffects` + `baseSystem/hitEffect` 模块，适用于 Cocos Creator 3.x + TypeScript。

---

## 一、概述

### 1.1 什么是命中效果

箭矢命中敌人后，需要依次执行一系列效果：**算伤害 → 暴击判定 → 吸血 → 施加灼烧 → 击退 → 闪电链 …**

这些效果来源不同（基础伤害是固有的，灼烧来自升级选择，闪电链来自进化组合），且顺序有优先级要求。命中效果系统就是管理这条**可扩展的命中管线**。

### 1.2 设计理念

| 原则 | 说明 |
|------|------|
| **优先级管线** | 所有命中效果按 `priority` 排序执行，低值先执行 |
| **共享上下文** | 一个 `GameHitContext` 贯穿整条管线，前序效果的计算结果传递给后续 |
| **工厂注册** | 命中效果类通过 `@hitEffect` 装饰器或 `HitEffectFactory.register` 注册 |
| **热插拔** | 通过 `HitEffectMgr.add/remove` 动态增删，升级系统和技能可随时改变管线 |

### 1.3 整体架构

```
ArrowProjectile._onArrival()
    ↓
onHit(target, damageRatio)
    ↓
createHitContext(attackerProp, attackerCombat, targetCombat, targetBuffMgr, targetBuffOwner)
    ↓
hitEffectMgr.execute(ctx)
    ↓
┌──────────────────────────────────────────────────┐
│ DamageHitEffect     (priority: 10)  → 计算伤害    │
│ CritBonusDamageEffect (priority: 15) → 暴击加伤   │
│ LifestealHitEffect  (priority: 20)  → 吸血回复    │
│ BurnOnHitEffect     (priority: 50)  → 施加灼烧    │
│ FrostOnHitEffect    (priority: 50)  → 施加冻伤    │
│ KnockbackEffect     (priority: 60)  → 击退位移    │
│ ChainLightningEffect(priority: 80)  → 闪电链     │
└──────────────────────────────────────────────────┘
```

## 二、GameHitContext —— 共享上下文

### 2.1 数据结构

```typescript
export interface GameHitContext {
    // ── 参与者 ──
    attackerProp: PlayerProperty;     // 攻击方属性
    attackerCombat: PlayerCombat;     // 攻击方战斗
    targetCombat: EnemyCombat;        // 被击方战斗
    targetBuffMgr: EntityBuffMgr;     // 被击方 Buff 管理器
    targetBuffOwner: IBuffOwner;      // 被击方 Buff 持有者

    // ── 空间信息 ──
    targetNode: Node | null;          // 被击节点（击退方向计算用）
    hitOriginPos: Vec3 | null;        // 攻击发起点

    // ── 伤害链 ──
    baseDamage: number;               // 基础伤害（DamageHitEffect 写入）
    rawDamage: number;                // 暴击前伤害
    finalDamage: number;              // 最终伤害（扣血用这个）
    isCrit: boolean;                  // 是否暴击
    critMultiplier: number;           // 暴击倍率
    totalHealed: number;              // 本次命中总回血量
    damageRatio: number;              // 伤害比例（穿透衰减、分裂衰减等）
}
```

### 2.2 创建辅助函数

```typescript
export function createHitContext(
    attackerProp: PlayerProperty,
    attackerCombat: PlayerCombat,
    targetCombat: EnemyCombat,
    targetBuffMgr: EntityBuffMgr,
    targetBuffOwner: IBuffOwner,
): GameHitContext {
    return {
        attackerProp, attackerCombat,
        targetCombat, targetBuffMgr, targetBuffOwner,
        targetNode: null,
        hitOriginPos: null,
        baseDamage: 0, rawDamage: 0, finalDamage: 0,
        isCrit: false, critMultiplier: 1,
        totalHealed: 0, damageRatio: 1,
    };
}
```

- 所有数值字段初始为 0 / 1
- `damageRatio` 由弹道阶段写入（穿透衰减 0.7、分裂衰减 0.4 等）
- 管线中的效果依次修改这些字段

## 四、HitEffectMgr —— 命中效果管理器

```typescript
export class HitEffectMgr {
    private _effects = new Map<string, HitEffectBase>();
    private _sorted: HitEffectBase[] = [];
    private _dirty = false;

    add(data: HitEffectData): boolean {
        if (this._effects.has(data.id)) return false;
        const effect = HitEffectFactory.create(data);
        if (!effect) return false;
        this._effects.set(data.id, effect);
        this._dirty = true;
        return true;
    }

    remove(id: string): boolean {
        const ok = this._effects.delete(id);
        if (ok) this._dirty = true;
        return ok;
    }

    execute(ctx: GameHitContext): void {
        if (this._dirty) {
            this._sorted = [...this._effects.values()].sort(
                (a, b) => (a.data.priority ?? 0) - (b.data.priority ?? 0),
            );
            this._dirty = false;
        }
        for (const eff of this._sorted) {
            eff.onHit(ctx);
        }
    }
}
```

**关键设计**：
- **惰性排序**：只在增删后标记 `_dirty`，下次 `execute` 时重排
- **顺序保证**：`priority` 小的先执行 → DamageHitEffect(10) 先算伤害，LifestealHitEffect(20) 再根据伤害吸血

## 五、已有命中效果一览

| 类名 | priority | 说明 |
|------|----------|------|
| `DamageHitEffect` | 10 | 读攻击/防御计算 `baseDamage`，暴击判定写 `isCrit` / `critMultiplier`，算 `finalDamage`，调 `targetCombat.takeDamage` |
| `CritBonusDamageEffect` | 15 | 暴击时追加额外伤害 |
| `LifestealHitEffect` | 20 | 读 `finalDamage` × 吸血比例 → 攻击者回血 |
| `LifeOnHitEffect` | 25 | 固定回血（不依赖伤害） |
| `BurnOnHitEffect` | 50 | 对目标施加灼烧 DOT Buff |
| `FrostOnHitEffect` | 50 | 对目标施加冻伤减速 Buff |
| `KnockbackEffect` | 60 | 根据 `hitOriginPos` → `targetNode` 方向击退 |
| `ChainLightningEffect` | 80 | 概率触发闪电链，跳跃多个敌人（伤害逐级衰减） |

## 六、伤害计算流程

以 `DamageHitEffect` 为核心的标准流程：

```
1. baseDamage = attackerProp.getValue('Attack') × damageRatio
2. rawDamage  = baseDamage - targetDefense（≥ 1）
3. 暴击判定：rand() < critRate → isCrit = true
4. finalDamage = isCrit ? rawDamage × critMultiplier : rawDamage
5. targetCombat.takeDamage(finalDamage)
```

后续效果读取 `ctx.finalDamage` 进行吸血等计算。

## 七、新建命中效果完整指南

### 7.1 步骤清单

1. 在 `game/hitEffects/` 下新建 `XxxEffect.ts`
2. 继承 `HitEffectBase`，实现 `onHit(ctx: GameHitContext)`
3. 文件末尾 `HitEffectFactory.register('XxxEffect', XxxEffect)`
4. 在 `hitEffects/index.ts` 中 `import './XxxEffect'`

### 7.2 示例：击杀回血效果

```typescript
import { HitEffectBase } from '../../baseSystem/hitEffect';
import { HitEffectFactory } from '../../baseSystem/hitEffect';
import type { GameHitContext } from './types';

export class KillHealEffect extends HitEffectBase {
    onHit(ctx: GameHitContext): void {
        if (ctx.targetCombat.isDead) {
            const healAmount = this.data.healOnKill ?? 10;
            ctx.attackerCombat.heal(healAmount);
            ctx.totalHealed += healAmount;
        }
    }
}

HitEffectFactory.register('KillHealEffect', KillHealEffect);
```

配置数据示例：

```typescript
{
    id: 'upg-kill-heal',
    effectClass: 'KillHealEffect',
    priority: 90,           // 在伤害之后执行，才能判断 isDead
    healOnKill: 15,
}
```

### 7.3 priority 选择建议

| 范围 | 用途 |
|------|------|
| 0–19 | 伤害计算、防御减免 |
| 20–39 | 吸血、回血等依赖伤害的效果 |
| 40–59 | 状态施加（灼烧、减速、中毒） |
| 60–79 | 位移效果（击退、拉拽） |
| 80–99 | 连锁效果（闪电链、爆炸扩散） |

## 八、与其他系统的交互

### 8.1 弹道系统

`ArrowProjectile` 命中时构造 `GameHitContext` 并调用 `hitEffectMgr.execute(ctx)`：

```typescript
const onHit = (target: EnemyControl, damageRatio: number) => {
    const hitCtx = createHitContext(
        ctx.playerProp, ctx.playerCombat,
        target.combat, target.buffMgr, target.buffOwner,
    );
    hitCtx.damageRatio = damageRatio;     // 穿透/分裂衰减
    hitCtx.targetNode = target.node;
    ctx.hitEffectMgr.execute(hitCtx);
};
```

### 8.2 升级系统

`UpgradeManager.apply()` 中 `type: 'hit_effect'` 直接调 `hitEffectMgr.add(data)`；移除升级时调 `hitEffectMgr.remove(id)`。

### 8.3 Buff 系统

命中效果可以对目标施加 Buff（如灼烧、减速），通过 `ctx.targetBuffMgr.addBuff(buffData, ctx.targetBuffOwner)` 实现。

## 十、常见问题

### Q1：命中效果和 Buff 效果有什么区别？

- **命中效果**：一次性触发，在箭矢命中瞬间执行
- **Buff 效果**：持续生效，有时长/叠加/tick 等生命周期

两者的关系：命中效果可以**施加** Buff（如灼烧 DOT），但 Buff 本身由 `EntityBuffMgr` 管理。

### Q2：如何让命中效果有概率触发？

在 `onHit()` 开头判断随机数：

```typescript
onHit(ctx: GameHitContext): void {
    if (Math.random() > (this.data.chance ?? 1)) return;
    // 触发逻辑
}
```

### Q3：多个相同类型的命中效果会冲突吗？

`HitEffectMgr` 用 `id` 去重——同 `id` 不会重复添加。不同 `id` 但同类的效果会各自执行。
