# 15 · 胜利判定与通关界面

## 本节目标

**所有箭头都飞出棋盘后弹出"You Win!"提示，点击任意位置进入下一关**（暂时"下一关"就是重载本关）。

预期：

- 3 根箭头全部飞出 → HUD 下方弹出 "You Win!" 文字覆盖层。
- 玩家点击任意位置 → 重新加载关卡 → 可以再玩一遍。
- Console：
  ```
  [Arrow] 🎉 Level passed!
  ```

一句话：**胜利路径闭环**。

---

## 需求分析

胜利条件参考 G3_FBase 的 `TouchEndToCombatDoPassDispatchLogic`：

```typescript
for (const arrowEid of this.combatArrowDomain.collect()) {
    if (arrowComponent.moveMode[arrowEid] < G3_FBase.ArrowMoveMode.Start) {
        this.router.dispatchAction = ae.DispatchAction.Cancel;
        return;
    }
}
// 条件满足 → 胜利
```

翻译成我们这儿：**所有 runtime 的 mode 都 >= Start（Start 或 End）**。

### 判定时机

每帧检查一次？开销大吗？

- 每帧遍历 N 根箭头的 mode：10 次比较，完全不费。
- 每帧调一次 `checkWin()`，不影响性能。

所以**放在 `update` 末尾**。

### 防止重复触发

胜利一旦触发，就不应再触发第二次（弹窗叠加）。加一个 `gameOver` 字段：

```typescript
private gameOver = false;
```

`checkWin()` 触发后设 true。重载关卡时重置 false。

---

## 实现思路

### 胜利函数（纯函数）

```typescript
export function isLevelPassed(runtimes: readonly ArrowRuntime[]): boolean {
    return runtimes.every(rt => rt.mode >= ArrowMoveMode.Start);
}
```

注意我们**不检查 "全部 End"**——Start 状态（飞到一半）也算"已经发射过"。对应 G3_FBase 的 `< Start → cancel` 的逆。

但 Start 状态还会变成 End 或 Collide。严格点说应该是**所有 runtime 的 mode ∈ {Start, End}**。

**再想想**：Start 状态在第 10 章实现中，飞出棋盘必然切 End。不会回到 Idle/Collide/Back。所以 "mode >= Start" 等价于 "mode ∈ {Start, End}"。**简单的判断对**。

但 Collide/Back 是明显的"失败中"，不该算胜利。我们写严格一点：

```typescript
return runtimes.every(rt =>
    rt.mode === ArrowMoveMode.Start || rt.mode === ArrowMoveMode.End
);
```

### 胜利界面

最简单的 MVP：在屏幕中央画一个半透明黑色蒙层 + 一个大字 "You Win!"。

用 `Graphics` 画矩形 + `Label` 画文字。挂在 HUD 下面的另一层节点。

### 点击后的逻辑

参考 G3_FHome `FirstUiStartLogic.onExecute`：

```typescript
if (this.combatInitLogic.storage.combat.win) {
    this.combatInitLogic.storage.level += 1;
    this.combatInitLogic.storage.combat.hp = -1;
    this.combatInitLogic.storage.combat.newCombat = true;
    this.combatInitLogic.storage.combat.win = false;
}
```

胜利后 level +1。我们简化：重载本关（level 暂不存，第 18 章再加）。

## 代码实现

```typescript
export function isLevelPassed(runtimes: readonly ArrowRuntime[]): boolean {
    return runtimes.every(rt =>
        rt.mode === ArrowMoveMode.Start || rt.mode === ArrowMoveMode.End

```

### 文件 2：`game/GameOverOverlay.ts`（新增）

胜利和失败界面长得很像（半透明蒙层 + 大字 + 点击任意处），**合并成一个组件**，用参数区分文字。

```typescript
import {
    _decorator, Component, Node, UITransform, Graphics, Label, Color,
    Widget, EventTouch, Input, view,
} from 'cc';
const { ccclass } = _decorator;

export type OverlayClickHandler = () => void;

/**
 * 通用全屏覆盖层。用于胜利/失败界面。
 * 调用 show(text, onClick) 显示，点击任意处触发 onClick 后 hide。
 */
@ccclass('GameOverOverlay')
export class GameOverOverlay extends Component {
    private bg: Node | null = null;
    private label: Label | null = null;
    private onClick: OverlayClickHandler | null = null;

    onLoad() {
        this.setupNodes();
        this.hide();
    }

    show(text: string, onClick: OverlayClickHandler) {
        if (this.label) this.label.string = text;
        this.onClick = onClick;
        this.node.active = true;
    }

    hide() {
        this.node.active = false;
        this.onClick = null;
    }

    private setupNodes() {
        // 让本节点本身铺满屏幕
        let ui = this.node.getComponent(UITransform);
        if (!ui) ui = this.node.addComponent(UITransform);
        const size = view.getVisibleSize();
        ui.setContentSize(size.width, size.height);

        // 确保 zIndex 够高
        this.node.setSiblingIndex(999);

        // 半透明黑色背景
        const bg = new Node('Bg');
        this.node.addChild(bg);
        bg.addComponent(UITransform).setContentSize(size.width, size.height);
        const g = bg.addComponent(Graphics);
        g.fillColor = new Color(0, 0, 0, 180);  // 70% 黑
        g.rect(-size.width / 2, -size.height / 2, size.width, size.height);
        g.fill();
        this.bg = bg;

        // 大字
        const textNode = new Node('Text');
        this.node.addChild(textNode);
        textNode.addComponent(UITransform).setContentSize(600, 120);
        const label = textNode.addComponent(Label);
        label.fontSize = 80;
        label.color = new Color(0xff, 0xff, 0xff, 0xff);
        label.string = '';
        this.label = label;

        // 点击监听
        this.node.on(Input.EventType.TOUCH_END, this.onTouchEnd, this);
    }

    private onTouchEnd(_event: EventTouch) {
        const handler = this.onClick;
        this.hide();
        handler?.();
    }

    onDestroy() {
        this.node.off(Input.EventType.TOUCH_END, this.onTouchEnd, this);
    }
}
```

**关键点**：

- **`active = false` 做隐藏**。Overlay 常驻，数据驱动显隐。
- **`setSiblingIndex(999)`** 让 overlay 排在同级节点的最后 = 最上层。
- **点击先 hide 再 callback**：防止 callback 里重新 show 时已经被 hide 覆盖。
- **`EventTouch` 参数加 `_` 前缀**表示"有参数但不用"，防 ESLint unused-vars 报错。

### 文件 3：`GameController.ts` 接入胜利逻辑

```typescript
import { GameOverOverlay } from './GameOverOverlay';
import { isLevelPassed } from '../core/ArrowState';

private overlay: GameOverOverlay | null = null;
private gameOver = false;

onLoad() {
    console.log('[Arrow] Game scene loaded.');
    this.boardView = this.createBoardView();
    this.input = this.boardView.node.addComponent(InputController);
    this.hud = this.createHUD();
    this.overlay = this.createOverlay();
    this.loadLevel(1);
}

private createOverlay(): GameOverOverlay {
    const node = new Node('GameOverOverlay');
    this.node.addChild(node);
    return node.addComponent(GameOverOverlay);
}

private onLevelLoaded(data: LevelData) {
    // ... 已有
    this.gameOver = false;
    this.overlay?.hide();
}

update(dt: number) {
    if (!this.levelData) return;
    if (this.gameOver) return;

    // ... 已有 for 循环推进箭头 ...

    // 末尾判胜利
    if (isLevelPassed(this.runtimes)) {
        this.gameOver = true;
        console.log('[Arrow] 🎉 Level passed!');
        this.overlay?.show('You Win!', () => this.loadLevel(1));
    }
}
```

**关键**：

- **`gameOver` 守卫**：防止每帧都触发 overlay.show。
- **overlay.show 里的回调重载关卡**。loadLevel 会走 `onLevelLoaded` 重置 runtimes 和 gameOver。

## 运行效果

首关 3 根箭头方向都朝右，没有互相阻挡，**依次点击所有 3 根**：

- 每点一根 → 变蓝飞出 → End。
- 第 3 根 End 后的下一帧 → isLevelPassed 返回 true → 弹出全屏 "You Win!"
- 点击屏幕 → overlay 消失 → 关卡重载 → 3 根灰箭头回到起始位置。

## 易错点

### 易错 1：判胜利条件写成 `>= Start`

```typescript
return runtimes.every(rt => rt.mode >= ArrowMoveMode.Start);
```

枚举值顺序：`Idle=0, Collide=1, Back=2, Start=3, End=4`。**Collide（1）和 Back（2）都 < Start（3）**，所以这个写法是对的……**但代码语义不清**。谁看得出来"Start 是数值 3，比 Back 大"？

**规则**：**显式判等** `rt.mode === Start || rt.mode === End`。代码自注释。**不要依赖枚举值的数字大小关系写生产代码**，除非像 G3_FBase 一样配有注释说明。

### 易错 2：overlay 挡住 InputController

Overlay 是全屏节点，它监听 TOUCH_END。没问题。但如果 overlay 显示后，BoardView 的 InputController 也能收到触摸（因为 BoardView 挂在 Canvas 下），玩家点击既退 overlay 又可能误触箭头。

**Cocos 3.8 默认行为**：节点按 sibling 顺序冒泡。**如果上层节点处理了事件（return false 或 stopPropagation），下层不会收到**。

**但默认 `TOUCH_END` 不会自动 stopPropagation**。需要在 overlay 的 onTouchEnd 里：

```typescript
import { NodeEventType } from 'cc';
// ...
this.node.on(NodeEventType.TOUCH_END, (e: EventTouch) => {
    e.propagationStopped = true;
    // ...
}, this);
```

**实际上更简单**：overlay 显示时 `this.gameOver = true`，GameController.update 里已经 `if (this.gameOver) return`——**即使 InputController 触发 onArrowClick，由 canFire 挡住**（但此时可能还是会触发 fire，如果 runtime 处于 Idle……**不会**，Idle 的 runtime 在胜利时是不存在的，所有都 >= Start）。

所以**胜利场景下不会有副作用**。但失败场景（第 16 章）玩家可能 HP=0 时还有箭头是 Idle，overlay 显示时玩家点击如果穿透到 InputController 会触发 fire。下一章处理。

### 易错 3：胜利后 gameOver 忘了 reset

```typescript
private onLevelLoaded() {
    // ... 忘了 this.gameOver = false;
}
```

结果：重载关卡后 gameOver 仍是 true，update 整个 return，箭头不动。表现：黑屏+"You Win"消失但游戏卡死。

### 易错 4：Label 字号放大后 contentSize 没扩

```typescript
textNode.addComponent(UITransform).setContentSize(100, 30);  // ❌
label.fontSize = 80;
```

Label 字号 80 + contentSize 100 宽 → "You Win" 被压缩到一起或截断。

**规则**：contentSize 给足（600×120 稳）。

### 易错 5：overlay 上那层没做响应式尺寸

窗口变大后半透明矩形还是原来那么大，屏幕外围露出一圈没盖。

**简化处理**：本教程的 overlay 只在胜利瞬间显示，之后立即隐藏。**不做响应式**。后续屏幕适配章节统一加上时，会在适配逻辑里一并重算 Graphics 矩形。

## 扩展练习

1. **"下一关"提示**：Overlay 文字从 "You Win!" 改成 "Level 1 Passed! Tap to continue.."（自己拼装关卡号）。

2. **播放入场动画**：overlay.show() 时让文字从 scale 0 缓动到 scale 1。用 `tween(node)` API。

3. **思考题**：现在胜利判定"所有箭头飞出"是合理的。但如果某关是 5×5 有 5 根箭头，最终会不会互相挡住、总有一两根永远没法飞出去？这种关卡叫"死局"。关卡设计要怎么避免？提示：做关卡前先用 `findCollision` 模拟所有顺序，验证存在可通关路径。
