# 16 · 失败判定与重试界面

## 本节目标

**HP 归零后弹出"You Lose..." + "Retry"按钮**，点击后重置本关 runtimes + HP，重新开始。

预期：

- 构造一个必输场景（1 HP + 前方有挡的箭头）。
- 点击必撞箭头 → Collide → Back → HP 变 0 → 0.5 秒后弹 Overlay。
- Overlay 显示 "You Lose..." + "Tap to retry"。
- 点击屏幕 → 关卡重载 → HP 恢复 3。

一句话：**玩家能"输"了，并且能"再来"**。

---

## 需求分析

对照 G3_FBase 的 `TouchEndToCombatDoFailDispatchLogic`：

```typescript
if (this.combatInitLogic.storage.combat.hp > 0) {
    this.router.dispatchAction = ae.DispatchAction.Cancel;
    return;
}
if (data.runtime.failTriggerTime === 0) {
    data.runtime.failTriggerTime = this.worldMeta.worldTime;
    return;
}
if (worldTime - failTriggerTime > 0.5) {
    this.router.dispatchAction = ae.DispatchAction.FireOnce;
}
```

两个细节：

1. **HP <= 0 才算失败**。
2. **失败触发后延迟 0.5 秒再弹 Overlay**，让玩家看清"最后一下撞击"的动画，不突兀。

我们原样沿用。

---

## 实现思路

### 失败触发点

跟胜利一样放在 `update` 末尾：

```typescript
if (this.hp <= 0) {
    // 触发失败（带延时）
}
```

### 延时机制

GameController 加字段：

```typescript
private failTriggerTime = 0;
```

首次检测到 hp <= 0 时，`failTriggerTime = now`。之后每帧比较 `now - failTriggerTime >= 0.5` 时才弹 Overlay。

### 防止失败过程中玩家继续点

失败触发到 Overlay 弹出之间那 0.5 秒，runtimes 上可能还有 Idle 的箭头。玩家这时点击会 fire。

**解决**：`gameOver = true` 一旦置位，`onArrowClick` 直接 return。

## 代码实现

### 文件 1：`GameController.ts` 加失败逻辑

```typescript
private failTriggerTime = 0;

private onLevelLoaded(data: LevelData) {
    // ... 已有
    this.gameOver = false;
    this.failTriggerTime = 0;
    this.overlay?.hide();
}

private onArrowClick(idx: number) {
    if (this.gameOver) return;
    const rt = this.runtimes[idx];
    if (!canFire(rt)) return;
    // ... 已有
}

update(dt: number) {
    if (!this.levelData) return;
    if (this.gameOver) return;

    // ... 已有箭头推进循环（Start / Collide / Back / hitFlash） ...

    // 胜利判定（见第 15 章）
    if (isLevelPassed(this.runtimes)) {
        this.triggerWin();
        return;
    }

    // 失败判定
    this.checkFail();
}

private triggerWin() {
    this.gameOver = true;
    console.log('[Arrow] 🎉 Level passed!');
    this.overlay?.show('You Win!', () => this.loadLevel(1));
}

private checkFail() {
    if (this.hp > 0) return;
    const now = director.getTotalTime() / 1000;
    if (this.failTriggerTime === 0) {
        this.failTriggerTime = now;
        return;
    }
    if (now - this.failTriggerTime < 0.5) return;
    // 延时已过，弹失败
    this.gameOver = true;
    console.log('[Arrow] 💀 Level failed');
    this.overlay?.show('You Lose...\nTap to retry', () => this.loadLevel(1));
}
```

**关键点**：

- **胜利优先**：如果某帧同时满足"全部飞出" + "hp=0"（理论边界情况），先判胜利。对应 G3_FBase 里 `TouchEndToCombatDoFailDispatchLogic` 也有 "passTime > 0 → cancel"。
- **`failTriggerTime = 0` 作为"未触发"标记**。借鉴 G3_FBase 的 pattern。
- **连续两行文字**：`'You Lose...\nTap to retry'`。Label 支持 `\n` 换行（只要 contentSize 高度够）。

### 文件 2：`GameOverOverlay.ts` 支持多行文字（可选）

15 章写的 `GameOverOverlay` 用的是 `label.fontSize = 80` + `contentSize 600×120`，那时候只有"You Win"一行够用。本章失败界面是 `'You Lose...\nTap to retry'` 两行，容器要加高：

```typescript
textNode.addComponent(UITransform).setContentSize(800, 240);  // 更高
label.overflow = Label.Overflow.NONE;                          // 不裁剪
```

Cocos 的 `Label` 默认支持 `\n` 断行，保证 contentSize 高度 >= 两行即可。

如果你的文字两行挤在一起，调：

```typescript
label.lineHeight = 96;  // 行高
```

## 易错点

### 易错 1：失败延时期间玩家继续操作

```typescript
private onArrowClick(idx: number) {
    // 没 if (this.gameOver) return;
    const rt = this.runtimes[idx];
    if (!canFire(rt)) return;
    ...
}
```

看起来没事（canFire 挡了 Idle 以外）。但 HP=0 触发失败延时的 0.5 秒里，如果有 Idle 箭头，玩家再点击一根，它会 fire 继续飞 → 可能 End → 胜利判定 true → Overlay 变成胜利而不是失败！

**规则**：**`gameOver` 语义是"游戏已结束，任何输入都无效"**。`onArrowClick` 第一行 `if (this.gameOver) return` 不可省。

### 易错 2：`failTriggerTime` 忘了 reset

关卡重载后，`failTriggerTime` 仍是上一次的值。下次 hp 到 0 时，立刻超过 0.5 秒 → 瞬间弹 Overlay，没延时效果。

**规则**：**`gameOver` 类字段都要在关卡重载时 reset**。建议建一个统一的 `resetGameState()` 方法。

### 易错 3：胜利和失败同时触发时 Overlay 连开两次

```typescript
update(dt) {
    // ... 箭头推进 ...
    if (isLevelPassed(runtimes)) this.triggerWin();
    this.checkFail();  // 没 else
}
```

如果 triggerWin 没把 gameOver 置位就 return，checkFail 又触发失败 → Overlay.show 两次。

**规则**：**两个互斥的终局判定用 if-else 或 early return**：

```typescript
if (isLevelPassed(this.runtimes)) { this.triggerWin(); return; }
this.checkFail();
```

（上面代码已经这么写了。）

### 易错 4：`director.getTotalTime()` 单位混用

```typescript
if (now - this.failTriggerTime < 500) return;  // ❌ 毫秒
```

`director.getTotalTime() / 1000` 后 now 是秒。阈值 0.5（秒），**不要写 500**。

### 易错 5：`\n` 在某些字体下无效

```typescript
label.string = 'You Lose...\nTap to retry';
```

Cocos 的 TTF 字体大多支持 `\n`。但**位图字体（BMFont）可能不支持**，要换成两个 Label 或手动拆字符串。本教程用系统字体，没问题。

## 扩展练习

1. **失败界面带重试按钮**：现在整屏点击都能重试。改成：只有"Retry"按钮能重试。提示：给 label 节点加子节点"RetryBtn"，只有该节点上的 TOUCH_END 触发回调，overlay 其他区域点击不响应。

2. **失败次数统计**：在 GameController 加 `failCount` 字段，每次失败 +1。在 HUD 显示。

3. **思考题**：现在 checkFail 用"0.5 秒延时"让玩家看清最后一击。但如果玩家**在延时期间**手动重试（按键、快捷键），应该立刻响应还是等延时结束？G3_FBase 是怎么处理这种"提前结算"的？提示：G3_FBase 的 `ResetUiOkLogic` 有独立的 reset 链，不等失败判定。
