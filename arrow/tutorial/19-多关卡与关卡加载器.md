# 19 · 多关卡 + 关卡加载器

## 本节目标

**造 5 关 JSON + 处理"关卡号超出"的情况**，让游戏真正能从 1 玩到 5：

- `level_01` ~ `level_05.json` 五关，难度递增。
- 通关第 5 关后显示"全部通关！"Overlay，点击回 Home。
- level=6 时加载失败的兜底（显示"Coming soon..."）。

一句话：**从"一关游戏"变成"多关游戏"**。

---

## 需求分析

造关卡不是技术问题，是关卡设计。我们用 G3_FBase 原始首关作为第 1 关，另外造 4 关**演示不同玩法**：

| 关卡 | 配置 | 玩法特色 |
|------|------|---------|
| 1 | 3 根箭头都朝右，互不干扰 | 热身 |
| 2 | 2 根箭头互相挡（一前一后） | 学习回弹 |
| 3 | 4 根箭头两两互挡 | 顺序很重要 |
| 4 | 箭头方向混合（上下左右） | 方向挑战 |
| 5 | 网格更大 7×7，5 根箭头 | 复杂度上来了 |

### 关卡加载器要做的事

- 按 level 加载对应 JSON。
- 文件不存在时触发"全部通关"逻辑，不崩。

对应 G3_FBase 的 `LevelConfigUrl.ConfigUrl = this.url('res/configs/Level_00001')` 是固定的。它处理多关卡的方式是"把关卡名预计算后从外部设进 URL"——复杂。我们简化为**按 level 拼文件名**。

---

## 实现思路

### 文件命名

```
level_01.json
level_02.json
...
level_05.json
```

加载路径：`resources.load('levels/level_' + pad2(levelNo), JsonAsset, ...)`。

### 加载失败处理

```typescript
resources.load(path, JsonAsset, (err, asset) => {
    if (err) {
        // level 超出
        if (levelNo > MAX_KNOWN_LEVEL) {
            this.onAllLevelsCompleted();
            return;
        }
        // 其他加载错
        console.error(`[Arrow] Load failed: ${path}`, err);
    }
    // ...
});
```

或者更优雅：**先检查 level 是否超过已知最大关**，超过就走"全部通关"分支，否则才 load。

### Home 场景的"全部通关"状态

- level > MAX_LEVEL → 显示 "All Done!" 而不是 "Level N"。
- Play 按钮隐藏或变成 "Restart"。

## 代码实现

### 文件 1：新建 5 个关卡 JSON

在 `assets/resources/levels/` 下创建以下文件。**可以直接在文件系统里新建**（Cocos 会自动 import）。

**`level_01.json`**（现有，保持不变）：

```json
{
  "rows": 5, "cols": 5,
  "arrows": [
    { "direction": [0, 1], "origin": [2, 4], "coords": [[2,2],[2,3],[2,4]] },
    { "direction": [0, 1], "origin": [3, 4], "coords": [[3,2],[3,3],[3,4]] },
    { "direction": [0, 1], "origin": [4, 4], "coords": [[4,2],[4,3],[4,4]] }
  ]
}
```

**`level_02.json`** —— 必撞+学习回弹：

```json
{
  "rows": 5, "cols": 5,
  "arrows": [
    { "direction": [0, 1], "origin": [3, 2], "coords": [[3,1],[3,2]] },
    { "direction": [0, 1], "origin": [3, 5], "coords": [[3,3],[3,4],[3,5]] }
  ]
}
```

玩法：箭头 1 必挡 0。正确解法：先发 1（先飞出右边界），再发 0。

**`level_03.json`** —— 四根交叉：

```json
{
  "rows": 5, "cols": 5,
  "arrows": [
    { "direction": [0, 1], "origin": [1, 3], "coords": [[1,1],[1,2],[1,3]] },
    { "direction": [0, -1], "origin": [1, 3], "coords": [[1,5],[1,4],[1,3]] },
    { "direction": [1, 0], "origin": [3, 3], "coords": [[1,3],[2,3],[3,3]] },
    { "direction": [-1, 0], "origin": [3, 3], "coords": [[5,3],[4,3],[3,3]] }
  ]
}
```

玩法：4 根箭头在 `[1,3]`/`[3,3]` 形成十字，互相挡。**注意 coords 写法**：箭头 1 方向是 `[0,-1]`（向左），所以 coords 从 "尾 [1,5] → 头 [1,3]"——**origin 是 head**。

**`level_04.json`** —— 纯方向挑战：

```json
{
  "rows": 5, "cols": 5,
  "arrows": [
    { "direction": [1, 0], "origin": [3, 1], "coords": [[1,1],[2,1],[3,1]] },
    { "direction": [-1, 0], "origin": [3, 3], "coords": [[5,3],[4,3],[3,3]] },
    { "direction": [0, 1], "origin": [1, 5], "coords": [[1,3],[1,4],[1,5]] }
  ]
}
```

三根不同方向，暂无互挡。简单关。

**`level_05.json`** —— 7×7 大网格：

```json
{
  "rows": 7, "cols": 7,
  "arrows": [
    { "direction": [0, 1], "origin": [1, 4], "coords": [[1,1],[1,2],[1,3],[1,4]] },
    { "direction": [0, 1], "origin": [3, 4], "coords": [[3,1],[3,2],[3,3],[3,4]] },
    { "direction": [0, 1], "origin": [5, 4], "coords": [[5,1],[5,2],[5,3],[5,4]] },
    { "direction": [1, 0], "origin": [7, 6], "coords": [[5,6],[6,6],[7,6]] },
    { "direction": [-1, 0], "origin": [1, 2], "coords": [[3,2],[2,2],[1,2]] }
  ]
}
```

**关卡设计不是这份教程的重点**。新人可以自己改数据试玩法。

### 文件 2：`common/Config.ts` 加常量

```typescript
export const Config = {
    gap: 100,
    pointSize: 12,
    arrowLineWidth: 10,
    arrowHeadSize: 24,
    arrowSpeed: 5,
    /** 已有关卡数 */
    maxLevel: 5,
} as const;
```

### 文件 3：`GameController.ts` 处理"超出最大关卡"

```typescript
import { Config } from '../common/Config';

private loadLevel(levelNo: number) {
    if (levelNo > Config.maxLevel) {
        // 全部通关
        console.log('[Arrow] All levels completed!');
        this.overlay?.show('All Clear!\nTap to restart', () => {
            GameData.level = 1;
            director.loadScene('Home');
        });
        return;
    }

    const no = levelNo < 10 ? `0${levelNo}` : `${levelNo}`;
    const path = `levels/level_${no}`;
    resources.load(path, JsonAsset, (err, asset) => {
        if (err) {
            console.error(`[Arrow] Load failed: ${path}`, err);
            return;
        }
        try {
            this.levelData = validateLevelData(asset.json);
            this.onLevelLoaded(this.levelData);
        } catch (e) {
            console.error(`[Arrow] Invalid level:`, e);
        }
    });
}
```

**关键**：超出最大关卡时**直接显示 Overlay**，不走正常的 loadLevel 流程。

### 文件 4：`HomeController.ts` 处理"超出最大关卡"

```typescript
private createLevelText(levelNo: number) {
    const n = new Node('LevelText');
    this.node.addChild(n);
    n.setPosition(0, 100, 0);
    n.addComponent(UITransform).setContentSize(400, 100);
    const label = n.addComponent(Label);
    label.fontSize = 80;
    label.color = new Color(0x60, 0x6a, 0xf8, 0xff);
    if (levelNo > Config.maxLevel) {
        label.string = 'All Clear!';
    } else {
        label.string = `Level ${levelNo}`;
    }
}
```

**Play 按钮**怎么改：最简单的做法是**不改**。玩家点 Play 进 Game，Game 检测到超级 level 立刻弹 "All Clear" overlay，相当于"奖励页"。

## 运行效果

1. 启动 → Home `Level 1` → Play → 玩 level_01 → 通关 → `Level 2`。
2. 继续 → 玩 level_02 → 通关 → `Level 3`。
3. ... → 玩 level_05 → 通关 → `Level 6` (但 6 已经超过 maxLevel)。
4. 再点 Play → Game 场景立刻弹 "All Clear! Tap to restart"。
5. 点击 → GameData.level 重置为 1 → 回 Home 显示 `Level 1`。

## 易错点

### 易错 1：关卡文件 JSON 格式错误

新手 copy-paste JSON 时常见问题：

- 多一个逗号：`[1,2,3,]` ❌
- 漏闭合括号
- 字段名用单引号：`'direction': [0,1]` ❌（JSON 标准只认双引号）

**排查**：F12 Console 会打 `[Arrow] Invalid level: ...`。打开文件看 JSON 合法性。

### 易错 2：coords 顺序错

关卡 `level_03.json` 的箭头 1 `direction: [0,-1]` 向左，coords 应该是"尾在右、头在左"：

```json
"coords": [[1,5], [1,4], [1,3]]
```

如果写成 `[[1,3],[1,4],[1,5]]`（反的），箭头头会渲染在 `[1,5]`（错的那一端），方向三角形却朝左 → 视觉错位。

**规则**：**coords 从尾到头，最后一个等于 origin**。

### 易错 3：level 跳过文件

如果你漏了 level_03.json，玩家通关 level_02 后加载 level_03 会**加载失败**但没有 "All Clear" 处理（它只处理 > maxLevel）。

**防御**：err 分支里打 warning 但不崩，overlay 显示"Coming soon..."回 Home。

```typescript
if (err) {
    console.warn(`[Arrow] Level ${levelNo} not found`);
    this.overlay?.show(`Level ${levelNo} coming soon...`, () => director.loadScene('Home'));
    return;
}
```

### 易错 4：关卡难度跳跃太大

设计关卡是另一门学问。如果 level_02 就搞 10 根箭头，新手直接劝退。**教程里我们给的 5 关是演示用的**，真实游戏要请关卡策划专门设计。

### 易错 5：7×7 关卡在小屏幕上看不清

第 06 章的 `computeBoardLayout` 算的 scale 受限。7×7 棋盘 + `gap=100` = 700×700 原始尺寸，小屏幕 scale 会小于 1。**点点太小看不清**。

**可选**：根据棋盘尺寸调整 Config.pointSize 或 arrowLineWidth。本教程不做。

## 扩展练习

1. **关卡选择页**：Home 场景加一个"LEVELS"按钮，点击进入关卡列表（5 个网格按钮，对应 5 关），选哪关进哪关。提示：新增一个 LevelSelectController，参考 HomeController 的写法。

2. **关卡难度标签**：JSON 加 `degree` 字段（"Easy"/"Medium"/"Hard"）。在 HUD 或 Home 显示。对应 G3_FBase `LevelConfig.degree`。

3. **思考题**：现在 `maxLevel = 5` 是 hardcode 的。G3_FBase 里关卡数来源于什么？（答：生产环境是从服务器拉关卡列表；本地有 LevelConfigGenLogic 生成。这在教程里过度复杂。）如果你要做到"加 level_06.json 文件就自动支持"，需要改哪些代码？（Hint：运行时 try-load 直到加载失败来确定 maxLevel。）
