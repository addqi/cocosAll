# 08 · 点击箭头 → 触发状态转移

## 本节目标

**让玩家点击任意一根箭头，它的颜色从白色变成蓝色**（Idle → Start），**画面位置还不动**（下一章再说）。

预期：

- 点击任何一根箭头 → 变蓝。
- 再次点击同一根 → **没反应**（不再是 Idle，`canFire` 返回 false）。
- 点击棋盘空白 → **没反应**（`findArrowIndex` 返回 -1）。
- Console 打印：`[Arrow] Arrow 0 fired. mode = Start`。

一句话：**用户操作第一次改变游戏状态**。

---

## 需求分析

这一章要连通三件事：

1. **拿到用户点击**（Cocos 3.8 的 `TOUCH_END` 事件）。
2. **把屏幕坐标翻译成"哪根箭头"**（`pixelToGrid` + 比对 `ArrowData.coords`）。
3. **把"点击"翻译成"状态转移"**（调 `fire(rt, false)`）。

### 一个重要前提：点击判定用"静态配置"还是"运行时状态"？

07 章我们给 `ArrowRuntime` 只保留了 `mode` + `hasFailed` 两个字段——**没有 `coords`**。

但点击判定必须知道"每根箭头占哪些格子"。这信息从哪里来？

答案：**`LevelData.arrows[i].coords`**。08 章箭头还不会动，**配置里写的 coords 就是此刻屏幕上的 coords**。用静态配置做命中判定，完全正确。

> **Linus 的品味**：不要为了"未来也许会变"而把不变的数据复制一份。**08 章箭头不动 → 点 `ArrowData.coords` → 最简**。09 章箭头开始动了再说。

### 怎么判断点到了哪根箭头

把玩家的屏幕坐标反算成格子坐标 `(row, col)`，然后**遍历所有箭头的 coords，看哪根箭头"占"着这个格子**：

```typescript
const { row, col } = pixelToGrid(localX, localY, rows, cols);
for (let i = 0; i < arrows.length; i++) {
    if (arrows[i].coords.some(c => c[0] === row && c[1] === col)) {
        return i;
    }
}
return -1;  // 没点中任何箭头
```

玩家点到箭头**身体任何一节**都算命中，L 形箭头的拐弯处也能点。

### 屏幕坐标 → 本地坐标

Cocos 3.8 触摸事件返回**UI 坐标**（原点 Canvas 左下角）。我们要的是 BoardView 节点的**本地坐标**（原点 BoardView 中心，和 `gridToPixel` 的产出一致）。

```typescript
const uiLoc = event.getUILocation();          // {x, y}
const world = new Vec3(uiLoc.x, uiLoc.y, 0);
const local = ui.convertToNodeSpaceAR(world); // 换到节点本地
```

`convertToNodeSpaceAR` 的 **AR** 表示 **Anchor-Relative**（考虑锚点）。BoardView 锚点默认 `(0.5, 0.5)`，所以 `local` 原点就是 BoardView 中心。

### 谁持有 `ArrowRuntime[]`

`ArrowRuntime[]` 是每根箭头的运行时状态（目前只有 mode + hasFailed）。放哪儿？

**放在 `GameController`**。理由：

- GameController 是场景总控。将来 HP、level 编号也放这里（14 / 18 章）。
- `InputController` 只管"解析输入"，不持有状态数据。
- `BoardView` 只管"画面怎么排"，也不持有逻辑状态。

## 代码实现

### 文件 1：扩展 `core/Coord.ts`（追加 `findArrowIndex`）

在 `Coord.ts` 文件**末尾追加**以下代码（文件头的 `import { Config }` 那行已有，不要重复；`ArrowData` 是新增的 import）：

```typescript
import { ArrowData } from './LevelData';  // 加在文件顶部已有 imports 旁边

/**
 * 根据本地像素坐标查找被点中的箭头索引。
 * 找不到返回 -1。
 *
 * 注：这里吃 ArrowData[]（静态配置）而不是 ArrowRuntime[]。
 * 本章箭头尚未移动，配置和运行时占位完全一致。用静态最简单。
 */
export function findArrowIndex(
    localX: number, localY: number,
    arrows: readonly ArrowData[],
    rows: number, cols: number,
): number {
    const { row, col } = pixelToGrid(localX, localY, rows, cols);
    for (let i = 0; i < arrows.length; i++) {
        if (arrows[i].coords.some(c => c[0] === row && c[1] === col)) {
            return i;
        }
    }
    return -1;
}
```

**关键点**：

- **签名吃 `readonly ArrowData[]`**：`readonly` 告诉 TS "我不会改这个数组"，调用方可以放心把 `LevelData.arrows` 传进来不怕被偷改。
- **`Coord.ts` 不依赖 `ArrowState.ts`**：保持 `core/Coord` 是**纯坐标工具**，不碰运行时状态。分层干净。

### 文件 2：扩展 `ArrowView.ts`（加 `refresh` 驱动颜色）

当前 `ArrowView` 每次 `initData` 都立刻画一次，颜色写死 `IDLE_COLOR`。现在状态会变，需要**一个可以按 runtime 刷颜色的入口**。

在 `assets/scripts/game/ArrowView.ts` 上做 3 处增量修改：

1. 加 `import { ArrowRuntime, ArrowMoveMode } from '../core/ArrowState';`
2. 把写死的 `IDLE_COLOR` 换成 3 个颜色常量 + 一个 `_pickColor(rt)` 纯函数。
3. 加一个公开方法 `refresh(rt)`，调用它可以按当前 runtime 重绘。

完整文件（替换掉 05 章之后的旧版）：

```typescript
import {
    _decorator, Component, UITransform, Graphics, Color,
} from 'cc';
import { ArrowData, Cell } from '../core/LevelData';
import { ArrowRuntime, ArrowMoveMode } from '../core/ArrowState';
import { gridToPixel, Pixel } from '../core/Coord';
import { Config } from '../common/Config';
const { ccclass } = _decorator;

/** 三种箭头颜色。数值取自 G3_FBase */
const COLOR_IDLE = new Color(0xff, 0xff, 0xff, 0xff); // 空闲：白
const COLOR_MOVE = new Color(0x5b, 0x72, 0xfe, 0xff); // 飞行 / 逃脱：蓝
const COLOR_STOP = new Color(0xfe, 0x4b, 0x5e, 0xff); // 撞击 / 回弹 / 曾失败：红

@ccclass('ArrowView')
export class ArrowView extends Component {
    private _graphics: Graphics | null = null;
    private _data: ArrowData | null = null;
    private _rows = 0;
    private _cols = 0;
    /** 当前用于绘制的 runtime 引用。null 时按 Idle 画（首次 initData 用）*/
    private _rt: ArrowRuntime | null = null;

    /** 由 BoardView 注入数据，注入后按 Idle 颜色画一次 */
    public initData(data: ArrowData, rows: number, cols: number) {
        if (!this._graphics) {
            this._graphics = this.getComponent(Graphics) ?? this.addComponent(Graphics);
        }
        if (!this.node.getComponent(UITransform)) {
            this.node.addComponent(UITransform);
        }
        this._data = data;
        this._rows = rows;
        this._cols = cols;
        this._redraw();
    }

    /** 由 GameController 在每次状态变化时调用，按 runtime 重绘 */
    public refresh(rt: ArrowRuntime) {
        this._rt = rt;
        this._redraw();
    }

    private _redraw() {
        if (!this._data || !this._graphics) {
            console.error('ArrowView: _data or _graphics is null');
            return;
        }
        const g = this._graphics;
        const { coords } = this._data;
        if (coords.length < 2) return;

        g.clear();

        const color = this._pickColor();
        const pixels: Pixel[] = coords.map(
            ([r, c]) => gridToPixel(r, c, this._rows, this._cols),
        );

        g.strokeColor = color;
        g.lineWidth = Config.arrowLineWidth;
        g.moveTo(pixels[0].x, pixels[0].y);
        for (let i = 1; i < pixels.length; i++) {
            g.lineTo(pixels[i].x, pixels[i].y);
        }
        g.stroke();

        this._drawHead(g, coords, pixels, color);
    }

    /** 按当前 runtime 派生颜色。无 runtime 时按 Idle。*/
    private _pickColor(): Color {
        const rt = this._rt;
        if (!rt) return COLOR_IDLE;
        if (rt.mode === ArrowMoveMode.Start || rt.mode === ArrowMoveMode.End) {
            return COLOR_MOVE;
        }
        if (rt.mode === ArrowMoveMode.Collide || rt.mode === ArrowMoveMode.Back) {
            return COLOR_STOP;
        }
        if (rt.hasFailed) return COLOR_STOP;
        return COLOR_IDLE;
    }

    /** 在折线头端画三角箭头。方向从 coords 末尾两格派生。*/
    private _drawHead(g: Graphics, coords: Cell[], pixels: Pixel[], color: Color) {
        const n = coords.length;
        const [hr, hc] = coords[n - 1];
        const [pr, pc] = coords[n - 2];
        const dr = hr - pr;
        const dc = hc - pc;

        const pdx = dc;
        const pdy = -dr;
        const nx = -pdy;
        const ny = pdx;

        const headPx = pixels[n - 1];
        const s = Config.arrowHeadSize;
        const tipX = headPx.x + pdx * s;
        const tipY = headPx.y + pdy * s;
        const leftX = headPx.x + nx * s / 2;
        const leftY = headPx.y + ny * s / 2;
        const rightX = headPx.x - nx * s / 2;
        const rightY = headPx.y - ny * s / 2;

        g.fillColor = color;
        g.moveTo(tipX, tipY);
        g.lineTo(leftX, leftY);
        g.lineTo(rightX, rightY);
        g.close();
        g.fill();
    }
}
```

**关键变化**：

- 多了一个 `_rt` 字段，保存当前 runtime 引用。
- `refresh(rt)` 是外部唯一入口：来了新的 runtime 就重绘。
- `_pickColor()` 是**状态→颜色**的**纯映射**，无 runtime 时默认 Idle。
- 绘制代码（折线 + 派生方向三角头）和 05 章保持一致，只是**颜色改成按 runtime 派生**。

> **不写响应式**：Cocos 3.8 没有"数据变了画面自动刷"的机制。用谁都知道的土办法——"改数据 + 手动调 refresh"。几千行代码规模下，响应式不值得引入。

### 文件 3：`game/InputController.ts`（新增）

```typescript
import {
    _decorator, Component, EventTouch, Input, UITransform, Vec3,
} from 'cc';
import { findArrowIndex } from '../core/Coord';
import { LevelData } from '../core/LevelData';
const { ccclass } = _decorator;

export type ArrowClickHandler = (arrowIndex: number) => void;

/**
 * 输入控制器。挂在 BoardView 节点上，监听 TOUCH_END。
 * 把"触摸坐标"翻译成"箭头索引"，通过回调交给 GameController。
 */
@ccclass('InputController')
export class InputController extends Component {
    private _data: LevelData | null = null;
    private _onArrowClick: ArrowClickHandler | null = null;

    /** 由 GameController 调用一次，注入关卡数据和点击回调 */
    public setup(data: LevelData, handler: ArrowClickHandler) {
        this._data = data;
        this._onArrowClick = handler;

        this._ensureTouchableSize();
        this.node.on(Input.EventType.TOUCH_END, this._onTouchEnd, this);
    }

    onDestroy() {
        this.node.off(Input.EventType.TOUCH_END, this._onTouchEnd, this);
    }

    private _ensureTouchableSize() {
        let ui = this.node.getComponent(UITransform);
        if (!ui) ui = this.node.addComponent(UITransform);
        // 给一个"足够大"的 contentSize，保证屏幕任何位置点击都能命中此节点。
        // 严谨做法是跟随 canvas-resize 动态调整，放到屏幕适配章节再管。
        ui.setContentSize(100000, 100000);
    }

    private _onTouchEnd(event: EventTouch) {
        if (!this._data || !this._onArrowClick) return;

        const uiLoc = event.getUILocation();
        const world = new Vec3(uiLoc.x, uiLoc.y, 0);
        const ui = this.node.getComponent(UITransform)!;
        const local = ui.convertToNodeSpaceAR(world);

        const idx = findArrowIndex(
            local.x, local.y,
            this._data.arrows,
            this._data.rows, this._data.cols,
        );
        if (idx < 0) return;
        this._onArrowClick(idx);
    }
}
```

**3 个设计决策**：

- **`setup` 只传 `data + handler`**，不传 `runtimes`。InputController 只认关卡配置，不碰运行时状态——**职责最小**。
- **`ensureTouchableSize`**：Cocos 3.8 的 `TOUCH_*` 事件只在节点实际占据的像素内触发。BoardView 节点默认 `contentSize=(0,0)`，不设大点永远收不到触摸。
- **回调 `(idx) => void` 而非传 `GameController`**：`InputController` 不认识 `GameController`，只认识"一个吃 idx 的函数"。解耦。

### 文件 4：改造 `GameController.ts`（持有 runtimes + 接入 InputController）

在 07 章完成态的基础上**增量修改**：

1. 新增字段 `runtimes: ArrowRuntime[]` 和 `input: InputController | null`。
2. `onLoad` 在创建 BoardView 时顺便挂一个 `InputController`。
3. `onLevelLoaded` 里创建 runtimes、首次 refresh、setup input。
4. 新增 `onArrowClick / refreshArrow / refreshAllArrows` 三个方法。
5. 07 章的 `stateMachineSelfTest` 可以保留，也可以删——它只是自测日志，不影响游戏。下面的完整代码保留它，要删自便。

完整文件：

```typescript
import { _decorator, Component, resources, JsonAsset, Node } from 'cc';
import { LevelData, validateLevelData, ArrowData } from '../core/LevelData';
import { BoardView } from './BoardView';
import {
    ArrowMoveMode, ArrowRuntime, createRuntime, canFire, fire,
    markEnd, markCollide, markBack, resetToIdle,
} from '../core/ArrowState';
import { InputController } from './InputController';
const { ccclass } = _decorator;

@ccclass('GameController')
export class GameController extends Component {
    private levelData: LevelData | null = null;
    private boardView: BoardView | null = null;
    private input: InputController | null = null;
    /** 箭头运行时状态，按索引和 levelData.arrows 一一对应 */
    private runtimes: ArrowRuntime[] = [];

    onLoad() {
        this.boardView = this.createBoardView();
        this.input = this.boardView.node.addComponent(InputController);
        this.loadLevel(1);
        this.stateMachineSelfTest();
    }

    private createBoardView(): BoardView {
        const node = new Node('BoardView');
        this.node.addChild(node);
        return node.addComponent(BoardView);
    }

    private loadLevel(levelNo: number) {
        const no = levelNo < 10 ? `0${levelNo}` : `${levelNo}`;
        const path = `levels/level_${no}`;
        resources.load(path, JsonAsset, (err, asset) => {
            if (err) {
                console.error(`[Arrow] Load level failed: ${path}`, err);
                return;
            }
            let data: LevelData;
            try {
                data = validateLevelData(asset.json);
            } catch (e) {
                console.error(`[Arrow] Level data invalid:`, e);
                return;
            }
            this.levelData = data;
            this.onLevelLoaded(data);
        });
    }

    private onLevelLoaded(data: LevelData) {
        console.log(`[Arrow] Level loaded: ${data.rows} x ${data.cols}, arrows = ${data.arrows.length}`);
        this.runtimes = data.arrows.map(a => createRuntime(a));
        this.boardView?.render(data);
        this.refreshAllArrows();
        this.input?.setup(data, (idx) => this.onArrowClick(idx));
    }

    private onArrowClick(idx: number) {
        const rt = this.runtimes[idx];
        if (!canFire(rt)) return;
        fire(rt, false);  // blocked 预检下一章再加
        console.log(`[Arrow] Arrow ${idx} fired. mode = ${ArrowMoveMode[rt.mode]}`);
        this.refreshArrow(idx);
    }

    private refreshAllArrows() {
        const views = this.boardView?.getArrowViews() ?? [];
        for (let i = 0; i < views.length; i++) {
            views[i].refresh(this.runtimes[i]);
        }
    }

    private refreshArrow(idx: number) {
        const views = this.boardView?.getArrowViews() ?? [];
        views[idx]?.refresh(this.runtimes[idx]);
    }

    /** 07 章引入的状态机自测。可保留，可删。 */
    private stateMachineSelfTest() {
        const fakeArrow: ArrowData = {
            direction: [0, 1],
            origin: [1, 3],
            coords: [[1, 1], [1, 2], [1, 3]],
        };
        const rt = createRuntime(fakeArrow);
        const lines: string[] = [];
        const pad = (s: string) => (s + '                    ').substring(0, 20);
        const step = (label: string) => lines.push(`${pad(label)} ${ArrowMoveMode[rt.mode]}`);

        step('init:');
        fire(rt, true);              step('after fire(blocked):');
        markCollide(rt);             step('after markCollide:');
        markBack(rt);                step('after markBack:');
        fire(rt, false);             step('after fire(false):');
        markEnd(rt);                 step('after markEnd:');
        resetToIdle(rt, fakeArrow);  step('after resetToIdle:');

        console.log('[Arrow] StateMachine self-test:\n  ' + lines.join('\n  '));
    }
}
```

**关键数据流**：

```
用户点击
  ↓
InputController._onTouchEnd
  ↓ findArrowIndex(local.x, local.y, data.arrows, ...)
onArrowClick(idx)   ← 回调到 GameController
  ↓ fire(rt, false)
runtimes[idx].mode = Start
  ↓ refreshArrow(idx)
ArrowView.refresh(rt)
  ↓ _pickColor → _redraw
画面变蓝
```

**一条单向链**。`InputController` 不认识 `fire`，`ArrowView` 不认识触摸，`GameController` 作为调度串起所有。

## 易错点

### 易错 1：BoardView 节点没 UITransform / contentSize 是 0

`TOUCH_*` 只在节点实际占据的像素内触发。BoardView 初始 `contentSize=(0,0)` 不改就永远收不到触摸。`InputController._ensureTouchableSize` 就是解决这个。

### 易错 2：用 `getLocation` 而不是 `getUILocation`

```typescript
event.getLocation()     // ❌ 屏幕物理像素（受 DPR 影响）
event.getUILocation()   // ✅ UI 坐标（已考虑设计分辨率和 DPR）
```

iPhone 的 DPR 是 2~3，用 `getLocation` 你的命中点会**偏移一倍**。

### 易错 3：`convertToNodeSpace` 少了 `AR`

```typescript
ui.convertToNodeSpace(world)   // ❌ 以节点左下为原点
ui.convertToNodeSpaceAR(world) // ✅ AR = Anchor-Relative，以锚点为原点
```

`gridToPixel` 输出的是"中心为原点"的坐标。没 AR 会全偏到节点左下。

### 易错 4：传 `runtimes` 给 `findArrowIndex`

07 章的 `ArrowRuntime` **没有 `coords` 字段**。你不小心写成 `findArrowIndex(x, y, this.runtimes, ...)` 会报 `类型"ArrowRuntime"上不存在属性"coords"`。

**规则**：本章点击判定走**静态配置**，传 `this.levelData!.arrows`。

### 易错 5：触摸回调里忘记 `refresh`

```typescript
fire(rt, false);
// 忘了 this.refreshArrow(idx)
```

状态变了、日志打了，但画面还是白色。Cocos 没响应式，**手动 refresh 是义务**。

### 易错 6：三个数组下标不对齐

`runtimes[i]` 必须对应 `levelData.arrows[i]` 对应 `boardView.getArrowViews()[i]`。哪天想排序或过滤，三处要同时改。
