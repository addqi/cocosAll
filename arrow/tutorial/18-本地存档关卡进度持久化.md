# 18 · 本地存档：关卡进度持久化

## 本节目标

**用 localStorage 持久化关卡号**：

- 首次启动：Home 显示 `Level 1`。
- 通关一次：level +1，保存。
- 刷新浏览器/关游戏重开：Home 显示 `Level 2`。
- Game 场景 HUD 也显示当前关卡号。

一句话：**状态跨会话持久化**。

---

## 需求分析

对照 G3_FBase 的 `CombatInitLogic.defineStorageData()`：

```typescript
defineStorageData() {
    return {
        level: 1,
        combat: { hp: -1, arrows: {...}, newCombat: true, win: false, dirty: 0 },
        reviveCount: 0,
        hintCount: 0,
        restartCount: 0,
        rayClickCount: 0,
    };
}
```

原子引擎的 storage 是引擎托管的，自动序列化/反序列化。Cocos 3.8 原生没有这个。**我们用 `sys.localStorage` 手动存**。

### 数据层

新建 `data/` 目录：

- `Storage.ts`：`load(key, defaultValue)` / `save(key, value)` 两个函数。封装 JSON 序列化和 try-catch。
- `GameData.ts`：游戏全局数据单例，持有 level（以后会扩展）。

**单例** 有两种写法：

1. 静态类 + 静态方法。
2. 导出一个全局实例对象。

**选 2**。更简单，避免 class 声明。

---

## 实现思路

### `GameData` 的数据结构

```typescript
interface GameDataSchema {
    level: number;
}
```

很简单。后续章节如果加"皮肤"、"总通关数"之类再往里加字段。

### 读写时机

- **Home.onLoad**：读取 level → 更新文字。
- **Game 胜利**：level +1 → 保存 → 回 Home（Home 会重新读取显示新数字）。

### 关卡号 vs 关卡文件

- `level = 1` → 加载 `levels/level_01.json`
- `level = 5` → 加载 `levels/level_05.json`

当前只有 level_01.json，其他关卡第 19 章造。本章**先只保证 level 字段能存/读**，即使实际加载的永远是 level_01.json 也行。

## 代码实现

### 文件 1：`data/Storage.ts`（新增）

```typescript
import { sys } from 'cc';

const KEY_PREFIX = 'arrow.';

/**
 * 简易 localStorage 封装。
 * - 自动 JSON 序列化
 * - 读取失败（key 不存在、JSON 损坏、storage 不可用）返回 defaultValue
 */
export const Storage = {
    load<T>(key: string, defaultValue: T): T {
        try {
            const raw = sys.localStorage.getItem(KEY_PREFIX + key);
            if (raw === null || raw === undefined) return defaultValue;
            return JSON.parse(raw) as T;
        } catch (e) {
            console.warn(`[Arrow] Storage.load(${key}) failed`, e);
            return defaultValue;
        }
    },

    save<T>(key: string, value: T): void {
        try {
            sys.localStorage.setItem(KEY_PREFIX + key, JSON.stringify(value));
        } catch (e) {
            console.warn(`[Arrow] Storage.save(${key}) failed`, e);
        }
    },

    clear(key: string): void {
        try {
            sys.localStorage.removeItem(KEY_PREFIX + key);
        } catch (e) {
            console.warn(`[Arrow] Storage.clear(${key}) failed`, e);
        }
    },
};
```

**关键点**：

- **`KEY_PREFIX = 'arrow.'`**：避免和同域其他项目的 localStorage key 冲突。
- **每次读都 try-catch**：localStorage 在某些环境（隐私模式）不可用，坏掉的 JSON 也要兜住。**数据层不崩**，默认值顶上。
- **`sys.localStorage` 而不是 `window.localStorage`**：Cocos 3.8 封装版本，在原生平台（iOS/Android 编译后）用原生接口。

### 文件 2：`data/GameData.ts`（新增）

```typescript
import { Storage } from './Storage';

interface GameDataSchema {
    level: number;
}

const DEFAULT: GameDataSchema = {
    level: 1,
};

const STORAGE_KEY = 'gameData';

/**
 * 游戏全局数据（单例）。修改后必须调 save() 持久化。
 */
class _GameData {
    private data: GameDataSchema = { ...DEFAULT };
    private loaded = false;

    load() {
        if (this.loaded) return;
        this.data = Storage.load<GameDataSchema>(STORAGE_KEY, { ...DEFAULT });
        this.loaded = true;
        console.log('[Arrow] GameData loaded:', JSON.stringify(this.data));
    }

    save() {
        Storage.save(STORAGE_KEY, this.data);
    }

    get level(): number {
        if (!this.loaded) this.load();
        return this.data.level;
    }

    set level(v: number) {
        if (!this.loaded) this.load();
        this.data.level = v;
        this.save();
    }

    reset() {
        this.data = { ...DEFAULT };
        this.save();
    }
}

export const GameData = new _GameData();
```

**关键点**：

- **懒加载**：第一次访问 `GameData.level` 时才 load。避免 Home 场景的 onLoad 顺序问题。
- **setter 自动 save**：外部 `GameData.level = 5` 就保存。减少忘记 save 的机会。对应 G3_FBase 的 `defineStorageData()` 引擎托管——我们手动在 setter 里做。
- **class + `new` 实例 + export**：JS/TS 的单例习惯做法。class 私有，实例对外。比 `namespace GameData` 更 TypeScript-idiomatic。

## 易错点

### 易错 1：字段直接暴露，外部改了不 save

```typescript
class _GameData {
    data: GameDataSchema = { ...DEFAULT };  // ❌ public
}

// 外部：
GameData.data.level = 5;   // ← 不触发 save
```

**规则**：**数据字段永远 private + getter/setter**。或者提供 `update(fn)` 方法。

### 易错 2：localStorage key 冲突

```typescript
sys.localStorage.setItem('level', ...);  // ❌ 没前缀
```

本地调试时看不出问题。但同域下如果还有其他项目也用 'level'，会互相覆盖。

**规则**：**永远加项目前缀**。`arrow.level` / `arrow.gameData`。

### 易错 3：JSON 损坏导致崩溃

localStorage 写入过程中断（比如浏览器关闭），下次读到半条 JSON → `JSON.parse` 抛错。

```typescript
const data = JSON.parse(raw);  // ❌ 没 try-catch
```

**规则**：**所有外部数据边界必须 try-catch + 默认值**。

### 易错 4：Storage 在原生平台用错 API

```typescript
window.localStorage.setItem(...);  // ❌ 原生平台没 window
```

**规则**：**永远用 `sys.localStorage`**。Cocos 封装了 Web/Native 差异。

### 易错 5：存档版本升级时兼容问题

将来你给 GameDataSchema 加字段 `skin: string`：

```typescript
interface GameDataSchema {
    level: number;
    skin: string;   // 新加
}
```

老用户的存档里**没有 skin 字段**。load 后 `this.data.skin` 是 undefined。

**防御式写法**：

```typescript
load() {
    const loaded = Storage.load<Partial<GameDataSchema>>(STORAGE_KEY, {});
    this.data = { ...DEFAULT, ...loaded };   // ← 用 DEFAULT 填充缺失字段
    this.loaded = true;
}
```

**规则**：**load 后用 DEFAULT 补齐**。这是存档兼容性的基础。建议从一开始就这么写。

## 扩展练习

1. **"清档"按钮**：Home 场景加一个小小的 "Reset" 按钮（不用画得很显眼），点击 `GameData.reset()` → 回到 Level 1。调试用。

2. **存档版本号**：在 GameDataSchema 加 `version: number`。Load 时检查版本：如果和当前代码不一致，执行迁移函数。

3. **思考题**：G3_FBase 的存档结构比我们复杂得多（`combat.hp / combat.arrows[i].coords / reviveCount ...`）。这种存中途状态（箭头位置）的好处是：断电后可以接着玩同一局。成本是：每次 hp/arrows 变化都要调 save，磁盘 IO 变多。本教程为什么不做这一步？

   （答：教程级别保持简单。真实项目按需做，关卡短可以不存中途只存关卡号，关卡长则存中途。）
